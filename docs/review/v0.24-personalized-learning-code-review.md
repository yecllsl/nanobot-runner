### 重要 (应该修复)

[IMPORTANT] CalibrationEngine 存在测试后门 `_test_pairs` 属性

- **文件**: [calibration_engine.py:L50-L53](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/calibration_engine.py#L50-L53)
- **问题**: 生产类 `CalibrationEngine` 中通过 `getattr(self, "_test_pairs", None)` 检测是否存在测试注入数据。这违反了关注点分离原则：测试后门不应暴露在生产代码中。
- **影响**: 生产环境中若意外设置 `_test_pairs` 属性（如通过反射、mock 泄漏），校准行为会完全绕过 EvolutionStore 数据源，产生不可预期的校准结果。虽然概率极低，但这种模式在严格的代码审查中不应通过。
- **修复建议**: 方案一（推荐）：在 `run_calibration` 方法中新增可选参数 `override_pairs: list[tuple[float, float]] | None = None`，测试通过参数注入。方案二：使用 `unittest.mock.patch` 在测试中 mock `_store.get_prediction_actual_pairs` 方法，完全移除 `_test_pairs`。

[IMPORTANT] `evolution_engine.py` 中存在 6 处 `# type: ignore[union-attr]` 注释

- **文件**: [evolution_engine.py:L120](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L120), [evolution_engine.py:L125](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L125), [evolution_engine.py:L133](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L133), [evolution_engine.py:L136](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L136), [evolution_engine.py:L144-L146](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L144-L146), [evolution_engine.py:L158](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.24.0/src/core/evolution/evolution_engine.py#L158)
- **问题**: 项目 AGENTS.md 明确规定"禁止使用 `# type: ignore`：必须写出正确的类型注解"。这些 `type: ignore[union-attr]` 是为了绕过 `Optional[T] | None` 的类型收窄问题。
- **影响**: 违反项目编码规范。虽然 `_require_v024_component` 在运行时会先检查非空，但 mypy 无法推断这种收窄。
- **修复建议**: 将 `_require_v024_component` 改为类型守卫模式，使用 `assert self._response_analyzer is not None` 替代，mypy 可以正确识别 assert 后的类型收窄。这 6 处都可以替换为：

  ```python
  def analyze_training_response(self, months: int = 6) -> TrainingResponseReport:
      self._require_v024_component("response_analyzer")
      assert self._response_analyzer is not None  # mypy 类型收窄
      return self._response_analyzer.analyze(months)
  ```
