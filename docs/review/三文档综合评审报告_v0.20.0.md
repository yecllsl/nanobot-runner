### [CRITICAL-2] pyproject.toml 缺少 v0.20.0 核心依赖

**文件**: [pyproject.toml](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/pyproject.toml)

**问题描述**:

架构设计说明书第 2 节明确列出了 v0.20.0 的 4 个新增核心依赖：

| 依赖 | 文档要求版本 | pyproject.toml 状态 |
|------|-------------|---------------------|
| scikit-learn | ≥1.3.0 | ❌ **未添加** |
| scipy | ≥1.11.0 | ❌ **未添加** |
| shap | ≥0.43.0 | ❌ **未添加** |
| joblib | ≥1.3.0 | ❌ **未添加** |

当前 pyproject.toml 的 `dependencies` 列表中完全没有这些包。

**影响**:
1. 开发工程师启动 v0.20.0 开发时，`uv run` 无法解析 ML 相关 import

```toml
"scikit-learn>=1.3.0",
"scipy>=1.11.0",
"shap>=0.43.0",
"joblib>=1.3.0",
```

### [CRITICAL-3] numpy 2.4.2 与 scikit-learn 1.3.0+ 兼容性未验证

**文件**: [pyproject.toml](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/pyproject.toml) + [架构设计说明书](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/docs/architecture/架构设计说明书.md) 第 2 节

**问题描述**:

当前项目使用 `numpy>=2.4.2`（2025 年发布的最新大版本），而 scikit-learn 1.3.x 系列主要针对 numpy 1.x 测试。numpy 2.x 存在大量 API 变更（如 `np.bool` → `np.bool_`、`np.object` → `np.object_` 等），可能导致：
- scikit-learn 内部使用已废弃的 numpy API 而报错
- GradientBoostingRegressor 的 C 扩展编译失败
- SHAP 的 TreeExplainer 与 numpy 2.x 不兼容

**影响**:
1. `uv add scikit-learn` 后可能无法正常 import
2. 运行时可能出现 `AttributeError` 或 `DeprecationWarning`
3. 最坏情况下需要降级 numpy 或升级 scikit-learn 到支持 numpy 2.x 的版本

**整改方案**:
1. 在开发环境执行 `uv add scikit-learn scipy shap joblib` 并运行 `python -c "import sklearn; print(sklearn.__version__)"` 验证
2. 如不兼容，评估两个方向：
   - 降级 numpy 到 `>=1.26.0,<2.0.0`
   - 升级 scikit-learn 到 `≥1.5.0`（首个官方支持 numpy 2.x 的版本）
3. 将验证结果记录到架构文档的风险清单中

def race_prediction_engine(self) -> Any:
