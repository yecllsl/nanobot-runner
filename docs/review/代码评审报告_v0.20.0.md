### [CRITICAL-1] context.py 中 prediction_engine 属性传递了不存在的构造函数参数，导致运行时 TypeError

**文件**: [context.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/base/context.py) 第 296-320 行

**问题描述**:

`AppContext.prediction_engine` 属性在创建各预测器实例时，传递了 `session_repo` 参数，但实际类的构造函数不接受此参数：

```python
# context.py 实际代码（第 296-320 行）
vdot_predictor = VDOTPredictor(
    session_repo=self.session_repo,       # ❌ VDOTPredictor 不接受 session_repo
    feature_engine=feature_engine,
    data_assessor=data_assessor,
    model_manager=model_manager,
)
race_predictor = RacePredictor(
    session_repo=self.session_repo,       # ❌ RacePredictor 不接受 session_repo
    data_assessor=data_assessor,
)
injury_predictor = InjuryPredictor(
    session_repo=self.session_repo,       # ❌ InjuryPredictor 不接受 session_repo
    feature_engine=feature_engine,
    data_assessor=data_assessor,
    model_manager=model_manager,
)
training_response_predictor = TrainingResponsePredictor(
    session_repo=self.session_repo,       # ❌ TrainingResponsePredictor 不接受 session_repo
)
```

**实际构造函数签名**:

| 类 | 实际参数 |
|----|---------|
| `VDOTPredictor` | `feature_engine, data_assessor, model_manager, banister_model, base_vdot` |
| `RacePredictor` | `feature_engine, data_assessor, model_manager, current_vdot, race_records` |
| `InjuryPredictor` | `feature_engine, data_assessor, model_manager, rule_baseline, logistic_model` |
| `TrainingResponsePredictor` | `banister_model, base_vdot` |

**影响**:
1. 任何通过 `context.prediction_engine` 访问预测引擎的代码路径都会抛出 `TypeError: __init__() got an unexpected keyword argument 'session_repo'`
2. 所有 Agent 工具（`predict_vdot_trend` 等 5 个）和 CLI 命令（`predict vdot/race/injury/response/status`）全部不可用
3. 这是 **阻塞性运行时错误**，代码完全无法工作

**整改方案**:

移除 `session_repo` 参数，补充缺失的必要参数：

```python

from src.core.prediction.baselines.banister_ir import BanisterIRModel

race_predictor = RacePredictor(
    feature_engine=feature_engine,

injury_predictor = InjuryPredictor(
```

### [HIGH-1] 多个预测器的 ML 增强层使用硬编码占位值，未实现真实 ML 推理

**涉及文件**:
- [vdot_predictor.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/prediction/vdot_predictor.py) 第 107-140 行 (`_predict_ml_enhanced`)
- [injury_predictor.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/prediction/injury_predictor.py) 第 82-140 行 (`_predict_ml_enhanced`)

**问题描述**:

两个核心预测器的 ML 增强层使用了硬编码的占位值，而非真实的 ML 模型推理：

`vdot_predictor.py`:
```python
predicted = self._base_vdot + 0.05 * days * 0.01  # 硬编码线性增长
std_dev = 0.3 + 0.01 * days                        # 硬编码标准差
```

`injury_predictor.py`:
```python
lr_proba = 0.3       # 硬编码逻辑回归概率
gbdt_proba = 0.35    # 硬编码 GBDT 概率
```

**影响**:
1. 即使数据充足（400+条），ML 增强预测返回的仍是假数据
2. 用户看到的 `prediction_type="ml_enhanced"` 实际是虚假标注
3. 与架构文档声称的"ML增强预测显著提升准确度"严重不符

**整改方案**:
1. 在 `vdot_predictor._predict_ml_enhanced` 中实现真实的 GradientBoostingRegressor 推理
2. 在 `injury_predictor._predict_ml_enhanced` 中实现真实的 LR+GBDT 集成推理
3. 如果 ML 模型未训练，应降级到参数化层而非返回假数据
4. 或者明确标注当前为"开发占位"，在 `prediction_type` 中增加 `"placeholder"` 标识

### [HIGH-3] prediction_handler.py 中 _get_engine() 的降级逻辑创建了无子预测器的空引擎

**文件**: [prediction_handler.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/cli/handlers/prediction_handler.py) 第 18-22 行

**问题描述**:

```python
def _get_engine(self) -> Any:
    engine = self.context.prediction_engine
    if engine is None:
        engine = PredictionEngine()  # ❌ 无任何子预测器的空引擎
    return engine
```

当 `context.prediction_engine` 为 None 时（例如在测试环境或配置不完整时），创建了一个没有任何子预测器的空 `PredictionEngine()`。调用任何预测方法都会因 `self._vdot_predictor is None` 而抛出 `AttributeError`。

**影响**:
1. 降级路径不可用，反而引入新的崩溃点
2. 与架构文档的"绝不阻塞用户"原则相悖

**整改方案**:
```python
def _get_engine(self) -> Any:
    engine = self.context.prediction_engine
    if engine is None:
        raise CLIError.config_error("预测引擎未初始化，请先运行 system init")
    return engine
```

### [HIGH-4] PredictionConfig.from_env() 中类型判断逻辑有 bug

**文件**: [config.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/prediction/config.py) 第 143-149 行

**问题描述**:

```python
elif f.type in ("int", int):       # ❌ f.type 返回的是 type 对象（如 <class 'int'>），不是字符串
    with contextlib.suppress(ValueError):
        kwargs[f.name] = int(val)
elif f.type in ("float", float):   # ❌ 同上
    with contextlib.suppress(ValueError):
        kwargs[f.name] = float(val)
```

`dataclasses.Field.type` 返回的是 Python 类型对象（如 `<class 'int'>`），而非字符串 `"int"`。因此 `f.type in ("int", int)` 中的字符串比较永远不会匹配，导致环境变量中的整型和浮点型配置项无法被正确解析。

**影响**:
1. 通过环境变量 `NANOBOT_PREDICTION_*` 设置的整型/浮点型配置全部被忽略
2. 用户无法通过环境变量覆盖默认配置值

**整改方案**:
```python
elif f.type is int:
    with contextlib.suppress(ValueError):
        kwargs[f.name] = int(val)
elif f.type is float:
    with contextlib.suppress(ValueError):
        kwargs[f.name] = float(val)
```

### [MEDIUM-3] banister_ir.py 存在 mypy 类型错误

**文件**: [banister_ir.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/prediction/baselines/banister_ir.py) 第 86 行

**问题描述**:

```python
def _banister_func(
    self, stress: np.ndarray, tau1: float, tau2: float, k1: float, k2: float
) -> np.ndarray:                    # 声明返回 np.ndarray
    fitness = self._ewma_sum(stress, tau1)
    fatigue = self._ewma_sum(stress, tau2)
    return k1 * fitness - k2 * fatigue  # 实际返回 float
```

`_ewma_sum` 返回 `float`，因此 `k1 * fitness - k2 * fatigue` 也是 `float`，但函数签名声明返回 `np.ndarray`。

**影响**:
- mypy 类型检查失败
- 可能导致 `curve_fit` 行为异常（期望返回与 ydata 同形的数组）

**整改方案**:
```python
def _banister_func(
    self, stress: np.ndarray, tau1: float, tau2: float, k1: float, k2: float
) -> np.ndarray:
    n = len(stress)
    result = np.zeros(n)
    for i in range(n):
        fitness = self._ewma_sum(stress[:i+1], tau1)
        fatigue = self._ewma_sum(stress[:i+1], tau2)
        result[i] = k1 * fitness - k2 * fatigue
    return result
```

### [MEDIUM-5] vdot_predictor.py 中降级判断逻辑与架构文档不一致

**文件**: [vdot_predictor.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/src/core/prediction/vdot_predictor.py) 第 48-56 行
