### 4.3 类型注解检查 ⚠️

| ID | 严重度 | 位置 | 问题 |
|----|--------|------|------|
| **TYPE-01** | P1 | `activities.py:26` `context: Any` | 应使用 `AppContext` 类型（如 `from src.core.base.context import AppContext`） |
| **TYPE-02** | P1 | `body_signal.py:17` `context: Any` | 同上，多处同步辅助函数使用 `Any` |
| **TYPE-03** | P1 | `dashboard.py:17` `context: Any` | 同上 |
| **TYPE-04** | P1 | `plan.py:26` `context: Any` | 同上 |
| **TYPE-05** | P1 | `evolution.py:23` `context: Any` | 同上 |
| **TYPE-06** | P1 | `settings.py:34` `context: Any` | 同上 |

> **说明**: 6 个路由文件中的所有同步辅助函数均使用 `context: Any` 作为参数类型。由于项目规范要求核心模块类型注解覆盖率 ≥ 80%，建议统一替换为 `AppContext`。但考虑到这些是路由层辅助函数（非核心模块），且 `TYPE_CHECKING` 导入可避免循环依赖，此问题评为 P1。
