## 4. 问题详细分析

### M-01: `_extract_plan_id` 键名不一致

**位置**: [outcome_collector.py:L479-L484](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L479-L484)

**根因**: `DecisionLogHook.before_execute_tools()` 捕获工具调用时使用的键是 `"arguments"`:

```python
# decision_log_hook.py:L116
self._tool_call_chain.append({
    "id": tool_id,
    "name": tool_name,
    "arguments": tool_args,  # 键为 "arguments"
})
```

但 `_extract_plan_id` 中访问的是 `"args"`:

```python
# outcome_collector.py:L481
args = call.get("args", {})  # 应为 "arguments"
```

**建议修复**:
```python
# 修改为
args = call.get("arguments", {})
```

### M-02: Handler 访问私有成员

**位置**: [evolution_handler.py:L127](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/cli/handlers/evolution_handler.py#L127), [L145](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/cli/handlers/evolution_handler.py#L145)

**问题**:
- `engine.outcome_collector._compute_accuracy_stats()` — 调用私有方法
- `engine.outcome_collector._store.query_outcomes()` — 访问私有属性

**建议修复**:
1. 在 `OutcomeCollector` 新增公开方法 `get_accuracy_stats(days: int)` 和 `get_fidelity_stats(days: int)`
2. Handler 通过这些公开方法获取统计数据
3. 确保 `days` 参数实际生效于日期过滤

### M-03: Docstring 与实现不一致

**位置**: [outcome_collector.py:L386](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L386)

**问题**: `generate_feedback_prompt` 方法的 docstring 写 "使用ConfirmScenario.GENERIC场景"，但代码实际使用 `ConfirmScenario.DECISION_FEEDBACK`

**修复**: 将 docstring 改为 "使用ConfirmScenario.DECISION_FEEDBACK场景"
