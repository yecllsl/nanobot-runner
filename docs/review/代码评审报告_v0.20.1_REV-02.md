### [NEW-P1-1] VDOTPredictor._build_training_data() 直接实例化 VDOTCalculator，违反依赖注入规范

**文件**: [vdot_predictor.py:L198-L200](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.20.1/src/core/prediction/vdot_predictor.py#L198-L200)

**问题描述**:

```python
def _build_training_data(self, vdot_sessions: list[Any]) -> ...:
    from src.core.calculators.vdot_calculator import VDOTCalculator
    calculator = VDOTCalculator()  # 直接实例化，未使用注入的实例
```

VDOTPredictor 的 `__init__` 未接收 `vdot_calculator` 参数，但 `FeatureEngine` 已注入 `vdot_calculator`。`_build_training_data()` 应复用 `self._feature_engine._vdot_calculator` 或通过构造函数注入。

**影响**: 创建了不必要的重复实例，且无法在测试中替换 VDOTCalculator 的行为。

**修复方案**: 
1. 通过 `self._feature_engine._vdot_calculator` 获取已注入的计算器实例
2. 或在 VDOTPredictor 构造函数中增加 `vdot_calculator` 参数

### [NEW-P2-1] FeatureEngine._safe_float() 使用 `Any` 类型降低类型安全

**文件**: [feature_engine.py:L333](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.20.1/src/core/prediction/feature_engine.py#L333)

**问题描述**:

```python
def _safe_float(self, name: str, func: Any) -> float:  # func: Any
```

`func` 参数应为 `Callable[[], float]`，使用 `Any` 降低了类型检查的有效性。

**修复方案**:

```python
from typing import Callable

def _safe_float(self, name: str, func: Callable[[], float]) -> float:
```

---

### [NEW-P2-2] 测试中 `_make_feature_engine()` 返回的 Mock 缺少 `data_quality` 字段

**文件**: [test_vdot_predictor.py:L28-L34](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.20.1/tests/unit/core/prediction/test_vdot_predictor.py#L28-L34), [test_injury_predictor.py:L28-L34](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.20.1/tests/unit/core/prediction/test_injury_predictor.py#L28-L34)

**问题描述**: 测试 Mock 的 `FeatureMatrix` 未设置 `data_quality` 属性，而真实 `FeatureEngine.extract_vdot_features()` 返回的 `FeatureMatrix` 包含 `data_quality` 字段。如果生产代码依赖此字段做降级判断，测试将无法覆盖。

**修复方案**: 在 Mock 的 `FeatureMatrix` 中添加 `matrix.data_quality = "sufficient"`。
