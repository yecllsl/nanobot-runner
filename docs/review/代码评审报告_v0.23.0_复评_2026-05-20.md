## 3. 问题清单

### 3.1 HIGH 级问题

---

[HIGH] **`calculate_fidelity()` 和 `calculate_prediction_error()` 存在除零风险**

- **文件**: [outcome_collector.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L155-L174)
- **Issue**: 
  1. `calculate_fidelity()` 第 140-144 行：当 `planned_volume_km` 或 `planned_duration_min` 为 0 时触发 `ZeroDivisionError`
  2. `calculate_prediction_error()` 第 167 行：当 `actual` 为 0 时触发 `ZeroDivisionError`

  这两个函数都是纯函数（无状态），调用方依赖它们返回 float 值而不预期异常。当前调用方 `check_plan_execution()` 和 `check_prediction_accuracy()` 未对这些异常做防御。

- **Fix**:
  ```python
  # calculate_fidelity 增加守卫
  if data.planned_volume_km == 0 or data.planned_duration_min == 0:
      return 1.0
  
  # calculate_prediction_error 增加守卫
  if actual == 0:
      return (0.0, "accurate") if predicted == 0 else (100.0, "overestimate" if predicted > 0 else "underestimate")
  ```

### 3.2 MEDIUM 级问题

---

[MEDIUM] **`OutcomeCollector.generate_feedback_prompt()` 使用 `ConfirmScenario.GENERIC` 而非 `DECISION_FEEDBACK`**

- **文件**: [outcome_collector.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L383)
- **Issue**: 当前代码 `scenario=ConfirmScenario.GENERIC`。`ConfirmScenario.DECISION_FEEDBACK` 枚举值已在 [ask_user_confirm.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/plan/ask_user_confirm.py#L32) 中定义，`AskUserConfirmManager.create_decision_feedback_prompt()` 也正确使用它。但 `OutcomeCollector.generate_feedback_prompt()` **未使用**专用场景枚举，而是回退到 `GENERIC`。

  架构设计说明书 Section 8.2.2 明确要求 "扩展 ConfirmScenario.DECISION_FEEDBACK"，意味着业务代码应使用该场景值。整改报告 L-01 标注为"已修复"但实际该行代码未被修改。

- **Fix**: 将 `confirmScenario.GENERIC` 改为 `ConfirmScenario.DECISION_FEEDBACK`，并考虑复用 `AskUserConfirmManager.create_decision_feedback_prompt()` 而非自行构造 ConfirmPrompt（后者有 3 个选项 accept/modify/reject，更符合决策反馈场景）。

---

[MEDIUM] **`EvolutionEngine.get_evolution_status()` 返回字段不完整**

- **文件**: [evolution_engine.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/evolution_engine.py#L176-L195)
- **Issue**: 架构设计说明书 Section 8.2.5 定义 `get_evolution_status()` 应返回：**总决策数/回填率/平均忠实度/平均预测误差/反馈收集率/数据质量指标**。当前实现仅返回 `total_decisions`、`status_distribution`、`type_distribution` 三个字段，缺少回填率、平均忠实度、平均预测误差、反馈收集率、数据质量指标。
- **Fix**: 补充 `outcome_statistics` 字段，调用 `self._outcome_collector._compute_accuracy_stats()` 获取预测精度，调用 `self._store.get_decision_outcome_pairs()` 计算回填率和平均忠实度。或将 `_compute_accuracy_stats` 提升为公开方法。

### 3.3 LOW 级问题

---

[LOW] **`get_decision_outcome_pairs()` 未传入日期范围，全量扫描 outcomes**

- **文件**: [evolution_store.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/evolution_store.py#L622)
- **Issue**: 第 624 行 `self._scan_outcomes()` 未传入 `start_date`/`end_date` 参数，扫描全部 outcomes 文件。当 outcome 数据量增长后，此调用会成为性能瓶颈。
- **Fix**: 将 `start_date`/`end_date` 参数传入 `_scan_outcomes()`，与 decisions 的日期范围保持一致。

---

[LOW] **bandit 安全扫描工具未安装，缺少自动化安全扫描能力**

- **Fix**: 在 dev 依赖中添加 bandit：`uv add --dev bandit`，或在 CI 流程中集成安全扫描步骤。

---

[LOW] **`runner_state` 类型注解为 `dict[str, Any]` 不够精确**

- **文件**: [models.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/models.py#L37)
