| H-01 | `calculate_fidelity()` + `calculate_prediction_error()` 除零风险 | ✅ 已修复 | 添加零值守卫条件 |

**H-01 整改详情**:

1. **`calculate_fidelity()`** — [outcome_collector.py:L137-L139](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L137-L139)
   - 新增守卫: `if data.planned_volume_km == 0 or data.planned_duration_min == 0: return 1.0`
   - 语义: 计划值为 0 表示无计划偏差，忠实度为 1.0

2. **`calculate_prediction_error()`** — [outcome_collector.py:L168-L172](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/src/core/evolution/outcome_collector.py#L168-L172)
   - 新增守卫: `if actual == 0:` 分支处理
   - `actual == 0 and predicted == 0` → `(0.0, "accurate")` — 预测完全准确
   - `actual == 0 and predicted > 0` → `(100.0, "overestimate")` — 高估
   - `actual == 0 and predicted < 0` → `(100.0, "underestimate")` — 低估

3. **新增测试用例** — [test_outcome_collector.py](file:///d:/yecll/Documents/LocalCode/RunFlowAgent.worktrees/feature-0.23.0/tests/unit/core/evolution/test_outcome_collector.py)

   - `TestCalculateFidelity.test_both_planned_zero_returns_one`
   - `TestCalculatePredictionError.test_zero_actual_and_zero_predicted`
   - `TestCalculatePredictionError.test_zero_actual_positive_predicted`
   - `TestCalculatePredictionError.test_zero_actual_negative_predicted`
