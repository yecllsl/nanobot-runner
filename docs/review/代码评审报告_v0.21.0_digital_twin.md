## 3. 整改详情

### 3.1 P1-01: 评分算法重写

**修改文件**: [digital_twin_engine.py](../../src/core/twin/digital_twin_engine.py)

**修改内容**:
- 旧算法: `vdot_delta * 10 + tsb * -2 + risk * -1`（简单线性加减，无归一化）
- 新算法: 按架构7.7节实现归一化+权重分配
  - VDOT评分: `min(100, max(0, vdot_delta * 20))` × 0.4
  - 风险评分: `max(0, 100 - peak_injury_risk)` × 0.35
  - 恢复评分: `(tsb_score * 0.5 + recovery_status_score * 0.5)` × 0.25
  - 总分: `round(vdot_score * 0.4 + risk_score * 0.35 + recovery_score * 0.25, 2)`

**新增测试**:
- `test_score_in_0_100_range`: 验证评分范围
- `test_high_vdot_high_score`: 验证高VDOT计划得分更高
- `test_weights_sum_to_one`: 验证权重和为1
- `test_vdot_delta_zero_gives_zero_vdot_score`: 验证零VDOT增量边界

### 3.2 P1-02: 状态向量缓存实现

**修改文件**: [digital_twin_engine.py](../../src/core/twin/digital_twin_engine.py)

**修改内容**:
- 添加 `_cache: StateVectorCache | None` 内存缓存属性
- `get_current_snapshot()` 实现三层缓存策略:
  1. 内存缓存有效 → 直接返回
  2. 磁盘缓存有效 → 加载返回
  3. 缓存失效 → 重新构建并持久化
- 添加 `_load_cache_from_disk()` / `_save_cache_to_disk()` 方法
- 添加 `_dict_to_*()` / `_str_to_data_quality()` 反序列化辅助方法
- 缓存文件路径: `{cache_dir}/twin/state_vector.json`

**新增测试**:
- `test_uses_memory_cache_on_second_call`: 验证内存缓存命中
- `test_disk_cache_hit`: 验证磁盘缓存命中
- `test_disk_cache_expired_rebuilds`: 验证过期缓存重建
- `test_saves_cache_to_disk`: 验证缓存持久化
