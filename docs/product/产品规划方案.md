## 1. 产品愿景与定位

### 1.1 产品愿景

**成为技术型跑者首选的本地化 AI 跑步数据分析工具**

Nanobot Runner 致力于解决数据隐私与深度分析的矛盾，通过本地化的数据存储和 AI 分析能力，让用户完全掌控自己的运动数据，同时享受企业级 BI 分析体验。

### 1.2 核心价值主张

| 维度 | 价值主张 |
|------|----------|
| **隐私优先** | 本地存储，零外联设计，数据完全用户可控 |
| **专业分析** | 基于运动科学的专业指标（VDOT、TSS、心率漂移等） |
| **AI 赋能** | 自然语言交互，智能训练建议 |
| **高性能** | Polars + Parquet，查询性能提升 ≥ 20% |
| **开发者友好** | 完善的 CLI 工具、API 文档和扩展能力 |

### 1.3 产品定位

**目标市场**: 个人跑步数据管理工具（细分市场）

**差异化定位**:
- 与 Garmin Connect / Strava 对比：数据本地化，隐私可控
- 与 TrainingPeaks 对比：免费开源，可定制性强
- 与一般跑步 App 对比：专业分析深度，AI 交互能力

### 1.4 目标用户画像

#### 主要用户：技术型严肃跑者

| 属性 | 描述 |
|------|------|
| **年龄段** | 25-45 岁 |
| **职业背景** | 程序员、工程师、数据分析师等技术从业者 |
| **跑步经验** | 规律跑步 2 年以上，有马拉松参赛经历 |
| **数据意识** | 习惯佩戴心率带、功率计等专业设备 |
| **核心痛点** | 担心云端数据隐私，需要更专业的数据分析 |
| **技术能力** | 熟悉命令行操作，具备一定编程基础 |

#### 次要用户：跑步数据分析师

| 属性 | 描述 |
|------|------|
| **使用场景** | 分析运动员训练数据，制定训练计划 |
| **核心需求** | 批量数据处理、自定义分析脚本、报告生成 |
| **技术能力** | Python 数据分析能力，熟悉 Pandas/Polars |

#### v0.9.5 版本定位

**版本主题**: 架构解耦与初始化流程整合

**核心策略**: 全面使用nanobot-ai提供的所有模块能力，不重复造轮子，仅替换配置管理层。

**技术收益**:
- 用户无需单独安装nanobot，降低使用门槛
- 配置统一管理，消除双配置系统困惑
- 初始化流程整合，提升用户体验
- 向后兼容已使用nanobot的用户

**关键交付物**:
- ProviderAdapter配置注入层
- NanobotConfigSync单向同步器
- 初始化流程整合（三种用户场景）
- 用户迁移路径文档

### 3.2 智能跑步计划版本：v0.10.x - v0.12.x

**版本目标**: 构建数据驱动的自适应智能跑步计划系统

#### v0.10.0 数据感知版（2026-04-20 ~ 2026-05-05）

**核心目标**: 建立训练执行数据的收集与分析能力

| 功能模块 | 实现方式 | 验收标准 |
|----------|----------|----------|
| 训练执行反馈收集 | CLI命令 + Agent工具 | 支持记录完成度、体感评分、备注 |
| 计划完成度跟踪 | 数据模型扩展 + 查询工具 | 可查询任意计划的执行统计 |
| 历史响应模式分析 | 规则引擎分析历史数据 | 识别用户对不同训练类型的响应 |

**新增Agent工具**:
- `RecordPlanExecutionTool` - 记录计划执行情况
- `GetPlanExecutionStatsTool` - 获取计划执行统计
- `AnalyzeTrainingResponseTool` - 分析训练响应模式

**CLI命令**:
```bash
uv run nanobotrun plan log --plan-id <id> --date <date> --completion 0.9 --effort 7
uv run nanobotrun plan stats --plan-id <id>
```

#### v0.11.0 智能调整版（2026-05-05 ~ 2026-05-25）

**核心目标**: 通过LLM实现智能计划调整和自然语言交互

| 功能模块 | 实现方式 | 验收标准 |
|----------|----------|----------|
| LLM驱动的计划调整 | Prompt工程 + 规则约束 | 调整建议符合运动科学原则 |
| 自然语言计划修改 | Agent多轮对话 | 支持"下周减量"等自然语言指令 |
| 上下文感知建议 | Agent记忆 + 用户画像 | 建议考虑用户历史偏好和当前状态 |

**新增Agent工具**:
- `AdjustPlanTool` - 调整训练计划（LLM驱动）
- `GetPlanAdjustmentSuggestionsTool` - 获取计划调整建议

**典型交互场景**:
```
用户: "下周我要出差，帮我调整计划"
Agent: [分析当前计划] → [调用AdjustPlanTool] → 
      "已为您调整：下周跑量从50km减至30km，长距离跑提前到今天"
```

**Prompt设计原则**:
- 注入用户画像（VDOT、周跑量、经验水平）
- 注入历史执行数据（完成率、体感）
- 注入运动科学约束（10%规则、阶段特点）
- 要求结构化输出（JSON格式调整指令）

#### v0.12.0 预测与规划版（2026-05-25 ~ 2026-06-15）

**核心目标**: 实现目标达成评估和长期训练规划

| 功能模块 | 实现方式 | 验收标准 |
|----------|----------|----------|
| 目标达成评估 | LLM推理 + 历史数据对比 | 给出目标达成概率和关键风险 |
| 长期周期规划 | 多周期计划生成 | 支持年度/赛季多周期规划 |
| 智能训练建议 | 上下文感知的实时建议 | 每次对话都能给出个性化建议 |

**新增Agent工具**:
- `EvaluateGoalAchievementTool` - 评估目标达成概率
- `CreateLongTermPlanTool` - 创建长期训练规划
- `GetSmartTrainingAdviceTool` - 获取智能训练建议

**典型交互场景**:
```
用户: "我想在10月跑北马，目标330，现实吗？"
Agent: [分析当前VDOT、训练基础、剩余时间] →
      "达成概率约65%。关键风险：当前周跑量偏低，建议未来8周逐步提升至60km"
```
