## 1. 产品愿景

### 1.1 核心价值主张

让拥有2年+跑步数据的资深跑者，拥有一个**真正懂自己、能预测未来、会自我进化**的 AI 训练科学家。

**四阶段愿景叙事**：

| 阶段 | 口号 | 核心能力 | 对应版本 | 状态 |
|------|------|----------|----------|------|
| **记录跑步** | "你的跑步数据管家" | FIT解析、数据存储、基础统计 | v0.5-v0.19 | ✅ 完成 |
| **预测跑步** | "你的数字孪生跑者" | ML增强预测、What-If推演、风险预警 | v0.20-v0.22 | ✅ 完成 |
| **进化跑步** | "越用越懂你的私人教练" | 决策追踪、自适应学习、个性化进化 | v0.23-v0.25 | ✅ 完成 |
| **交互升级** | "随时随地与你对话的AI教练" | 底座升级、新特性适配、WebUI | v0.26-v0.29 | 📋 规划中 |

### 1.2 战略维度

| 战略维度 | 用户价值 |
|----------|----------|
| **隐私可控** | 跑者的数据只属于自己，不上传、不泄露 |
| **专业可信** | 基于运动科学的分析，AI建议可追溯、可验证 |
| **预测未来** | 基于2年+数据，预测伤病风险、VDOT趋势、比赛成绩 |
| **自我进化** | 系统从每次训练和反馈中学习，越来越懂个人特点 |
| **随时可达** | 通过WebUI随时随地与AI教练对话，数据可视化一目了然 |

### 1.3 目标用户分层

| 用户层级 | 数据量 | 可用能力 | 产品体验 |
|----------|--------|----------|----------|
| **初级** | <6个月 | 基础统计、简单趋势 | 感受"数据被记录" |
| **中级** | 6-18个月 | 线性回归预测、Jack Daniels公式 | 感受"数据在说话" |
| **高级** | 18个月+ | ML增强预测 | 感受"数据懂自己" |
| **资深** | 24个月+ | 数字孪生推演 + 自适应进化 + WebUI | 感受"系统在进化，随时可对话" |

## 3. Phase D：交互升级（v0.26 - v0.29）

### 3.1 Phase 概述

**背景**：nanobot-ai 0.2.0 发布，引入 GoalState、FallbackProvider、WebUI 内置打包、Model Presets、推理可见化等重要新能力。RunFlowAgent 需要升级底座并规划新特性适配，同时建设 WebUI 交互能力。

**Phase 目标**：
1. 安全升级底座到 nanobot-ai 0.2.0，零功能回归
2. 适配三项关键新特性，增强现有功能
3. 建设WebUI交互能力，从纯CLI进化到Web交互
4. 评估剩余新特性，决定是否纳入后续版本

**设计文档**：[底座升级设计文档](../superpowers/specs/2026-05-23-nanobot-0.2.0-upgrade-design.md)

### 3.2 v0.26.0 — 底座升级 + 新特性适配

**主题**：nanobot-ai 0.2.0 底座升级 + GoalState/推理可见化/Model Presets 适配

| 维度 | 内容 |
|------|------|
| 核心目标 | 将 `nanobot-ai>=0.1.5.post2` 升级为 `nanobot-ai>=0.2.0`，全量测试通过，适配三项新特性 |
| 关键工作 | 依赖升级、API兼容性检查、测试修复、配置迁移、依赖冲突预检、GoalState/推理可见化/Model Presets 适配 |
| 策略 | 底座升级先行验证兼容性，确认无破坏性变更后同版本适配新特性 |
| 不做的事 | 不适配 FallbackProvider/MCP Resources/Prompts（独立评估）、不做 WebUI、不重构现有代码 |

**验收标准**：

**底座升级**：
1. `uv run pytest tests/` 全量通过
2. `uv run ruff check src/` 无新增错误
3. CLI 核心命令功能正常
4. 核心 CLI 命令响应时间不退化 >20%（以 v0.25.0 为基线）
5. `uv pip install --dry-run` 无依赖冲突

**新特性**：
6. 创建训练计划后，Agent在新对话中能回忆当前计划目标
7. 飞书对话中可见Agent推理过程
8. 可通过配置定义多个Model Presets，飞书/WebUI 中通过 `/model <preset>` 切换预设（nanobot-ai 内置命令）

### 3.3 v0.27.0 — WebUI 基础

**主题**：复用 nanobot-ai 原生 WebUI，实现 AI 对话交互 + 基础设置

| 功能 | 来源 |
|------|------|
| AI 对话交互 | 原生 WebUI |
| 流式输出 | 原生 WebSocket |
| 推理展示 | 原生 WebUI（v0.26.0 已适配） |
| 多会话管理 | 原生 WebUI |
| 基础设置 | 原生 WebUI |
| GoalState 展示 | 原生 WebUI |
| 品牌自定义 | 原生配置 |

**WebUI 可行性论证**：

| 维度 | 论证 |
|------|------|
| 工具调用 | RunFlowAgent 所有工具已注册为 nanobot-ai 工具，WebSocket 协议天然支持 |
| 流式对话 | WebSocket 多路复用协议支持 delta/stream_end/reasoning_delta |
| 会话管理 | 原生 WebUI 支持多会话、标题自动生成、历史记录 |
| 部署方式 | WebUI 随 wheel 分发，`nanobot gateway` 启动即可用 |

**后遗症及缓解**：

| 后遗症 | 缓解措施 |
|--------|----------|
| 上游 UI 变更影响 | 锁定 nanobot-ai 版本；自定义组件独立维护 |
| 跑步数据无图表 | v0.28.0 扩展；v0.27.0 通过 Agent 文本/表格输出 |
| 品牌不一致 | 通过 bot_name/bot_icon 配置自定义 |

**验收标准**：
1. 浏览器访问 WebUI 可正常对话
2. 所有 RunFlowAgent 工具在 WebUI 中可正常调用
3. 流式输出、推理展示正常

### 3.4 v0.28.0 — WebUI 数据可视化

**主题**：扩展 WebUI，增加跑步数据可视化能力

| 功能 | 实现方式 |
|------|----------|
| VDOT 趋势图 | 扩展 WebUI 前端组件 |
| 训练负荷图（ATL/CTL/TSB） | 扩展 WebUI 前端组件 |
| 身体信号面板（HRV/疲劳度/恢复） | 扩展 WebUI 前端组件 |
| 数据概览仪表盘 | 扩展 WebUI 前端组件 |

**技术方案**：后端新增 WebUI 专用 API 端点 + 前端新增"数据"标签页 + Recharts/ECharts 渲染

**验收标准**：
1. WebUI 中可查看 VDOT 趋势图、训练负荷图
2. 图表数据与 CLI 命令输出一致
3. 图表支持时间范围筛选（7/30/90/365天）

### 3.5 v0.29.0 — WebUI 管理控制台

**主题**：训练计划管理 + 进化引擎控制台

| 功能 | 描述 |
|------|------|
| 训练计划查看 | 查看当前训练计划、每日安排 |
| 计划执行进度 | 查看计划完成率、忠实度 |
| 计划调整 | 通过对话调整计划 |
| 进化状态面板 | 查看进化引擎状态、触发条件 |
| 提示参数调整 | 可视化调整 PromptTuner 4维参数 |
| 月度进化报告 | 查看历史进化报告 |

**验收标准**：
1. WebUI 中可查看当前训练计划及执行进度
2. 可通过对话调整训练计划
3. 进化引擎状态可视化展示
4. PromptTuner 参数可通过滑块调整

### 3.6 独立评估任务（不占版本号）

**主题**：FallbackProvider + MCP Resources/Prompts 评估

> 说明：此两项工作为评估性质，根据评估结论决定是否在后续版本中实施，不单独占用版本号。

| 评估项 | 评估要点 |
|--------|----------|
| **FallbackProvider** | 多 Provider 配置需求、故障转移对建议一致性的影响、断路器与 PromptTuner 交互 |
| **MCP Resources/Prompts** | 当前 MCP Tool 映射是否已满足、Resources/Prompts 增量价值、与 WebUI 数据可视化协同 |

**验收标准**：
1. 完成 FallbackProvider 评估报告，明确是否采纳及实施方案
2. 完成 MCP Resources/Prompts 评估报告，明确是否采纳及实施方案
3. 若采纳，相关功能开发完成并通过测试

## 5. 风险与缓解

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|----------|
| 决策日志数据膨胀 | 中 | 长期运行后日志过大 | Parquet按月分片；自动归档旧数据 |
| 用户反馈稀疏 | 高 | 缺乏足够反馈驱动进化 | 轻量反馈机制；主动询问；隐式反馈自动记录 |
| 进化方向偏差 | 中 | 系统学习到的偏好与真实目标偏离 | 保留人工覆盖机制；月度进化报告review；reset_to_default()回退 |
| nanobot-ai 0.2.0 API breaking change | 低 | 升级后现有功能异常 | 0.2.0 对 AgentHook 接口纯增量扩展，无破坏性变更；v0.26.0 全量测试覆盖 |
| 间接依赖冲突 | 中 | nanobot-ai 0.2.0 间接依赖与现有依赖冲突 | `uv pip install --dry-run` 预检 |
| 原生 WebUI 不支持跑步数据可视化 | 中 | v0.27.0 无图表能力 | v0.27.0 先验证基础链路，v0.28.0 再扩展 |
| 上游 WebUI 版本变更影响 | 低 | nanobot-ai WebUI 更新影响 RunFlowAgent | 锁定 nanobot-ai 版本；自定义组件独立维护 |
| FallbackProvider 推迟导致 LLM 调用稳定性风险 | 低 | 单 Provider 故障时无容错 | 当前单 Provider 场景下风险可控；独立评估后决定是否纳入后续版本 |
| MCP Resources/Prompts 推迟导致工具扩展性受限 | 低 | MCP 生态利用不充分 | 当前 MCP Tool 映射已满足需求；独立评估后决定是否纳入后续版本 |

### 6.2 术语表

| 术语 | 定义 |
|------|------|
| **ML增强预测** | 使用机器学习模型替代简单统计模型的预测能力 |
| **数字孪生** | 基于数据构建的、可推演的跑者生理模型 |
| **What-If推演** | 模拟不同训练方案下的状态演变 |
| **DecisionLog** | AI决策日志，记录决策完整上下文 |
| **OutcomeRecord** | 决策执行结果记录，通过decision_id关联DecisionLog |
| **执行忠实度** | 计划训练vs实际训练的吻合程度(0-1) |
| **偏差修正层** | CalibrationEngine，在预测输出上加bias+scale线性修正 |
| **参数化提示调优** | PromptTuner，4维参数空间调整LLM输出风格 |
| **GoalState** | nanobot-ai 0.2.0 持久化目标系统，支持跨轮次目标追踪 |
| **FallbackProvider** | nanobot-ai 0.2.0 故障转移链，主备模型自动切换+断路器 |
| **Model Presets** | nanobot-ai 0.2.0 命名预设系统，快速切换模型+参数组合 |
| **推理可见化** | nanobot-ai 0.2.0 Chain-of-Thought 流式展示能力 |
| **MCP Resources/Prompts** | MCP 协议三要素中的资源和提示模板映射 |
