## 1. 产品愿景

### 1.1 核心价值主张

让拥有2年+跑步数据的资深跑者，拥有一个**真正懂自己、能预测未来、会自我进化**的 AI 训练科学家。

**四阶段愿景叙事**：

| 阶段 | 口号 | 核心能力 | 对应版本 | 状态 |
|------|------|----------|----------|------|
| **记录跑步** | "你的跑步数据管家" | FIT解析、数据存储、基础统计 | v0.5-v0.19 | ✅ 完成 |
| **预测跑步** | "你的数字孪生跑者" | ML增强预测、What-If推演、风险预警 | v0.20-v0.22 | ✅ 完成 |
| **进化跑步** | "越用越懂你的私人教练" | 决策追踪、自适应学习、个性化进化 | v0.23-v0.25 | ✅ 完成 |
| **交互升级** | "随时随地与你对话的AI教练" | 底座升级、新特性适配、WebUI | v0.26-v0.29 | 🏗️ 进行中（v0.26-v0.27 完成，v0.28-v0.29 规划中） |

### 1.2 战略维度

| 战略维度 | 用户价值 |
|----------|----------|
| **隐私可控** | 跑者的数据只属于自己，不上传、不泄露 |
| **专业可信** | 基于运动科学的分析，AI建议可追溯、可验证 |
| **预测未来** | 基于2年+数据，预测伤病风险、VDOT趋势、比赛成绩 |
| **自我进化** | 系统从每次训练和反馈中学习，越来越懂个人特点 |
| **随时可达** | 通过WebUI随时随地与AI教练对话，数据可视化一目了然 |

### 1.3 目标用户分层

| 用户层级 | 数据量 | 可用能力 | 产品体验 |
|----------|--------|----------|----------|
| **初级** | <6个月 | 基础统计、简单趋势 | 感受"数据被记录" |
| **中级** | 6-18个月 | 线性回归预测、Jack Daniels公式 | 感受"数据在说话" |
| **高级** | 18个月+ | ML增强预测 | 感受"数据懂自己" |
| **资深** | 24个月+ | 数字孪生推演 + 自适应进化 + WebUI | 感受"系统在进化，随时可对话" |

## 3. Phase D：交互升级（v0.26 - v0.29）

### 3.1 Phase 概述

**背景**：nanobot-ai 0.2.0 发布，引入 GoalState、FallbackProvider、WebUI 内置打包、Model Presets、推理可见化等重要新能力。RunFlowAgent 需要升级底座并规划新特性适配，同时建设 WebUI 交互能力。

**Phase 目标**：
1. 安全升级底座到 nanobot-ai 0.2.0，零功能回归
2. 适配三项关键新特性，增强现有功能
3. 建设WebUI交互能力，从纯CLI进化到Web交互
4. 评估剩余新特性，决定是否纳入后续版本

**设计文档**：[底座升级设计文档](../superpowers/specs/2026-05-23-nanobot-0.2.0-upgrade-design.md)

### 3.2 v0.26.0 — 底座升级 + 新特性适配 ✅ 已完成

**主题**：nanobot-ai 0.2.0 底座升级 + GoalState/推理可见化/Model Presets 适配

**交付成果**：
- 依赖升级至 `nanobot-ai>=0.2.0`，5075 用例零失败，覆盖率 82.63%
- GoalState 适配：Agent 跨轮次记忆训练计划目标
- 推理可见化：飞书对话中展示 Chain-of-Thought
- Model Presets：支持 `/model <preset>` 快速切换模型配置

**验收结果**：✅ 全部通过（底座升级 5 项 + 新特性 3 项）

> 详细设计见 Git 历史版本与 [底座升级设计文档](../superpowers/specs/2026-05-23-nanobot-0.2.0-upgrade-design.md)

### 3.3 v0.27.0 — WebUI 基础 ✅ 已完成

**主题**：复用 nanobot-ai 原生 WebUI，实现 AI 对话交互 + 基础设置

**交付成果**：
- WebSocket 通道配置：`config.json` 新增 `websocket` 配置节
- Gateway CLI 增强：`gateway start --webui` 标志启用 WebUI
- 品牌自定义：bot_name="Nanobot-Runner" / bot_icon="🏃‍♂️"
- 安全认证：默认 127.0.0.1 + token 认证 + 短期令牌签发
- 性能指标：4489 用例零失败，覆盖率 81%，WebUI 首屏 <3s，流式延迟 <200ms

**验收结果**：✅ 全部通过（3 项验收标准）

> 详细设计见 Git 历史版本与 [架构设计说明书 v17.0.0](../architecture/架构设计说明书.md) §10

### 3.4 v0.28.0 — WebUI 数据可视化

**主题**：扩展 WebUI，增加跑步数据可视化能力（参考COROS Dashboard + Activities设计）

**设计参考**：基于COROS WebUI调研，采用左侧导航+右侧内容区布局，数据卡片+趋势图表组合展示

| 功能模块 | 功能描述 | 设计参考 |
|----------|----------|----------|
| **首页仪表盘** | 今日概览卡片（距离/时长/配速/心率）+ 本周统计 + 快捷入口 | COROS Dashboard "今日数据"卡片 |
| **VDOT趋势页** | VDOT历史趋势图 + 预测区间 + 关键节点标注 | COROS趋势图表设计 |
| **训练负荷页** | ATL/CTL/TSB堆叠展示 + 疲劳状态指示 + 趋势预警 | COROS多维度数据展示 |
| **活动列表页** | 跑步记录列表（距离/时长/配速/心率）+ 筛选器 + 无限滚动 | COROS Activities列表设计 |
| **活动详情页** | 单次跑步详细数据 + 配速/心率曲线 + 数据标签 | COROS活动详情页 |
| **身体信号页** | HRV/疲劳度/恢复状态卡片展示 | COROS身体指标卡片 |

**技术方案**：
- 后端：新增WebUI专用API端点（`/api/webui/dashboard`、`/api/webui/activities`、`/api/webui/vdot`等）
- 前端：扩展nanobot-ai原生WebUI，新增"数据"标签页
- 图表库：Recharts（React生态兼容性好）
- 统一时间范围控件：7/30/90/365天切换，所有图表共用

**验收标准**：
1. WebUI首页展示今日概览卡片和本周统计数据
2. 可查看VDOT趋势图、训练负荷图（ATL/CTL/TSB堆叠）
3. 活动列表支持筛选（时间范围/距离/类型）和无限滚动
4. 图表数据与CLI命令输出一致
5. 图表支持时间范围筛选（7/30/90/365天）

### 3.5 v0.29.0 — WebUI 管理控制台

**主题**：训练计划管理 + 进化引擎控制台（参考COROS Schedule + Personal设计）

**设计参考**：基于COROS WebUI调研，采用日历/列表双视图展示计划，设置中心统一管理配置

| 功能模块 | 功能描述 | 设计参考 |
|----------|----------|----------|
| **训练计划日历** | 日历视图展示每日训练安排 + 训练卡片（类型/距离/配速）| COROS Schedule日历视图 |
| **训练计划列表** | 列表视图展示计划详情 + 完成状态标记 | COROS Schedule列表视图 |
| **计划执行进度** | 完成率环形图 + 忠实度指标 + 阶段总结 | 可视化进度展示 |
| **计划调整-AI模式** | 通过AI对话调整计划（自然语言交互）| 复用WebUI对话能力 |
| **计划调整-手工模式** | 直接编辑训练详情（日期/距离/配速/类型）| COROS训练编辑表单 |
| **进化状态面板** | 进化引擎状态卡片 + 触发条件展示 + 最近进化记录 | 状态仪表盘设计 |
| **提示参数调优** | PromptTuner 4维参数滑块调整（tone/detail/aggressive/data-driven）| 滑块交互组件 |
| **月度进化报告** | 历史进化报告列表 + 报告详情页（图表展示）| 与v0.28.0图表能力复用 |
| **设置中心** | 个人资料 + 偏好设置 + 连接状态 + 系统配置 | COROS Personal页面设计 |
| **快捷操作栏** | 常用操作一键直达（导入数据/生成报告/调整计划）| 快捷入口设计 |

**技术方案**：
- 后端：新增计划管理API（`/api/webui/plan`、`/api/webui/plan/adjust`等）、进化引擎API（`/api/webui/evolution`等）
- 前端：扩展WebUI新增"计划"、"进化"、"设置"标签页
- 计划调整双模式：AI模式调用Agent对话接口，手工模式调用计划更新API
- 组件库：复用v0.28.0图表组件，保持视觉一致性

**验收标准**：
1. WebUI中可查看当前训练计划（日历/列表双视图）及执行进度
2. 支持双模式调整训练计划：AI对话模式 + 手工编辑模式
3. 进化引擎状态可视化展示（状态卡片+触发条件）
4. PromptTuner 4维参数可通过滑块可视化调整
5. 设置中心可管理个人资料和系统配置

### 3.6 独立评估任务（不占版本号）

**主题**：FallbackProvider + MCP Resources/Prompts 评估

> 说明：此两项工作为评估性质，根据评估结论决定是否在后续版本中实施，不单独占用版本号。

| 评估项 | 评估要点 |

**验收标准**：
1. 完成 FallbackProvider 评估报告，明确是否采纳及实施方案
2. 完成 MCP Resources/Prompts 评估报告，明确是否采纳及实施方案

## 5. 风险与缓解

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|----------|
| 决策日志数据膨胀 | 中 | 长期运行后日志过大 | Parquet按月分片；自动归档旧数据 |
| 用户反馈稀疏 | 高 | 缺乏足够反馈驱动进化 | 轻量反馈机制；主动询问；隐式反馈自动记录 |
| 进化方向偏差 | 中 | 系统学习到的偏好与真实目标偏离 | 保留人工覆盖机制；月度进化报告review；reset_to_default()回退 |
| nanobot-ai 0.2.0 API breaking change | 低 | 升级后现有功能异常 | 0.2.0 对 AgentHook 接口纯增量扩展，无破坏性变更；v0.26.0 全量测试覆盖 |
| 间接依赖冲突 | 中 | nanobot-ai 0.2.0 间接依赖与现有依赖冲突 | `uv pip install --dry-run` 预检 |
| 原生 WebUI 不支持跑步数据可视化 | 中 | v0.27.0 无图表能力 | v0.27.0 先验证基础链路，v0.28.0 再扩展 |
| 上游 WebUI 版本变更影响 | 低 | nanobot-ai WebUI 更新影响 RunFlowAgent | 锁定 nanobot-ai 版本；自定义组件独立维护 |
| FallbackProvider 推迟导致 LLM 调用稳定性风险 | 低 | 单 Provider 故障时无容错 | 当前单 Provider 场景下风险可控；独立评估后决定是否纳入后续版本 |
| MCP Resources/Prompts 推迟导致工具扩展性受限 | 低 | MCP 生态利用不充分 | 当前 MCP Tool 映射已满足需求；独立评估后决定是否纳入后续版本 |

### 6.2 术语表
