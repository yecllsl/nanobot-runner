# v0.29.0 WebUI管理控制台 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 v0.28.0 数据可视化基础上，新增训练计划管理、进化引擎控制台、设置中心三大功能模块，提供 12 个新 API 端点和 4 个新前端页面。

**Architecture:** FastAPI 后端薄封装核心模块（PlanManager/EvolutionController/PromptTuner/EvolutionReporter/ConfigManager），同步方法通过 `run_in_threadpool` 包装。React SPA 前端复用 v0.28.0 的 `useApi` hook、`apiClient` 和组件模式，自建 CSS Grid 日历组件（ADR-024），手动刷新进化状态（ADR-025）。

**Tech Stack:** Python 3.11+ / FastAPI / Pydantic / React 18 / TypeScript / TailwindCSS / date-fns / Recharts

## Task 1: 训练计划 API 路由 (plan.py)

**Files:**
- Create: `src/core/webui/routes/plan.py`
- Test: `tests/unit/webui/test_plan_routes.py`

- [ ] **Step 1: 编写 plan.py 路由模块**

创建 `src/core/webui/routes/plan.py`，包含 5 个 API 端点：

| 端点 | 方法 | 同步函数 | 核心模块调用 |
|------|------|----------|-------------|
| `/webui/plan/list` | GET | `_get_plan_list` | `PlanManager.list_plans()` |
| `/webui/plan/calendar` | GET | `_get_plan_calendar` | `PlanManager.get_active_plan()` |
| `/webui/plan/{plan_id}` | GET | `_get_plan_detail` | `PlanManager.get_plan()` |
| `/webui/plan/progress/{plan_id}` | GET | `_get_plan_progress` | `PlanManager.get_plan()` + `PlanExecutionRepository.get_plan_execution_stats()` |
| `/webui/plan/daily/{plan_id}/{date}` | PUT | `_update_daily_plan` | `PlanManager.record_execution()` |

关键实现要点：
- `DailyPlanUpdateRequest` Pydantic 模型：`completion_rate(0.0-1.0)`, `effort_score(1-10)`, `notes(max 500)`, `actual_distance_km(>0)`, `actual_duration_min(>0)`, `actual_avg_hr(30-250)`
- `_get_plan_calendar` 中包含训练类型中文标签映射（easy_run→轻松跑等11种）
- `_get_plan_progress` 优先使用 `PlanExecutionRepository`，异常时降级手动计算
- `_update_daily_plan` 校验日期格式 `^\d{4}-\d{2}-\d{2}$`
- 所有端点通过 `Depends(get_current_user)` 认证
- 同步方法通过 `run_in_threadpool` 包装
- 404 错误返回 "计划不存在"，400 错误返回具体校验失败信息
