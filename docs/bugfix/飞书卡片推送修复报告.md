# 飞书卡片推送 Bug 修复报告

## 📋 问题描述

执行 `uv run nanobotrun report --push` 命令时，推送飞书卡片消息失败，出现以下错误：

```
飞书 API 调用失败：Invalid parameter type in json: content. Invalid parameter value:
{"config":{"wide_screen_mode":true},"elements":[...]}
```

## 🔍 问题分析

### 错误堆栈

```
File "src/core/report_service.py", line 358, in push_report
    feishu = self._get_feishu_bot()
File "src/notify/feishu.py", line 699, in send_card
    return self.message_api.send_card(...)
File "src/notify/feishu.py", line 241, in send_card
    "content": card_content,
```

### 根本原因

飞书 API 的 `send_card` 接口要求 `content` 字段是**JSON 字符串**，而不是 Python 字典对象。

**错误的实现**：
```python
payload = {
    "receive_id": receive_id,
    "msg_type": "interactive",
    "content": card_content,  # ❌ 错误：直接传递字典对象
}
```

**飞书 API 要求**：
```python
payload = {
    "receive_id": receive_id,
    "msg_type": "interactive",
    "content": json.dumps(card_content),  # ✅ 正确：JSON 字符串
}
```

### 卡片结构问题

飞书卡片消息的 `header.title` 字段必须是一个**对象**，而不是字符串。

**错误的结构**：
```json
{
  "header": {
    "title": "☀️ 每日跑步晨报"  // ❌ 错误：字符串
  }
}
```

**正确的结构**：
```json
{
  "header": {
    "title": {
      "tag": "plain_text",
      "content": "☀️ 每日跑步晨报"  // ✅ 正确：对象
    }
  }
}
```

## ✅ 修复方案

### 修复 1：将卡片内容序列化为 JSON 字符串

修改 `src/notify/feishu.py` 的 `send_card` 方法：

```python
def send_card(
    self,
    card_content: Dict[str, Any],
    receive_id: str,
    receive_id_type: str = "user_id",
) -> Dict[str, Any]:
    """发送卡片消息"""
    import json

    endpoint = "/messages"
    params = {"receive_id_type": receive_id_type}
    payload = {
        "receive_id": receive_id,
        "msg_type": "interactive",
        "content": json.dumps(card_content, ensure_ascii=False),  # ✅ 修复：序列化为 JSON 字符串
    }

    return self._request("POST", endpoint, params=params, json=payload)
```

### 修复 2：修正卡片结构

修改 `src/notify/feishu.py` 的 `send_card` 调用处的卡片结构：

```python
card_payload = {
    "config": {"wide_screen_mode": True},
    "header": {
        "title": {
            "tag": "plain_text",
            "content": title,  # ✅ 修复：使用对象而非字符串
        },
        "template": "blue",
    },
    "elements": [
        {
            "tag": "div",
            "text": {
                "tag": "lark_md",
                "content": content,  # ✅ 修复：使用对象而非字符串
            },
        },
    ],
}
```

## 📝 飞书卡片消息格式详解

### 完整的卡片消息结构

```json
{
  "config": {
    "wide_screen_mode": true
  },
  "header": {
    "title": {
      "tag": "plain_text",
      "content": "卡片标题"
    },
    "template": "blue"
  },
  "elements": [
    {
      "tag": "div",
      "text": {
        "tag": "lark_md",
        "content": "**粗体文本**\n普通文本"
      }
    }
  ]
}
```

### 关键字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `config.wide_screen_mode` | boolean | 是否宽屏模式 |
| `header.title` | object | 标题对象（不能是字符串） |
| `header.title.tag` | string | 文本类型，固定为 `plain_text` |
| `header.title.content` | string | 标题内容 |
| `header.template` | string | 卡片颜色主题 |
| `elements` | array | 卡片元素列表 |
| `elements[].tag` | string | 元素类型（`div`, `markdown` 等） |
| `elements[].text` | object | 文本对象 |
| `elements[].text.tag` | string | 文本渲染类型（`lark_md`, `plain_text`） |
| `elements[].text.content` | string | 文本内容 |

## 💡 经验总结

### 问题根源

1. **API 文档理解偏差**：飞书 API 文档中 `content` 字段的示例是 JSON 对象，但实际要求是 JSON 字符串
2. **类型检查缺失**：代码中没有对卡片内容进行序列化验证
3. **错误提示不明确**：飞书 API 返回的错误信息不够直观

### 改进建议

1. **添加 JSON 序列化验证**：
   ```python
   try:
       json.dumps(card_content)
   except TypeError as e:
       logger.error(f"卡片内容序列化失败：{e}")
       return {"success": False, "error": "卡片内容格式错误"}
   ```

2. **添加卡片结构校验**：
   ```python
   def validate_card_structure(card: dict) -> bool:
       """验证卡片结构是否符合飞书 API 要求"""
       required_keys = ["config", "header", "elements"]
       return all(key in card for key in required_keys)
   ```

3. **改进错误日志**：
   ```python
   except json.JSONEncodeError as e:
       logger.error(f"卡片内容 JSON 序列化失败：{e}")
       logger.debug(f"卡片内容：{card_content}")
   ```

4. **添加单元测试**：
   - 测试卡片内容序列化
   - 测试卡片结构验证
   - 测试各种边界情况
