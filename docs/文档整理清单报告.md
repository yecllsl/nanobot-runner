## 📋 归档规则说明

### 归档标准

1. **版本过期**: 非当前版本的规划、需求、测试文档
2. **任务完成**: 已完成的临时任务文档
3. **内容重复**: 被新版本替代的旧版本文档
4. **历史参考**: 仅作历史参考的评审报告

### 保留标准

1. **当前版本**: v0.4.x版本的核心文档
2. **持续有效**: 规范、指南、模板类文档
3. **参考价值**: 架构设计、数据模型等基础文档
4. **配置文件**: .trae目录下的所有配置文件

### 归档目录结构

```
docs/archive/
├── v0.1.0/          # v0.1.0版本归档
├── v0.2.0/          # v0.2.0版本归档
├── v0.3.0/          # v0.3.0版本归档
├── v0.4.0/          # v0.4.0版本归档
└── v0.4.1/          # v0.4.1版本归档
```

## ✅ 整理效果

### 优化成果

1. **目录结构清晰**: 按版本归档历史文档，保留当前开发所需文档
2. **查找效率提升**: 减少主目录文档数量，便于快速定位
3. **版本追溯完整**: 归档文档按版本组织，支持历史追溯
4. **项目功能无损**: 归档操作不影响当前功能及开发进度

### 维护建议

1. **定期归档**: 每个版本发布后，将旧版本文档归档
2. **命名规范**: 新文档遵循现有命名规范，便于管理
3. **版本标记**: 重要文档应标注适用版本
4. **清理周期**: 建议每季度进行一次文档整理
