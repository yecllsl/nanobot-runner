### 自动迁移机制

系统在首次启动时会自动检测并迁移现有配置：

```python
def _migrate_old_cron_config(self):
    """迁移旧的定时任务配置到新位置"""
    old_cron_store = Path.home() / ".nanobot" / "cron" / "jobs.json"
    new_cron_store = self.cron_store
    
    if old_cron_store.exists() and not new_cron_store.exists():
        try:
            shutil.copy2(old_cron_store, new_cron_store)
            logger.info(f"已迁移定时任务配置：{old_cron_store} -> {new_cron_store}")
        except Exception as e:
            logger.warning(f"迁移定时任务配置失败：{e}")
```

## 故障排除

### 常见问题

#### 问题1：定时任务配置迁移失败

**症状**: 定时任务无法正常执行

**解决方案**:
```bash
# 手动迁移配置
cp ~/.nanobot/cron/jobs.json ~/.nanobot-runner/cron/jobs.json

# 检查权限
chmod 644 ~/.nanobot-runner/cron/jobs.json
```

#### 问题2：版本升级后定时任务丢失

**症状**: 原有的定时任务配置不见了

**解决方案**:
```bash
# 检查备份文件
ls -la ~/.nanobot/cron/

# 如果有备份，手动恢复
cp ~/.nanobot/cron/jobs.json.backup ~/.nanobot-runner/cron/jobs.json
```

### 回滚方案

如果需要回滚到0.4.0版本：

```bash
# 卸载当前版本
pip uninstall nanobot-runner

# 安装0.4.0版本
pip install nanobot-runner==0.4.0

# 恢复配置路径（如果需要）
mv ~/.nanobot-runner/cron/jobs.json ~/.nanobot/cron/jobs.json
```
