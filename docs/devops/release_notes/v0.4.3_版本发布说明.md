## ✨ 主要变更

### 1. 配置 Schema 验证机制

**问题**: 配置加载无验证，直接硬编码默认值，导致配置错误时运行时异常

**解决方案**:
- 新增 `src/core/config_schema.py` 模块（172 行）
- 实现 `AppConfig` 数据类，定义配置结构
- 自动验证功能：
  - ✅ 必填字段验证（`version`, `data_dir`）
  - ✅ 类型检查（所有配置项类型自动验证）
  - ✅ 版本号格式验证（正则表达式 `x.y.z`）
  - ✅ 枚举值验证（`feishu_receive_id_type`）
- 配置加载/保存时自动验证
- 验证失败提供详细错误提示

**测试**: 22 个单元测试，覆盖率 100%

**代码示例**:
```python
from src.core.config_schema import AppConfig

# 自动验证配置
config_dict = load_config()
app_config = AppConfig.from_dict(config_dict)  # 验证失败会抛出 ValueError

# 手动验证
is_valid, errors = AppConfig.validate(config_dict)
if not is_valid:
    print(f"配置验证失败：\n" + "\n".join(f"  - {e}" for e in errors))
```
