# v0.4.0 Release Notes

**发布日期**: 2026-03-20
**版本号**: v0.4.0
**发布类型**: 功能版本

---

## 概述

v0.4.0 版本聚焦于**智能化跑步助理**核心能力建设，成功实现了用户画像、训练计划、比赛预测和报告服务四大核心功能模块，为 nanobot-runner 从数据管理工具向 AI 智能助理转型奠定了坚实基础。

---

## 新增功能

### FR-001: 用户画像引擎 (覆盖率 87%)

**核心能力**:
- `RunnerProfile` 数据类：完整的用户画像数据结构
- `ProfileEngine`：基于训练数据自动构建画像
- 双存储机制：profile.json + MEMORY.md
- 数据新鲜度检查：自动检测过期数据（7天阈值）
- 异常数据过滤：智能识别并过滤异常训练数据
- 健身水平评估：VDOT 为基础的四级评估体系
- 训练模式分析：识别用户训练习惯
- 受伤风险评估：多因素综合评估

**技术亮点**:
- 使用 `@dataclass` 确保类型安全
- 配置化的异常过滤规则 `ANOMALY_FILTER_RULES`
- 双存储机制实现程序可读性与 Agent 可读性的平衡

---

### FR-002: 训练计划引擎 (覆盖率 97%)

**核心能力**:
- `TrainingPlan` 数据类：完整的训练计划结构
- `WeeklySchedule`：周训练计划安排
- `DailyPlan`：每日训练详情
- 分阶段训练：base → progression → peak → race → recovery
- 目标配速计算：基于 VDOT 的配速体系
- 动态调整：根据心率漂移和 RPE 调整计划
- 体能水平适配：根据体能水平动态调整配置

**技术亮点**:
- 配置化的训练阶段 `PHASE_CONFIG`
- 基于 VDOT 的科学配速计算
- 支持心率漂移和 RPE 的动态调整算法

---

### FR-003: 飞书日历同步

**核心能力**:
- `FeishuCalendarSync`：训练计划同步到飞书日历
- `FeishuCalendarAPI`：飞书日历 API 封装
- 冲突检测：智能识别日程冲突
- 4 种冲突解决策略：KEEP_REMOTE / KEEP_LOCAL / MERGE / MANUAL
- 每日训练同步：自动创建日历事件
- Webhook 安全验证：签名验证机制

**技术亮点**:
- 异步 API 调用设计
- 自动 Token 管理和刷新
- 配置化的重试机制

---

### FR-004: 飞书机器人交互

**核心能力**:
- 快捷指令支持：/stats, /plan, /vdot 等
- 卡片消息格式：支持交互式卡片
- 消息响应时间：< 3秒
- 多种消息类型：文本、卡片、富文本

**技术亮点**:
- 中英文命令解析
- 完善的错误处理机制
- 支持交互式卡片消息

---

### FR-005: 比赛成绩预测 (覆盖率 90%)

**核心能力**:
- `RacePredictionEngine`：基于 Jack Daniels VDOT 公式
- VDOT ↔ 时间转换：标准距离预测
- 训练效应预测：考虑训练周期影响
- 配速计算：目标比赛配速
- 置信度评分：预测可信度评估

**技术亮点**:
- 采用业界公认的 Jack Daniels VDOT 公式
- 二分查找实现时间到 VDOT 的反推
- 基于趋势稳定性的置信度计算

---

### FR-006: 智能训练回顾 (覆盖率 80%)

**核心能力**:
- 日报生成：每日训练总结
- 周报生成：周训练统计、亮点识别、问题发现
- 月报生成：月度训练总结
- 飞书推送：支持文本和卡片消息
- 定时调度：基于 nanobot cron 服务
- 亮点/问题识别：智能分析训练数据
- 建议生成：基于训练负荷的个性化建议

**技术亮点**:
- 模板引擎设计，支持自定义模板
- 依赖注入设计，便于测试
- 与飞书推送深度集成

## 已知问题

### 测试相关问题 (不影响业务功能)

1. **asyncio event loop 问题** (14 个用例)
   - 原因：Python 3.10+ 中 `asyncio.get_event_loop()` 行为变更
   - 影响：仅影响单元测试执行
   - 计划：v0.4.1 修复

2. **Schema datetime 格式问题** (1 个用例)
   - 原因：测试用例使用了不完整的日期格式
   - 影响：仅影响该测试用例
   - 计划：v0.4.1 修复

### 功能限制

1. **报告服务**: 部分高级功能待完善（如自定义报告模板）
2. **训练计划**: 需要真实训练数据验证调整算法
3. **比赛预测**: 未考虑天气、赛道等外部因素

## 升级指南

### 安装

```bash
# 使用 pip 安装
pip install nanobot-runner==0.4.0

# 或使用 uv
uv pip install nanobot-runner==0.4.0
```

### 从 v0.3.x 升级

v0.4.0 新增了以下配置项，请在 `~/.nanobot-runner/config.json` 中添加：

```json
{
  "feishu": {
    "calendar_enabled": false,
    "calendar_id": "",
    "webhook_enabled": false,
    "webhook_port": 8080
  }
}
```

### 数据迁移

v0.4.0 兼容 v0.3.x 的数据格式，无需手动迁移。首次使用新功能时，系统会自动创建必要的目录和文件：

- `~/.nanobot-runner/data/profile.json` - 用户画像数据
- `~/.nanobot-runner/data/plans/` - 训练计划存储
