## 2. 变更内容

### 2.1 新增功能

#### 配置管理基础设施
- **ConfigManager**: 支持环境变量覆盖、无配置模式、配置验证
- **EnvManager**: 管理 .env.local 文件和环境变量
- **WorkspaceManager**: 支持工作区路径解析和创建
- **MigrationEngine**: 支持版本间数据迁移
- **BackupManager**: 支持配置和数据备份/恢复
- **VerifyManager**: 支持文件完整性校验
- **InitWizard**: 提供交互式初始化流程

#### 初始化命令
- **nanobotrun init**: 新增初始化命令，支持全新初始化、迁移模式、修复模式
- **交互式配置**: 支持 LLM Provider 选择、业务配置、飞书集成配置
- **配置验证**: 初始化后自动验证配置有效性

#### 系统管理命令
- **nanobotrun system validate**: 新增配置验证命令
- **nanobotrun system backup**: 新增备份命令
- **nanobotrun system restore**: 新增恢复命令
- **nanobotrun system migrate**: 新增迁移命令

### 2.2 架构改进

#### 配置管理架构
- **配置分层**: 支持默认值 < 配置文件 < 环境变量的优先级覆盖
- **配置验证**: 支持格式验证、完整性验证、有效性验证、一致性验证
- **配置迁移**: 支持从旧版本配置自动迁移到新版本

#### 依赖注入扩展
- **AppContext v3**: 扩展上下文管理器，支持配置管理相关组件
- **SessionRepository**: 新增会话仓储层，提供类型安全的 Session 查询

### 2.3 测试覆盖提升

- **BUG-001**: MigrationEngine 测试覆盖率从 41% 提升至 99%

#### 问题1: 版本号不一致导致发布失败

**问题描述**:

- 原因：README.md中的版本号为v0.9.3，与pyproject.toml和CHANGELOG.md中的v0.9.4不一致

**解决方案**:
1. 更新README.md中的版本号从v0.9.3改为v0.9.4
2. 提交并推送main分支
3. 删除远程和本地的v0.9.4 tag
4. 重新创建并推送v0.9.4 tag
