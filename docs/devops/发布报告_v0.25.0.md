## 2. 变更内容

### 新增功能

#### EvolutionController（进化控制器）
- 4 种触发规则：数据量触发（>=100 决策）、时间触发（每周）、拒绝触发（>=3 次连续拒绝）、性能退化触发（MAE 上升 >=15%）
- persist-first 执行策略（先落盘再响应）
- 触发历史记录（trigger_history）与幂等性保障（同一天同类型不重复触发）

#### PromptTuner（提示词调谐器）
- 4 维参数空间：tone（正式-随意）、detail（简洁-详细）、aggressive（保守-激进）、data_driven（直觉-数据驱动）
- 参数地板保护（aggressive>=0.1, data_driven>=0.2）与反弹机制（过调后回弹 50%）
- JSON 持久化（tuning/ 目录）
- IncrementalLearnResult 结构化部分失败跟踪

#### EvolutionReporter（进化报告生成器）
- 月度进化报告生成
- 覆盖触发统计、调谐参数变化、性能趋势

#### CLI 命令新增
- `nanobotrun evolution triggers` — 查看触发状态
- `nanobotrun evolution report` — 生成报告
- `nanobotrun evolution tune` — 查看当前调谐参数

#### Agent 工具新增
- `check_evolution_triggers` — 检查触发条件
- `get_evolution_report` — 获取进化报告
- `adjust_prompt_params` — 调整提示词参数

### 架构改进
- DecisionLogHook 持有 EvolutionEngine 引用（编排层一致性）
- AppContext 扩展支持新模块注入
- ContextExtension 适配进化模块

### 修复
| 编号 | 问题 | 修复方式 |
|------|------|---------|
| H-01 | DecisionLogHook 缺少编排层引用 | after_iteration 回调注入 EvolutionEngine |
| H-02 | 部分失败无结构化跟踪 | IncrementalLearnResult 数据模型 |
| H-03 | 参数无下限保护 | 地板保护（aggressive>=0.1, data_driven>=0.2）+ 反弹机制 |
| - | PromptTuningParams.with_updates() data_driven 逻辑错误 | 修复四则运算逻辑 |
| - | EvolutionController rejection 触发检查字段错误 | outcome vs decision 字段修正 |
