## 一、测试说明

### 1.1 测试目的

验证飞书 Gateway 模式是否满足用户需求，确保：

- 飞书机器人能正常启动和运行
- 命令交互功能完整、准确
- 自然语言交互流畅、智能
- 心跳检测和定时任务正常工作
- 用户体验良好

### 1.2 测试环境要求

| 项目     | 要求                           |
| ------ | ---------------------------- |
| 操作系统   | Windows 10/11 或 macOS 10.15+ |
| Python | 3.11 或更高版本                   |
| 网络环境   | 需要稳定的网络连接                    |
| 飞书账号   | 需要飞书开发者账号和应用                 |
| 测试数据   | 至少准备 7-14 天的跑步记录             |

### 1.3 测试前准备

#### 1.3.1 飞书应用配置

1. **创建飞书应用**
   - 访问飞书开放平台：<https://open.feishu.cn>
   - 创建企业自建应用
   - 获取 App ID 和 App Secret
2. **配置应用权限**
   - 消息：`im:message`（获取与发送单聊、群组消息）
   - 用户：`contact:user.base:readonly`（获取用户基本信息）
3. **配置事件订阅**
   - 订阅消息事件：`im.message.receive_v1`
   - 配置请求网址：`http://<服务器IP>:18790/webhook/feishu`
4. **发布应用**
   - 提交审核并发布应用
   - 将应用添加到企业可用应用列表

#### 1.3.2 本地配置

```bash
# 1. 进入项目目录
cd nanobot-runner

# 2. 激活虚拟环境
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # macOS/Linux

# 3. 配置飞书应用信息
uv run nanobotrun system config

# 编辑配置文件 ~/.nanobot-runner/config.json
{
  "feishu_app_id": "<你的App ID>",
  "feishu_app_secret": "<你的App Secret>",
  "feishu_receive_id": "<接收消息的用户ID>",
  "feishu_receive_id_type": "user_id"
}
```

#### 1.3.3 数据准备

```bash
# 1. 导入测试数据（至少 7-14 天的跑步记录）
uv run nanobotrun data import <FIT文件路径>

# 2. 验证数据已导入
uv run nanobotrun data stats
```

#### 1.3.4 网络配置

```bash
# 确保服务器端口可访问
# 默认端口：18790

# Windows 防火墙配置
netsh advfirewall firewall add rule name="Nanobot Gateway" dir=in action=allow protocol=tcp localport=18790

# macOS 防火墙配置（系统偏好设置 -> 安全性与隐私 -> 防火墙）
```

### 4.3 常见问题排查

#### 问题 1：服务启动失败

**可能原因**:

- 飞书应用配置错误
- 端口被占用
- 网络连接问题

**排查步骤**:

```bash
# 1. 检查配置
uv run nanobotrun system config

# 2. 检查端口占用
netstat -ano | findstr :18790

# 3. 测试网络连接
ping open.feishu.cn
```

#### 问题 2：机器人无响应

**可能原因**:

- 事件订阅配置错误
- 网络防火墙阻止
- 服务未正常运行

**排查步骤**:

```bash
# 1. 检查服务状态
# 查看终端是否有错误信息

# 2. 检查事件订阅
# 飞书开放平台 -> 应用详情 -> 事件订阅

# 3. 检查防火墙
# 确保 18790 端口可访问
```

#### 问题 3：响应数据不准确

**可能原因**:

- 数据文件损坏
- 数据导入不完整
- 计算逻辑错误

**排查步骤**:

```bash
# 1. 验证数据
uv run nanobotrun data stats

# 2. 重新导入数据
uv run nanobotrun data import <路径> --force

# 3. 检查日志
uv run nanobotrun gateway start --logs
```
