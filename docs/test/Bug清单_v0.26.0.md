## Bug详情

### BUG-001: 偶发性测试隔离问题（flaky test）

| 字段 | 内容 |
|------|------|
| **Bug ID** | BUG-001 |
| **所属模块** | tests/integration/scene/test_v0140_memory_personality.py |
| **严重等级** | 优化 |
| **Bug标题** | test_memory_version_list_and_restore 偶发失败（测试隔离性问题） |
| **出现版本** | v0.26.0（基线中已存在，非本次引入） |
| **优先级** | 低 |
| **测试人员** | 测试工程师 |
| **创建时间** | 2026-05-24 |
| **状态** | 待修复 |

#### 复现步骤
1. 执行全量测试: `uv run pytest tests/ --tb=short -q`
2. 偶发出现失败（非100%复现）
3. 失败用例: `tests/integration/scene/test_v0140_memory_personality.py::TestMemoryFlow::test_memory_version_list_and_restore`

#### 实际结果
```
FAILED tests/integration/scene/test_v0140_memory_personality.py::TestMemoryFlow::test_memory_version_list_and_restore
AssertionError: assert 1 == 2
```

#### 预期结果
全量测试执行时该用例应稳定通过，不受其他测试执行顺序影响。

#### 根因分析
该用例为**flaky test（不稳定测试）**，根因为测试间状态污染：
1. 测试用例依赖全局状态（如内存管理器版本列表、文件系统临时目录等）
2. 其他测试用例的执行顺序或副作用可能影响该用例的前置条件
3. 当测试以不同顺序执行时，版本列表数量可能不符合预期（assert 1 == 2 暗示预期有2个版本但实际只有1个）
4. 单独运行时通过，说明用例逻辑本身正确，问题在于测试隔离性不足

#### 修复建议
1. **短期**: 该bug不影响功能发布，属于测试代码质量问题
2. **中期**: 建议开发工程师优化该测试用例的fixture隔离性
   - 检查 `setUp` / `tearDown` 是否正确清理全局状态
   - 使用 `tmp_path` fixture 替代全局临时目录
   - 确保每个测试用例独立初始化内存管理器状态
   - 考虑使用 `pytest-randomly` 检测测试顺序依赖问题
3. **长期**: 在CI中引入flaky test检测机制，对偶发失败用例进行标记和隔离
