## 一、概述

### 1.1 目的

本文档详细说明软件发布流程的测试用例，确保发布过程的稳定性和可靠性，避免v0.4.1版本中遇到的CI Pipeline和Release Pipeline问题再次发生。

### 1.2 背景

v0.4.1版本发布过程中遇到以下问题：
1. CI Pipeline持续失败（mypy类型检查、bandit安全扫描）
2. Release Pipeline权限错误（403，缺少contents: write权限）

### 1.3 测试范围

- CI Pipeline测试
- Release Pipeline测试
- 发布前检查
- 发布后验证

def test_version_consistency():
    """测试版本一致性"""
    import subprocess
    import re
    from pathlib import Path
    
    # 从pyproject.toml读取版本
    pyproject = Path('pyproject.toml').read_text()
    toml_version = re.search(r'version = "([^"]+)"', pyproject).group(1)
    
    # 从Git标签读取版本
    result = subprocess.run(
        ['git', 'describe', '--tags', '--abbrev=0'],
        capture_output=True,
        text=True
    )
    
    if result.returncode == 0:
        git_version = result.stdout.strip().lstrip('v')
        assert toml_version == git_version, \
            f"版本不一致: pyproject.toml={toml_version}, git={git_version}"

## 七、故障排查指南

### 7.1 CI Pipeline失败

#### 问题: mypy检查失败

**症状**:
```
error: Cannot find implementation or library stub for module named "requests"
```

**解决方案**:
```bash
# 1. 安装类型存根
pip install types-requests

# 2. 更新CI配置
# 在ci.yml中添加:
- name: Install type stubs
  run: pip install types-requests
```

#### 问题: 缓存导致依赖问题

**症状**:
```
ERROR: Could not find a version that satisfies the requirement
```

**解决方案**:
```yaml
# 清理缓存
- name: Clean cache
  run: |
    pip cache purge
    pip install -e . --no-cache-dir
```

### 7.2 Release Pipeline失败

#### 问题: 403权限错误

**症状**:
```
Error: Resource not accessible by integration
```

**解决方案**:
```yaml
# 在release.yml中添加权限配置
permissions:
  contents: write
```

#### 问题: Token过期

**症状**:
```
Error: Bad credentials
```

**解决方案**:
- 检查GITHUB_TOKEN是否配置
- 检查Token权限范围
- 重新生成Token

## 八、自动化发布脚本

### 8.1 发布脚本

```python
#!/usr/bin/env python3
# scripts/release.py
"""自动化发布脚本"""

import subprocess
import sys
from pathlib import Path


def run_command(cmd: list[str], description: str) -> bool:
    """运行命令"""
    print(f"\n{'='*60}")
    print(f"执行: {description}")
    print(f"命令: {' '.join(cmd)}")
    print('='*60)
    
    result = subprocess.run(cmd)
    if result.returncode != 0:
        print(f"❌ {description} 失败")
        return False
    print(f"✅ {description} 成功")
    return True


def main():
    """主函数"""
    print("开始发布流程...")
    
    steps = [
        (['python', '-m', 'pytest'], "运行测试"),
        (['python', '-m', 'black', '--check', 'src/', 'tests/'], "代码格式化检查"),
        (['python', '-m', 'isort', '--check-only', 'src/', 'tests/'], "导入排序检查"),
        (['python', '-m', 'mypy', 'src/'], "类型检查"),
        (['python', '-m', 'bandit', '-r', 'src/'], "安全扫描"),
        (['python', '-m', 'build'], "构建包"),
    ]
    
    for cmd, desc in steps:
        if not run_command(cmd, desc):
            print("\n❌ 发布流程中止")
            sys.exit(1)
    
    print("\n✅ 发布前检查全部通过")
    print("请手动执行:")
    print("  1. git tag v0.x.x")
    print("  2. git push origin v0.x.x")


if __name__ == "__main__":
    main()
```

### 8.2 使用说明

```bash
# 运行发布前检查
python scripts/release.py

# 创建并推送标签
git tag v0.4.2
git push origin v0.4.2
```
