## 四、环境初始化步骤

### 4.1 Windows环境初始化

```powershell
# 1. 克隆代码仓库
git clone <repository-url>
cd RunFlowAgent

# 2. 创建虚拟环境
uv venv

# 3. 激活虚拟环境
.venv\Scripts\activate

# 4. 安装依赖
uv sync --all-extras

# 5. 验证安装

uv run pytest tests/unit/ -v
```

### 5.2 测试数据生成

```python
# scripts/generate_test_data.py
"""生成测试数据脚本"""

import json
import random
from datetime import datetime, timedelta
from pathlib import Path


def generate_fit_metadata():
    """生成模拟FIT文件元数据"""
    base_date = datetime(2024, 1, 1)
    
    activities = []
    for i in range(100):
        date = base_date + timedelta(days=i)
        distance = random.uniform(3, 21)  # 3-21km
        duration = distance * random.uniform(300, 360)  # 5-6 min/km
        
        activities.append({
            "activity_id": f"act_{i:04d}",
            "timestamp": date.isoformat(),
            "filename": f"run_{date.strftime('%Y%m%d')}.fit",
            "total_distance": round(distance, 2),
            "total_timer_time": round(duration, 0),
            "avg_heart_rate": random.randint(120, 160),
        })
    
    return activities


def main():
    """主函数"""
    data_dir = Path.home() / ".nanobot-runner" / "data"
    data_dir.mkdir(parents=True, exist_ok=True)
    
    activities = generate_fit_metadata()
    
    # 保存为元数据文件
    metadata_file = data_dir / "test_metadata.json"
    with open(metadata_file, "w", encoding="utf-8") as f:
        json.dump(activities, f, indent=2, ensure_ascii=False)
    
    print(f"测试数据已生成: {metadata_file}")
    print(f"共生成 {len(activities)} 条活动记录")


if __name__ == "__main__":
    main()
```

## 六、环境变量配置

### 6.1 必需环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `NANOBOT_RUNNER_DATA_DIR` | 数据目录 | `~/.nanobot-runner/data` |
| `NANOBOT_RUNNER_CONFIG_DIR` | 配置目录 | `~/.nanobot-runner/config` |

### 6.2 可选环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `FEISHU_APP_ID` | 飞书App ID | `cli_xxx` |
| `FEISHU_APP_SECRET` | 飞书App Secret | `xxx` |
| `NANOBOT_MODEL` | AI模型 | `deepseek-chat` |
| `LOG_LEVEL` | 日志级别 | `INFO` |

### 6.3 环境变量配置示例

#### Windows (PowerShell)

```powershell
# 临时设置
$env:NANOBOT_RUNNER_DATA_DIR = "$env:USERPROFILE\.nanobot-runner\data"
$env:LOG_LEVEL = "DEBUG"

# 永久设置 (用户级别)
[Environment]::SetEnvironmentVariable("NANOBOT_RUNNER_DATA_DIR", "$env:USERPROFILE\.nanobot-runner\data", "User")
```

#### macOS/Linux (Bash)

```bash
# 临时设置
export NANOBOT_RUNNER_DATA_DIR="$HOME/.nanobot-runner/data"
export LOG_LEVEL="DEBUG"

# 永久设置 (添加到 ~/.bashrc 或 ~/.zshrc)
echo 'export NANOBOT_RUNNER_DATA_DIR="$HOME/.nanobot-runner/data"' >> ~/.bashrc
```

## 七、常见问题排查

### 7.1 依赖安装问题

#### 问题: uv sync失败

**症状**:
```
error: Failed to download distribution
```

**解决方案**:
```bash
# 清理缓存
uv cache clean

# 重新安装
uv sync --reinstall

# 使用国内镜像 (中国用户)
uv sync --index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### 7.2 权限问题

#### 问题: 无法写入数据目录

**症状**:
```
PermissionError: [Errno 13] Permission denied
```

**解决方案**:
```bash
# Windows (PowerShell)
$dir = "$env:USERPROFILE\.nanobot-runner"
New-Item -ItemType Directory -Path $dir -Force
icacls $dir /grant "$env:USERNAME:(OI)(CI)F"

# macOS/Linux
mkdir -p ~/.nanobot-runner
chmod 755 ~/.nanobot-runner
```

### 7.3 测试失败问题

#### 问题: 测试用例失败

**症状**:
```
FAILED tests/unit/test_parser.py::test_parse_fit
```

**解决方案**:
```bash
# 检查测试数据是否存在
ls tests/data/fixtures/

# 重新生成测试数据
uv run python tests/scripts/generate_test_data.py

# 运行特定测试查看详细错误
uv run pytest tests/unit/test_parser.py -v --tb=long
```
