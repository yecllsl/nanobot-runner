## 4. 新发现问题

### BUG-003: WebUI UI 层 E2E 测试大面积失败

- **严重等级**: P2（一般）
- **模块**: webui/frontend
- **描述**: 45 个 Playwright UI 测试失败，前端 SPA 页面渲染后无法定位到预期的 DOM 元素
- **复现步骤**:
  1. 启动 WebUI E2E 服务器
  2. 使用 Playwright 访问各页面（/vdot, /training-load, /activities 等）
  3. 断言页面元素（h2 标题、按钮、图表）可见
  4. 实际页面元素不可见或未渲染
- **根因分析**: 前端 React 组件渲染与测试断言的 DOM 选择器不匹配，可能原因：
  1. 前端组件结构变更后测试未同步更新
  2. 异步数据加载后页面未完全渲染
  3. Token 认证导致页面重定向到登录页
- **修复建议**: 开发工程师需排查前端组件渲染与测试选择器的匹配问题
- **影响范围**: 不影响后端 API 功能，仅影响前端 UI 自动化验证
