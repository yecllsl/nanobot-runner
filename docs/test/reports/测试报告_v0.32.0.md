## 3. 测试修复详情

### 3.1 配置注入集成测试修复

**文件**: `tests/integration/module/test_config_injection.py`

**根因**: v0.32.0 配置分离后，`RunnerProviderAdapter` 从 `load_nanobot_config()` 读取配置，但测试 mock 未设置该方法的返回值。

**修复**: 更新 `_make_mock_config` 方法，添加 `load_nanobot_config.return_value` 返回 nanobot 原生格式配置字典。

### 3.2 WebUI 路由认证修复

**文件**: `src/core/webui/app.py`

**根因**: `create_app()` 使用硬编码默认值而非从 `context.config.get_webui_config()` 读取配置，导致测试中 `token_secret` 不匹配，JWT 验证失败返回 401。

**修复**: 将第 70 行从硬编码改为 `webui_config = context.config.get_webui_config()`。

### 3.3 API Key 断言修复

**文件**: `tests/integration/module/test_config_injection.py`

**根因**: `test_injection_chain_no_api_key` 断言 `api_key is None`，但 `_from_nanobot_config` 中 `provider_cfg.get("apiKey", "")` 返回空字符串。

**修复**: 将断言改为 `assert not llm_config.api_key`，兼容 None 和空字符串。
