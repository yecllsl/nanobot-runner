## 4. 测试用例问题说明

### 4.1 问题描述

测试文件 `tests/integration/module/test_plan_cli_integration_bug001.py` 中的 `test_plan_create_with_options` 用例存在mock配置与实际代码不一致的问题。

### 4.2 根因分析

```python
# 测试代码期望
mock_context.config.user_id = "test_user"

# 实际代码实现 (src/cli/commands/plan.py:42)
plan = engine.generate_plan(
    user_id="default",  # 硬编码,未使用context.config.user_id
    ...
)
```

### 4.3 修复建议(给开发工程师)

**方案A**: 修改测试用例,将期望值改为`user_id='default'`
```python
mock_engine.generate_plan.assert_called_once_with(
    user_id="default",  # 修改此处
    goal_distance_km=21.1,
    ...
)
```

**方案B**: 修改CLI代码,使用context中的user_id
```python
plan = engine.generate_plan(
    user_id=context.config.user_id,  # 使用配置中的user_id
    ...
)
```

**推荐**: 方案B更符合架构设计,但需要评估是否影响其他依赖`user_id="default"`的逻辑。
