**标题**: StorageManager缺少save_activities方法

**状态**: 🔴 待修复
**优先级**: P0

**测试阶段**: 集成测试
**测试人员**: 测试工程师

**Bug描述**:
StorageManager类缺少save_activities方法，导致数据存储功能无法正常工作，影响核心业务流程。

**复现步骤**:
1. 执行集成测试中的存储相关测试用例
2. 调用StorageManager实例的save_activities方法
3. 观察是否出现AttributeError异常

```python

```


**预期结果**:

- 无异常抛出

- 方法命名不一致，实际方法名为`save_data`
- 代码重构过程中方法名未统一
- 缺少方法存在性验证机制

**影响范围**:
- ✅ 数据导入功能受影响
- ✅ 数据存储功能受影响
- ✅ 集成测试无法通过
- 🔴 核心业务流程受阻

**修复建议**:
1. 统一方法命名规范，将`save_data`重命名为`save_activities`
2. 更新所有调用该方法的地方
3. 添加方法存在性验证
4. 添加单元测试验证方法功能
