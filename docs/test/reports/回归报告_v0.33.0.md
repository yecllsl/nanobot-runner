## 4. 内存与性能专项验证

### 4.1 两阶段调度效果

| 指标 | 旧方案 | v0.33.0 方案 | 改善 |
|------|--------|------------|------|
| 峰值 Python 进程数 | 8-9 | ≤3 | -66% |
| 峰值内存（监控） | 18 GB | 3.99 GB | -78% |
| 嵌套并行 | ThreadPoolExecutor(4) + pytest-xdist(-n 4) | 轻量并行 + 重量级串行 | 消除 |

**监控脚本**: `scripts/verify_memory_peak.py`（psutil 后台采样进程树）

### 4.2 单实例锁验证

| 场景 | 预期 | 实际 | 结论 |
|------|------|------|------|
| 无锁文件 | 成功获取 | is_locked=True | ✅ |
| 有效锁存在 | 拒绝并退出 | acquire()=False | ✅ |
| 过期锁（>300s） | 覆写 | 重新获取成功 | ✅ |
| 死进程锁 | 覆写 | 检测 PID 不存在 | ✅ |

### 4.3 Windows 路径处理

| 场景 | 旧方案 | v0.33.0 方案 |
|------|--------|------------|
| `scripts\foo.py` | shlex 吃反斜杠 → `scriptsfoo.py` → ruff 找不到 | `as_posix()` 强制 `/` + shlex 前 `replace("\\","/")` |

**根因**: shlex.split(posix=True) 把 `\d` 当作转义序列并吃掉反斜杠
**修复**: `f.relative_to(self.project_root).as_posix()` + `command.replace("\\", "/")`
