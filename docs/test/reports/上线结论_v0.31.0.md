## 4. 上线条件

### 4.1 必须满足的条件（已满足）

1. 无致命/严重 Bug -- 已满足
2. 核心 API 功能 100% 通过 -- 已满足（41/41 API 测试通过）
3. 后端业务逻辑无回归 -- 已满足（4338 单元测试 + 367 集成测试 + 76 E2E 测试全部通过）
4. 性能无退化 -- 已满足（26/26 性能测试通过）
5. 死代码清理完成 -- 已满足（3 个关键词零匹配）

### 4.2 可接受的风险（需知悉）

1. **覆盖率 82% vs 基线 83%**: 差 1 个百分点，核心模块覆盖率均在 80% 以上，低覆盖率集中在非核心的 UI 交互模块
2. **WebUI UI 层 45 个 Playwright 测试失败**: 不影响后端 API 功能，不影响 CLI 命令，仅影响前端自动化验证。前端功能需通过手工验证确认
3. **BUG-003 待修复**: 前端渲染与测试选择器匹配问题，建议在后续迭代中修复

## 5. 上线建议

### 5.1 可发布范围

- CLI 命令行功能
- 后端 API 接口
- 数据处理与存储
- AI 对话功能

  - 训练负荷页
  - 活动记录页
  - 身体信号页
  - 进化控制台页
  - 训练计划页
  - 设置中心页

### 5.3 后续迭代建议

1. 修复 BUG-003（WebUI UI 层 Playwright 测试匹配问题）
2. 补充覆盖率至 83% 以上（重点: evolution_store.py、provider_adapter.py）
3. 考虑为 Playwright 测试增加更健壮的等待策略

**v0.31.0 版本有条件通过上线门禁。**

核心后端功能（CLI、API、数据处理、AI 对话、训练计划、进化引擎）全部测试通过，无致命/严重 Bug，无性能回归。WebUI 前端可视化页面需手工验证确认后可发布。

**发布前置条件**: 完成 WebUI 前端页面手工验证，确认核心可视化功能可用。
