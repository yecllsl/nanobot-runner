## 3. 准入准出标准

### 3.1 准入标准
- 开发工程师已完成自测
- 单元测试覆盖率≥80%
- 无ruff ERROR级别告警

### 3.2 准出标准（上线门禁）
- P0用例通过率≥95%
- 无致命/严重bug
- 一般bug修复率≥90%
- 核心业务流程全量闭环

## 4. 风险与应对
| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 路由认证失败 | 安全风险 | 验证所有端点都需要认证 |
| 数据序列化异常 | 功能异常 | Mock边界值测试 |
| 前端构建失败 | 无法发布 | 单独验证构建流程 |
