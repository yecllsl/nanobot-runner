## 1. 测试范围

### 1.1 版本概述

**版本主题**: ML增强预测 —— 为数据充足用户提供更精准的未来洞察  
**核心目标**: 基于18个月+历史数据，用ML模型替代简单线性回归，显著提升预测准确度  
**目标用户**: 数据充足的高级用户（18个月+跑步数据，500+条记录）

### 1.2 测试需求覆盖

| 需求ID | 需求名称 | 优先级 | 测试类型 | 用例数 |

### 1.3 不纳入测试范围

- 第三方依赖内部实现（scikit-learn、scipy、shap、joblib等）
- nanobot-ai框架核心功能
- 操作系统级别的文件系统行为
- LLM模型输出内容的不确定性验证
- v0.21+版本功能（数字孪生、多视角验证等）

## 2. 测试类型与策略

### 2.1 单元测试（Unit Testing）

**职责**: 开发工程师主责，测试工程师负责规范指导和结果校验

**覆盖范围**:
- `src/core/prediction/` 核心业务逻辑
- 特征工程模块（时序特征提取、负荷特征、身体信号特征）
- ML模型训练与推理逻辑
- 数据充足度评估逻辑
- 配置解析与校验
- 数据模型序列化/反序列化

**覆盖率要求**:

| 模块 | 最低覆盖率 | 说明 |
|------|-----------|------|
| `src/core/prediction/` | ≥80% | 核心预测逻辑 |
| `src/core/prediction/baselines/` | ≥80% | 参数化基线模型 |
| `src/core/prediction/features/` | ≥80% | 特征工程 |
| `src/core/prediction/models.py` | ≥90% | 数据模型 |
| `src/core/prediction/config.py` | ≥90% | 配置校验 |

### 2.2 集成测试（Integration Testing）

**职责划分**:

| 测试类型 | 目录 | 负责人 |
|---------|------|--------|
| 模块内集成测试 | `tests/integration/module/` | 开发工程师 |
| 场景级集成测试 | `tests/integration/scene/` | 测试工程师 |

**场景级集成测试覆盖**:
- 数据导入→特征提取→模型训练→预测输出完整链路
- 数据充足度评估→自动降级策略→预测输出完整链路
- 伤病报告→标签存储→模型训练→风险预测完整链路
- 模型管理→版本回滚→预测校准完整链路
- v0.19身体信号引擎→v0.20预测引擎数据传递链路

### 2.3 端到端测试（E2E Testing）

**职责**: 测试工程师主责

**覆盖范围**:
- CLI命令完整执行流程（predict status/vdot/race/injury-risk/model）
- Agent工具调用→数据分析→结果返回完整链路
- 降级策略端到端验证（数据不足→参数化基线→基础预测）
- 用户体验流程（首次使用→数据积累→解锁ML预测）

**测试环境**: 使用临时测试目录，Mock所有外部依赖，使用`tests/data/fixtures/`下的样本FIT文件

## 3. 门禁规则

### 3.1 测试准入规则

代码进入测试环节前，必须满足以下条件：

| 条件 | 验证方式 | 责任人 |
|------|---------|--------|
| 需求规格说明书已评审通过 | 文档存在且版本匹配(v8.1) | 架构师 |
| 架构设计说明书已评审通过 | 文档存在且版本匹配(v7.1.0) | 架构师 |
| 开发完成并通过自测 | 开发者自测报告 | 开发工程师 |
| 单元测试覆盖率达标 | `pytest --cov` 报告（prediction模块≥80%） | 开发工程师 |
| 代码质量检查通过 | `ruff check` 零警告 | 开发工程师 |
| 类型检查通过 | `mypy` 无新增错误 | 开发工程师 |
| 无未解决的P0/P1 Bug | Bug清单状态 | 开发工程师 |
| ML依赖安装验证 | `uv run python -c "import sklearn, scipy, shap, joblib"` | 开发工程师 |

**准入验证命令**:
```bash
uv run ruff check src/ tests/
uv run ruff format --check src/ tests/
uv run mypy src/ --ignore-missing-imports
uv run pytest tests/unit/ --cov=src/core/prediction --cov-report=term-missing
uv run python -c "import sklearn, scipy, shap, joblib; print('ML dependencies OK')"
```

### 3.2 测试准出规则

测试完成并允许发布，必须满足以下条件：

| 条件 | 标准 | 验证方式 |
|------|------|---------|
| P0级用例通过率 | 100% | 测试报告 |
| P1级用例通过率 | ≥95% | 测试报告 |
| 致命Bug | 0个 | Bug清单 |
| 严重Bug | 0个 | Bug清单 |
| 一般Bug修复率 | ≥90% | Bug清单 |
| 核心业务流程 | 全量闭环 | E2E测试报告 |
| ML预测准确率 | VDOT误差<5%，全马误差<8分钟 | 性能测试报告 |
| 伤病预警召回率 | 3周前置预警召回率>75% | 性能测试报告 |
| 性能要求 | ML预测响应<5秒，模型训练<5分钟 | 性能测试报告 |
| 安全合规 | 无敏感信息泄露，模型本地存储 | 安全扫描报告 |
| 降级策略 | 数据不足时自动降级，不阻塞用户 | E2E测试报告 |

**准出验证命令**:
```bash
uv run pytest tests/ -v --tb=short
uv run pytest tests/integration/ -v
uv run pytest tests/e2e/ -v
uv run pytest tests/performance/ -v
```

### 3.3 上线门禁规则

**绝对禁止发布的情况**:
- 存在任何致命或严重级Bug
- P0级用例通过率 < 100%
- ML预测准确率不达标（VDOT误差≥5%或全马误差≥8分钟）
- 伤病预警召回率<75%
- 降级策略失效（数据不足时阻塞用户）
- 核心业务流程未闭环
- 测试报告未输出或未评审通过
- 安全扫描发现敏感信息泄露

**允许发布的条件**（全部满足）:
- P0-P1级用例100%通过
- 无致命/严重级Bug
- 一般级Bug修复率≥90%
- 核心业务流程全量闭环
- ML预测准确率达标
- 伤病预警召回率>75%
- 性能要求达标
- 符合需求验收标准
- 测试报告已输出并评审通过

### 7.2 测试执行顺序

```
单元测试 → 模块集成测试 → 场景级集成测试 → E2E测试 → 性能测试 → 安全测试 → 回归测试
```

### 7.3 回归测试范围

| 触发条件 | 回归范围 |
|---------|---------|
| P0 Bug修复 | P0用例全量回归+相关P1用例 |
| P1 Bug修复 | 相关用例回归 |
| 代码重构 | 受影响模块用例全量回归 |
| 依赖升级 | 全量回归 |

## 8. 风险评估与应对

### 8.1 测试风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| ML模型训练不稳定 | 预测准确率不达标 | 中 | 调整超参数，增加训练数据，使用交叉验证 |
| 测试数据不足 | 无法充分验证ML模型 | 中 | 使用合成数据补充，构造边界场景数据 |
| 性能不达标 | 预测响应时间>5秒 | 低 | 优化特征提取，使用模型缓存，减少不必要的计算 |
| 降级策略失效 | 数据不足时阻塞用户 | 低 | 充分测试各降级路径，确保无缝切换 |
| 依赖兼容性问题 | sklearn/scipy/shap版本冲突 | 低 | 锁定依赖版本，充分测试兼容性 |

### 8.2 质量风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 特征工程覆盖不足 | 预测准确率偏低 | 确保≥5类特征，每类≥2个具体特征 |
| 置信区间不合理 | 用户信任度下降 | 验证p10<p50<p90，区间宽度合理 |
| SHAP解释不准确 | 可解释性下降 | 使用标准SHAP库，验证特征重要性排序 |
| 模型退化 | 新版本模型不如旧版本 | 模型版本管理，支持回滚，准确性追踪 |
