## 3. 门禁规则

### 3.1 测试准入规则

代码进入测试环节前，必须满足以下条件：

| 条件 | 验证方式 | 责任人 |
|------|---------|--------|
| PRD v0.17.0 已评审通过 | 文档存在且版本匹配 | 架构师 |
| 架构设计已更新 | 文档存在且版本匹配 | 架构师 |
| 开发完成并通过自测 | 开发者自测报告 | 开发工程师 |
| 单元测试覆盖率达标 | `pytest --cov` 报告 | 开发工程师 |
| 代码质量检查通过 | `ruff check` 零警告 | 开发工程师 |
| 类型检查通过 | `mypy` 无新增错误 | 开发工程师 |
| 无未解决的 P0/P1 Bug | Bug 清单状态 | 开发工程师 |

**准入验证命令**:
```bash
uv run ruff check src/ tests/
uv run ruff format --check src/ tests/
uv run mypy src/ --ignore-missing-imports
uv run pytest tests/unit/ --cov=src --cov-report=term-missing
```

### 3.2 测试准出规则

测试完成并允许发布，必须满足以下条件：

| 条件 | 标准 | 验证方式 |
|------|------|---------|
| P0 级用例通过率 | 100% | 测试报告 |
| P1 级用例通过率 | ≥95% | 测试报告 |
| 致命 Bug | 0 个 | Bug 清单 |
| 严重 Bug | 0 个 | Bug 清单 |
| 一般 Bug 修复率 | ≥90% | Bug 清单 |
| 核心业务流程 | 全量闭环 | E2E 测试报告 |
| 性能指标 | 无显著退化 | 性能测试报告 |
| 安全合规 | 无敏感信息泄露 | 安全扫描报告 |

**准出验证命令**:
```bash
uv run pytest tests/ -v --tb=short
uv run pytest tests/integration/ -v
uv run pytest tests/e2e/ -v
uv run pytest tests/performance/ -v
```

### 3.3 上线门禁规则

**绝对禁止发布的情况**:
- 存在任何致命或严重级 Bug
- P0 级用例通过率 < 100%
- 核心业务流程未闭环
- v0.17.0 P0 功能（流式输出、Subagent、Cron、错误处理）未通过测试
- 测试报告未输出或未评审通过
- 安全扫描发现敏感信息泄露

**允许发布的条件**（全部满足）:
- P0-P1 级用例 100% 通过
- 无致命/严重级 Bug
- 一般级 Bug 修复率 ≥90%
- 核心业务流程全量闭环
- v0.17.0 所有 P0 功能全量通过
- 符合 PRD 验收标准
- 测试报告已输出并评审通过

## 7. 风险评估

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|----------|
| 流式输出与 Rich Console 兼容性 | 中 | CLI 显示异常 | 先实现基础版本，再优化样式 |
| Subagent 调用链复杂度 | 中 | 调用失败或性能差 | 仅验证基础调用，提供降级方案 |
| on_stream Hook 与现有 Hook 冲突 | 中 | 透明化追踪数据丢失 | 确保不干扰 before_iteration/after_iteration |
| Cron 任务过多影响性能 | 低 | 系统资源占用 | 限制并发任务数 ≤ 5 |
| ask_user 与 CLI 集成 | 低 | CLI 不支持按钮交互 | 降级为文本选择 |
