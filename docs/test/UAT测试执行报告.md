### 3.2 Bug详情

#### BUG-001: 报告生成失败（严重）

| 项目 | 内容 |
|------|------|
| Bug ID | BUG-001 |
| 所属模块 | 训练计划与报告 |
| 关联用例 | UAT-019 |
| 严重等级 | 严重 |
| 标题 | 周报告生成失败：`'VdotTrendItem' object has no attribute 'get'` |
| 复现步骤 | 1. 导入FIT文件 2. 执行 `nanobotrun report weekly` |
| 实际结果 | 报告生成失败，抛出属性错误 |
| 预期结果 | 成功生成周报告，包含VDOT趋势、训练负荷等 |
| 根因分析 | 报告模块在访问VDOT趋势数据时，使用了 `.get()` 方法，但 `VdotTrendItem` 是数据类而非字典 |
| 修复建议 | 修改报告生成代码，使用属性访问 `item.vdot` 替代 `item.get('vdot')` |
| 出现版本 | v0.18.0 |
| 优先级 | P0 |

#### BUG-002: VDOT趋势图显示异常（一般）

| 项目 | 内容 |
|------|------|
| Bug ID | BUG-002 |
| 所属模块 | 数据可视化 |
| 关联用例 | UAT-061 |
| 严重等级 | 一般 |
| 标题 | VDOT趋势图ANSI颜色码未正确渲染，数据点重复显示 |
| 复现步骤 | 1. 导入FIT文件 2. 执行 `nanobotrun viz vdot --days 30` |
| 实际结果 | 图表显示大量重复的同一日期数据点，ANSI颜色码未正确解析 |
| 预期结果 | 显示不同日期的VDOT趋势，颜色正常渲染 |
| 根因分析 | 可视化模块在生成图表时，数据聚合逻辑存在问题，导致同一日期数据重复输出 |
| 修复建议 | 检查VDOT趋势数据聚合逻辑，确保按日期去重；修复ANSI颜色码渲染问题 |
| 出现版本 | v0.18.0 |
| 优先级 | P1 |

#### BUG-003: 可视化中文乱码（一般）

| 项目 | 内容 |
|------|------|
| Bug ID | BUG-003 |
| 所属模块 | 数据可视化 |
| 关联用例 | UAT-062, UAT-063, UAT-064 |
| 严重等级 | 一般 |
| 标题 | PowerShell输出中文显示为乱码 |
| 复现步骤 | 1. 执行任意可视化命令 `nanobotrun viz vdot/load/hr-zones` |
| 实际结果 | 图表标题和表头中文显示为乱码（如"瓒嬪娍鍥捐〃"） |
| 预期结果 | 中文正常显示 |
| 根因分析 | PowerShell终端编码与Rich库输出编码不匹配，需设置正确的代码页 |
| 修复建议 | 在可视化模块输出前设置 `sys.stdout.reconfigure(encoding='utf-8')`，或在文档中说明需要设置 `chcp 65001` |
| 出现版本 | v0.18.0 |
| 优先级 | P1 |
