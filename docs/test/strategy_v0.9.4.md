# v0.9.4 测试策略

> **文档版本**: v1.0.0  
> **编制日期**: 2026-04-18  
> **版本目标**: 优化用户初始化流程和数据迁移体验  
> **需求来源**: v0.9.4需求规格说明书  
> **架构依据**: v0.9.4架构设计说明书

---

## 1. 测试概述

### 1.1 测试目标

确保v0.9.4版本的四大核心功能（一键初始化向导、智能数据迁移、配置验证工具、Workspace位置规范）功能正确、性能达标、用户体验良好，无致命/严重级别缺陷。

### 1.2 测试范围

| 功能模块 | 测试范围 | 优先级 |
|---------|---------|--------|
| **初始化向导** | InitWizard、ConfigGenerator、EnvManager、环境检测、目录创建、配置引导、配置验证、配置文件生成 | P0 |
| **数据迁移** | MigrationEngine、MigrationStrategy、BackupManager、VerifyManager、版本检测、备份机制、迁移执行、回滚机制、完整性校验 | P0 |
| **配置验证** | ConfigValidator、格式验证、完整性验证、有效性验证、一致性验证、API连通性测试 | P0 |
| **Workspace管理** | WorkspaceManager、路径解析、环境变量支持、配置文件支持、默认路径、跨平台兼容 | P0 |
| **CLI集成** | system命令（init/migrate/validate/config）、参数解析、错误处理、输出格式 | P0 |
| **配置管理** | ConfigManager扩展（allow_default模式）、配置加载优先级、配置一致性校验 | P0 |

### 1.3 不在测试范围内

| 模块 | 原因 |
|------|------|
| 飞书通知配置(F1.1.6) | P2优先级，可选功能，后续迭代测试 |
| 增量迁移支持(F2.2.9) | P1优先级，后续迭代测试 |
| 一键回滚机制(F2.2.12) | P1优先级，后续迭代测试 |
| templates/skills/ | 非核心功能，独立测试 |

## 11. 测试规范

### 11.1 测试用例设计规范

1. **用例命名**: `TC-{模块缩写}-{序号}`，如`TC-INIT-001`
2. **用例标题**: 简洁明了，描述测试目的
3. **前置条件**: 明确列出测试前需要准备的条件
4. **测试步骤**: 步骤清晰，可执行，无歧义
5. **预期结果**: 明确、可验证，无模糊描述
6. **优先级**: P0/P1/P2三级，P0为必须通过

### 11.2 自动化测试开发规范

1. **测试文件命名**: `test_{模块名}.py`
2. **测试函数命名**: `test_{功能描述}()`
3. **夹具使用**: 使用`@pytest.fixture`管理测试数据
4. **Mock策略**: Mock外部API，不Mock内部业务逻辑
5. **断言规范**: 使用明确的断言，避免模糊断言

### 11.3 bug提交规范

1. **bug标题**: 简洁明了，描述问题现象
2. **复现步骤**: 详细、可复现，无遗漏
3. **预期结果**: 明确描述期望的行为
4. **实际结果**: 准确描述实际发生的行为
5. **根因分析**: 分析问题产生的根本原因
6. **修复建议**: 给出具体的修复建议
