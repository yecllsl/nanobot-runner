# UAT - ML智能预测模块 (v0.20)

> **模块版本**: v0.20.0
> **用例范围**: UAT-081 ~ UAT-091
> **优先级分布**: P0: 5个, P1: 6个

---

## UAT-081: VDOT趋势预测

- **优先级**: P0
- **前置条件**: 已导入≥18个月跑步数据（400+条）
- **操作步骤**:
  1. 执行 `uv run nanobotrun predict vdot`
- **预期结果**: 显示VDOT趋势预测，含ML增强/参数化/基础三种模式结果，当前数据层级自动选择最优模式
- **验证要点**: 预测值合理（VDOT 30-80范围），置信区间显示正常，SHAP特征重要性可读

## UAT-082: VDOT预测 - 数据不足场景

- **优先级**: P1
- **前置条件**: 跑步数据<200条

- **预期结果**: 降级为基础预测模式，显示数据积累建议和当前进度
- **验证要点**: 降级提示清晰，显示数据充足度进度百分比，不崩溃

- **优先级**: P1

- **操作步骤**:
  1. 执行 `uv run nanobotrun predict model status --type all`

## UAT-090: 模型回滚

- **优先级**: P1
- **前置条件**: 已训练至少2个版本的模型
- **操作步骤**:
  1. 先训练一次VDOT模型（如已训练则跳过）
  2. 再次训练VDOT模型（生成v2版本）
  3. 执行 `uv run nanobotrun predict model rollback --type vdot`
- **预期结果**: 回滚成功，显示回滚消息
- **验证要点**: 回滚后模型版本号减小，状态仍为"可用"

## UAT-091: 预测 - 数据不足降级链路

- **优先级**: P0
- **前置条件**: 数据量分别处于L1/L2/L3层级
- **操作步骤**:
  1. L3(<200条): 执行 `uv run nanobotrun predict vdot` → 预期基础模式
  2. L2(200-400条): 执行 `uv run nanobotrun predict vdot` → 预期参数化模式
  3. L1(400+条): 执行 `uv run nanobotrun predict vdot` → 预期ML增强模式
- **预期结果**: 各层级自动降级到对应预测模式
- **验证要点**: 降级逻辑正确，提示清晰，prediction_type字段正确（basic/parametric/ml_enhanced）
