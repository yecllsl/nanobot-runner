# Nanobot Runner 用户验收测试（UAT）指南

> **文档版本**: v2.0  
> **适用版本**: v0.9.4  
> **编写日期**: 2026-04-19  
> **测试类型**: 用户验收测试（User Acceptance Testing）  
> **自动化支持**: 本文档已补充标准输出示例和判定规则，支持AI Agent自动化执行

---

## 1. 文档目的

本指南用于指导用户（技术型跑者/跑步数据分析师）或AI Agent对 Nanobot Runner v0.9.4 版本进行验收测试，验证产品是否满足业务需求和使用预期。

**UAT 与系统测试的区别**:
| 维度 | 系统测试 | UAT |
|------|---------|-----|
| 执行者 | 测试工程师 | 最终用户 / AI Agent |
| 目标 | 发现技术缺陷 | 验证业务需求 |
| 视角 | 技术实现 | 用户体验 |
| 环境 | 测试环境 | 类生产环境 |

## 2. 测试准备

### 2.1 环境要求

| 项目 | 要求 | 验证方法 |
|------|------|---------|
| 操作系统 | Windows 10+/macOS 12+ | `uname -a` 或系统设置 |
| Python | 3.11+ | `python --version` |
| uv包管理器 | 已安装 | `uv --version` |
| 磁盘空间 | ≥500MB | `df -h` |
| 网络 | 可选（仅Agent功能需要） | `ping github.com` |

### 2.2 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/yecllsl/nanobot-runner.git
cd nanobot-runner

# 2. 创建虚拟环境
uv venv

# 3. 激活虚拟环境（Windows PowerShell）
.venv\Scripts\activate

# 4. 同步依赖
uv sync --all-extras

# 5. 验证安装
uv run nanobotrun --help
```

**安装成功标志**: 显示帮助信息，包含 `data`、`analysis`、`agent`、`report`、`system`、`init` 等子命令。

**自动化判定规则**:
- ✅ **通过**: 命令退出码为0，输出包含"Usage:"或"Commands:"关键字
- ❌ **失败**: 命令退出码非0，或输出包含"Error"、"ModuleNotFoundError"

### 2.3 测试数据准备

**方式一：使用项目自带样本数据**

```bash
# 样本数据位于 tests/data/fixtures/ 目录
ls tests/data/fixtures/
```

| 文件 | 类型 | 用途 |
|------|------|------|
| `easy_run_*.fit` | 轻松跑 | 基础功能验证 |
| `tempo_run_*.fit` | 节奏跑 | 强度分析验证 |
| `long_run_*.fit` | 长距离跑 | 耐力分析验证 |
| `interval_*.fit` | 间歇跑 | 间歇训练验证 |

**方式二：使用个人真实数据**

```bash
# 准备个人FIT文件目录
mkdir ~/test-fit-files
# 将Garmin/Coros导出的FIT文件复制到该目录
```

### 2.4 初始化配置

```bash
# 交互式初始化（推荐新用户）
uv run nanobotrun system init

# 自动模式（适合有经验用户）
uv run nanobotrun system init --auto
```

**初始化成功标志**:
- 创建 `~/.nanobot-runner/` 目录
- 生成 `config.json` 配置文件
- 显示初始化完成提示

**自动化判定规则**:
- ✅ **通过**: 退出码为0，输出包含"✓ 初始化完成"或"配置文件:"
- ❌ **失败**: 退出码非0，或输出包含"✗ 初始化失败"

### 2.5 测试环境清理（新增）

**测试前清理**:
```bash
# 清理旧配置和数据（谨慎执行）
rm -rf ~/.nanobot-runner/
```

**测试后清理**:
```bash
# 清理测试数据
rm -rf ~/.nanobot-runner/
# 如需保留配置，仅清理数据
rm -f ~/.nanobot-runner/data/activities_*.parquet
```

## 6. 常见问题排查

### 6.1 安装问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `uv: command not found` | uv未安装 | 执行 `curl -LsSf https://astral.sh/uv/install.sh | sh` |
| `Python version error` | Python版本<3.11 | 升级Python到3.11+ |
| `Dependency conflict` | 依赖冲突 | 执行 `uv sync --reinstall` |

### 6.2 导入问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `File not found` | 文件路径错误 | 检查文件路径是否正确 |
| `Invalid FIT file` | 文件格式不正确 | 确认是标准FIT格式 |
| `Permission denied` | 权限不足 | 检查文件读写权限 |

### 6.3 分析问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `No data available` | 未导入数据 | 先执行数据导入 |
| `VDOT calculation failed` | 距离<1500m | VDOT需要≥1500m的跑步记录 |
| `Heart rate drift N/A` | 无心率数据 | 需要包含心率数据的FIT文件 |

#### 8.3.3 自动化执行脚本示例

```bash
#!/bin/bash
# UAT自动化执行脚本示例

PASS_COUNT=0
FAIL_COUNT=0
TOTAL_COUNT=0

run_test() {
    local test_id=$1
    local command=$2
    local pass_keyword=$3
    local fail_keyword=$4
    local expected_exit_code=$5
    
    TOTAL_COUNT=$((TOTAL_COUNT + 1))
    
    # 执行命令并捕获输出和退出码
    output=$(eval "$command" 2>&1)
    exit_code=$?
    
    # 判定逻辑
    if [ $exit_code -eq $expected_exit_code ]; then
        if echo "$output" | grep -q "$pass_keyword"; then
            echo "✅ $test_id: 通过"
            PASS_COUNT=$((PASS_COUNT + 1))
        else
            echo "❌ $test_id: 失败 (未找到通过关键字: $pass_keyword)"
            FAIL_COUNT=$((FAIL_COUNT + 1))
        fi
    else
        echo "❌ $test_id: 失败 (退出码: $exit_code, 期望: $expected_exit_code)"
        FAIL_COUNT=$((FAIL_COUNT + 1))
    fi
}

# 执行测试用例
run_test "UAT-DI-001" "uv run nanobotrun data import tests/data/fixtures/easy_run_20240101.fit" "[OK] 导入成功" "错误:" 0
run_test "UAT-DQ-001" "uv run nanobotrun data stats" "[Stats] 统计信息" "暂无数据" 0
run_test "UAT-AN-001" "uv run nanobotrun analysis vdot" "VDOT趋势分析" "暂无VDOT数据" 0

# 输出汇总
echo ""
echo "=== 验收测试汇总 ==="
echo "总用例数: $TOTAL_COUNT"
echo "通过: $PASS_COUNT"
echo "失败: $FAIL_COUNT"
echo "通过率: $((PASS_COUNT * 100 / TOTAL_COUNT))%"
```
