## 一、测试说明

### 1.1 测试目的

本指南用于验证 Nanobot Runner 是否满足用户需求，确保系统功能完整、操作便捷、体验良好。

### 1.2 测试环境要求

| 项目     | 要求                           |
| ------ | ---------------------------- |
| 操作系统   | Windows 10/11 或 macOS 10.15+ |
| Python | 3.11 或更高版本                   |
| 网络环境   | 需要网络连接（用于安装依赖和 AI 功能）        |
| 测试数据   | 至少准备 3-5 个 FIT 文件（跑步记录）      |

### 1.3 测试前准备

1. **安装项目**
   ```bash
   # 方式一：一键安装
   curl -fsSL https://raw.githubusercontent.com/yecllsl/nanobot-runner/main/scripts/install.sh | bash

   # 方式二：手动安装
   git clone https://github.com/yecllsl/nanobot-runner.git
   cd nanobot-runner
   uv venv
   .venv\Scripts\activate  # Windows
   uv sync --all-extras
   ```
2. **准备测试数据**
   - 从运动手表导出 FIT 文件（Garmin、Coros 等）
   - 建议准备不同类型的跑步记录：轻松跑、间歇跑、长距离跑
3. **配置 AI 功能**（可选）
   - 如需测试 Agent 聊天功能，需配置 API Key

## 四、附录

### 4.1 常用命令速查

```bash
# 系统管理
nanobotrun system init          # 初始化工作区
nanobotrun system version       # 查看版本

# 数据管理
nanobotrun data import <路径>   # 导入数据
nanobotrun data stats           # 查看统计
nanobotrun data stats --year 2024  # 按年份统计

# 数据分析
nanobotrun analysis vdot        # VDOT 分析
nanobotrun analysis load        # 训练负荷分析
nanobotrun analysis hr-drift    # 心率漂移分析

# Agent 交互
nanobotrun agent chat           # 启动 AI 聊天

# 报告生成
nanobotrun report weekly        # 生成周报
nanobotrun report monthly       # 生成月报
nanobotrun report profile show  # 查看用户画像
```

### 4.2 测试数据建议

为充分验证系统功能，建议准备以下类型的测试数据：

| 数据类型   | 数量    | 用途        |
| ------ | ----- | --------- |
| 轻松跑记录  | 3-5 个 | 验证基础功能    |
| 间歇跑记录  | 2-3 个 | 验证高强度训练分析 |
| 长距离跑记录 | 2-3 个 | 验证耐力训练分析  |
| 损坏文件   | 1 个   | 验证异常处理    |
| 重复文件   | 1 个   | 验证去重功能    |
