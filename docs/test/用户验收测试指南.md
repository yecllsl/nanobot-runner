# Nanobot Runner 用户验收测试（UAT）指南

> **文档版本**: v3.0  
> **适用版本**: v0.12.0  
> **编写日期**: 2026-04-25  
> **测试类型**: 用户验收测试（User Acceptance Testing）  
> **自动化支持**: 本文档已补充标准输出示例和判定规则，支持AI Agent自动化执行

---

## 1. 文档目的

本指南用于指导用户（技术型跑者/跑步数据分析师）或AI Agent对 Nanobot Runner v0.12.0 版本进行验收测试，验证产品是否满足业务需求和使用预期。

**UAT 与系统测试的区别**:
| 维度 | 系统测试 | UAT |
|------|---------|-----|
| 执行者 | 测试工程师 | 最终用户 / AI Agent |
| 目标 | 发现技术缺陷 | 验证业务需求 |
| 视角 | 技术实现 | 用户体验 |
| 环境 | 测试环境 | 类生产环境 |

## 2. 测试准备

### 2.1 环境要求

| 项目 | 要求 | 验证方法 |
|------|------|---------|
| 操作系统 | Windows 10+/macOS 12+ | `uname -a` 或系统设置 |
| Python | 3.11+ | `python --version` |
| uv包管理器 | 已安装 | `uv --version` |
| 磁盘空间 | ≥500MB | `df -h` |
| 网络 | 可选（仅Agent功能需要） | `ping github.com` |

### 2.2 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/yecllsl/nanobot-runner.git
cd nanobot-runner

# 2. 创建虚拟环境
uv venv

# 3. 激活虚拟环境（Windows PowerShell）
.venv\Scripts\activate

# 4. 同步依赖
uv sync --all-extras

# 5. 验证安装
uv run nanobotrun --help
```

**安装成功标志**: 显示帮助信息，包含 `data`、`analysis`、`agent`、`report`、`system`、`init`、`plan` 等子命令。

**自动化判定规则**:
- ✅ **通过**: 命令退出码为0，输出包含"Usage:"或"Commands:"关键字
- ❌ **失败**: 命令退出码非0，或输出包含"Error"、"ModuleNotFoundError"

### 2.3 测试数据准备

**方式一：使用项目自带样本数据**

```bash
# 样本数据位于 tests/data/fixtures/ 目录
ls tests/data/fixtures/
```

| 文件 | 类型 | 用途 |
|------|------|------|
| `easy_run_*.fit` | 轻松跑 | 基础功能验证 |
| `tempo_run_*.fit` | 节奏跑 | 强度分析验证 |
| `long_run_*.fit` | 长距离跑 | 耐力分析验证 |
| `interval_*.fit` | 间歇跑 | 间歇训练验证 |

**方式二：使用个人真实数据**

```bash
# 准备个人FIT文件目录
mkdir ~/test-fit-files
# 将Garmin/Coros导出的FIT文件复制到该目录
```

### 2.4 初始化配置

```bash
# 交互式初始化（推荐新用户）
uv run nanobotrun system init

# 自动模式（适合有经验用户）
uv run nanobotrun system init --auto
```

**初始化成功标志**:
- 创建 `~/.nanobot-runner/` 目录
- 生成 `config.json` 配置文件
- 显示初始化完成提示

**自动化判定规则**:
- ✅ **通过**: 退出码为0，输出包含"✓ 初始化完成"或"配置文件:"
- ❌ **失败**: 退出码非0，或输出包含"✗ 初始化失败"

### 2.5 测试环境清理（新增）

**测试前清理**:
```bash
# 清理旧配置和数据（谨慎执行）
rm -rf ~/.nanobot-runner/
```

**测试后清理**:
```bash
# 清理测试数据
rm -rf ~/.nanobot-runner/
# 如需保留配置，仅清理数据
rm -f ~/.nanobot-runner/data/activities_*.parquet
```

| **操作步骤** | 1. 执行 `uv run nanobotrun plan delete --plan-id plan_20240115_001 --confirm` |
| **预期结果** | 成功删除指定计划 |
| **优先级** | P2 |

**标准输出示例**:
```
╭──────────────── [Delete] 删除计划 ───────────────╮
计划ID: plan_20240115_001
目标: 全程马拉松 3:30:00

⚠️  此操作不可恢复，确定删除? [y/N]: y

✓ 计划已删除
╰──────────────────────────────────────────────────╯
```

**自动化判定规则**:
- ✅ **通过**: 退出码为0，输出包含"✓ 计划已删除"
- ❌ **失败**: 退出码非0，或输出包含"删除失败"
