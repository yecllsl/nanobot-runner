## 4. Bug清单

### BUG-001: 月报生成失败（P0）

| 项目 | 内容 |
|------|------|
| **Bug ID** | BUG-001 |
| **所属模块** | report_service.py |
| **严重等级** | P0（核心功能异常） |
| **Bug标题** | 月报生成时跨年份查询导致ColumnNotFoundError |
| **复现步骤** | 1. 导入跨年份数据（2022年和2026年）<br>2. 执行 `uv run nanobotrun report monthly` |
| **实际结果** | 报错：`ColumnNotFoundError: unable to find column "session_start_time"; valid columns: []` |
| **预期结果** | 正常生成月报 |
| **根因分析** | `storage.py` 的 `query_by_date_range` 方法在查询时，可能未正确加载所有年份的Parquet文件，导致LazyFrame为空，无法找到 `session_start_time` 列 |
| **修复建议** | 检查 `storage.py` 中 `query_by_date_range` 方法的实现，确保跨年份查询时正确合并所有年份的Parquet文件数据 |
| **出现版本** | v0.12.0 |
| **状态** | 待修复 |

### BUG-002: 用户画像显示无数据（P0）

| 项目 | 内容 |
|------|------|
| **Bug ID** | BUG-002 |
| **所属模块** | profile |
| **严重等级** | P0（核心功能异常） |
| **Bug标题** | 用户画像显示"暂无跑步数据"，但实际已导入数据 |
| **复现步骤** | 1. 导入10个FIT文件<br>2. 执行 `uv run nanobotrun data stats` 确认有数据<br>3. 执行 `uv run nanobotrun report profile show` |
| **实际结果** | 输出"暂无跑步数据" |
| **预期结果** | 显示用户画像信息（跑步次数、距离、能力指标等） |
| **根因分析** | 用户画像模块的数据查询逻辑可能与 `data stats` 使用不同的数据源或查询方法，导致无法正确读取已导入的数据 |
| **修复建议** | 检查 `profile` 模块的数据加载逻辑，确保与 `data stats` 使用相同的数据源和查询方法 |
| **出现版本** | v0.12.0 |
| **状态** | 待修复 |

### BUG-003: 日期范围查询返回空结果（P1）

| 项目 | 内容 |
|------|------|
| **Bug ID** | BUG-003 |
| **所属模块** | data stats |
| **严重等级** | P1（重要功能异常） |
| **Bug标题** | 按日期范围查询返回"暂无数据"，但按年份查询有数据 |
| **复现步骤** | 1. 导入2022年和2026年数据<br>2. 执行 `uv run nanobotrun data stats --year 2022` 确认有9条记录<br>3. 执行 `uv run nanobotrun data stats --start 2022-03-01 --end 2022-03-31` |
| **实际结果** | 返回"暂无数据" |
| **预期结果** | 返回2022年3月的跑步统计数据 |
| **根因分析** | 日期范围过滤逻辑可能存在问题，与按年份过滤使用不同的代码路径，导致过滤条件未正确应用 |
| **修复建议** | 检查 `data stats` 命令中 `--start` 和 `--end` 参数的处理逻辑，确保日期过滤条件正确应用到Parquet查询 |
| **出现版本** | v0.12.0 |
| **状态** | 待修复 |

## 9. 测试结论

### 综合评估: **暂不建议发布**

Nanobot Runner v0.12.0 核心功能（数据导入、VDOT分析、训练负荷、训练计划）运行正常，性能指标达标。但存在以下问题需要修复后才能发布：

1. **月报生成失败**（P0）：跨年份数据查询时出现ColumnNotFoundError
2. **用户画像无数据**（P0）：已导入数据但用户画像模块无法读取
3. **日期范围查询异常**（P1）：按日期范围过滤返回空结果

**修复建议优先级**:
1. 🔴 BUG-001（月报生成）- 影响报告功能完整性
2. 🔴 BUG-002（用户画像）- 影响核心用户体验
3. 🟡 BUG-003（日期范围查询）- 影响数据查询灵活性

**预计修复后回归测试时间**: 1-2小时
