## 一、文档概述

### 1.1 目的

本文档为 v0.4.0 版本提供完整的人工验收测试用例执行手册，用于指导测试人员按照标准化流程完成所有验收测试用例的执行与结果记录。

### 1.2 适用范围

本文档适用于 v0.4.0 版本的最终验收测试，涵盖以下新增功能模块：

- Profile 用户画像引擎
- TrainingPlan 训练计划引擎
- RacePrediction 比赛预测引擎
- ReportService 报告服务
- FeishuCalendar 日历同步
- FeishuBot 机器人交互

### 1.3 测试通过标准

| 指标       | 标准   | 说明             |
| -------- | ---- | -------------- |
| P0 功能通过率 | 100% | 所有 P0 功能必须全部通过 |
| P1 功能通过率 | ≥95% | 允许少量非关键功能缺陷    |
| 严重缺陷数    | 0    | 不允许存在严重及以上级别缺陷 |
| 一般缺陷数    | ≤3   | 一般缺陷需有规避方案     |
| 测试用例执行率  | 100% | 所有测试用例必须执行     |

## 二、测试环境要求

### 2.1 硬件环境

| 项目  | 最低要求      | 推荐配置       |
| --- | --------- | ---------- |
| CPU | 双核 2.0GHz | 四核 2.5GHz+ |
| 内存  | 4GB       | 8GB+       |
| 硬盘  | 10GB 可用空间 | 20GB+ SSD  |
| 网络  | 稳定互联网连接   | 宽带网络       |

### 2.2 软件环境

| 软件     | 版本要求                               | 验证命令               |
| ------ | ---------------------------------- | ------------------ |
| 操作系统   | Windows 10/11, macOS 10.15+, Linux | -                  |
| Python | 3.11+                              | `python --version` |
| pip    | 最新版                                | `pip --version`    |
| Git    | 2.0+                               | `git --version`    |

### 2.3 依赖安装

```bash
# 1. 克隆仓库
git clone https://github.com/yecllsl/nanobot-runner.git
cd nanobot-runner

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 4. 安装依赖
pip install -e .[dev,test]

# 5. 验证安装
nanobotrun --version
```

### 2.4 配置要求

| 配置项        | 配置方法      | 说明            |
| ---------- | --------- | ------------- |
| 飞书 Webhook | 配置文件或环境变量 | 用于测试报告推送功能    |
| 测试数据       | 准备 FIT 文件 | 用于测试数据导入和分析功能 |

## 三、测试数据准备

### 3.1 FIT 文件准备

**测试数据要求**:

| 数据类型       | 数量    | 用途     |
| ---------- | ----- | ------ |
| 正常跑步记录     | ≥10 个 | 功能测试   |
| 异常数据文件     | 2-3 个 | 异常处理测试 |
| 大文件(>10MB) | 1-2 个 | 性能测试   |
| 重复文件       | 2 个   | 去重测试   |

**测试数据获取**:

- 从 Garmin 设备导出真实 FIT 文件
- 使用项目提供的测试数据: `tests/data/real_fit_files/`

### 3.2 配置数据准备

**飞书配置** (可选):

```bash
# 设置飞书 Webhook (用于测试推送功能)
export FEISHU_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/your-webhook-url"
```

### 3.3 用户信息准备

| 信息项    | 示例值          | 用途          |
| ------ | ------------ | ----------- |
| 用户年龄   | 30           | 画像构建、训练计划生成 |
| 目标比赛距离 | 21.0975 (半马) | 训练计划生成      |
| 目标比赛日期 | 2026-06-01   | 训练计划生成      |

### 10.3 常见问题 FAQ

**Q1: FIT 文件导入失败怎么办？**
A: 检查文件格式是否正确，确保是 Garmin 设备导出的标准 FIT 文件。

**Q2: 飞书推送失败怎么办？**
A: 检查 Webhook 配置是否正确，确保网络连接正常。

**Q3: 训练计划生成失败怎么办？**
A: 确保已导入足够的训练数据（至少 10 条记录），并且已构建用户画像。
