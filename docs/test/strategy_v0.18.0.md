## 1. 测试范围

### 1.1 测试目标

v0.18.0版本核心目标为**终端内可视化 + 数据可导出**，测试策略聚焦于：
- 终端图表渲染正确性与兼容性
- 数据导出功能完整性与格式正确性
- 报告图表增强功能
- 性能、安全性、兼容性非功能需求

### 1.2 测试类型

| 测试类型 | 测试范围 | 测试方式 | 负责人 |
|---------|---------|---------|--------|
| **单元测试** | 新增模块核心逻辑（visualization/export） | pytest | 开发工程师 |
| **集成测试** | 模块间交互、数据流闭环 | pytest + 测试数据 | 开发工程师 |
| **场景级集成测试** | 端到端业务流程（导入→分析→可视化→导出） | CLI命令执行 | 测试工程师 |
| **E2E测试** | 用户真实操作路径、终端渲染效果 | 手工+自动化 | 测试工程师 |
| **性能测试** | 图表渲染时间、导出时间、内存占用 | 计时+内存监控 | 测试工程师 |
| **兼容性测试** | Windows终端（PowerShell/CMD/Windows Terminal） | 多终端执行 | 测试工程师 |

## 8. 上线门禁规则

### 8.1 强制门禁

| 门禁项 | 要求 | 验证方式 |
|--------|------|---------|
| P0用例通过率 | 100% | 测试执行报告 |
| P1用例通过率 | ≥90% | 测试执行报告 |
| 致命/严重bug | 0个 | bug清单 |
| 一般bug修复率 | ≥90% | bug清单 |
| 核心流程闭环 | 100% | E2E测试报告 |
| 覆盖率要求 | 新增模块≥80% | pytest-cov报告 |

### 8.2 建议门禁

| 门禁项 | 要求 | 验证方式 |
|--------|------|---------|
| 性能指标 | 全部达标 | 性能测试报告 |
| 兼容性 | Windows终端全通过 | 兼容性测试报告 |
| 导出格式验证 | Pandas/Polars可读 | 格式验证测试 |

**只有满足全部强制门禁，才能输出测试通过报告，放行进入发布环节。**
