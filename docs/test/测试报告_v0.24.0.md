## 3. 失败用例详细信息

### 3.1 唯一失败用例

| 字段 | 内容 |
|------|------|
| **测试文件** | `tests/unit/agents/test_tools.py` |
| **测试类** | `TestCreateTools` |
| **测试方法** | `test_create_tools_returns_list` |
| **错误行号** | 538 |
| **错误消息** | `assert 54 == 51` |
| **实际值** | 54（当前注册的 Agent 工具总数） |
| **期望值** | 51（测试中硬编码的期望数量） |
| **差值** | +3 |

**根因分析**:

v0.23.0 进化模块新增了以下工具，但 `test_create_tools_returns_list` 中的期望计数未同步更新：

1. `record_feedback` -- 记录用户反馈
2. `check_plan_execution` -- 检查计划执行状态
3. `check_prediction_accuracy` -- 检查预测准确度
4. `get_decision_history` -- 查询决策历史
5. `analyze_training_response` -- v0.24 新增：分析训练响应性
6. `get_calibration_status` -- v0.24 新增：查看校准状态

实际工具数量比测试中硬编码的期望值多 3 个。这是一个典型的**测试断言过期**问题：工具注册正常增加了 3 个新工具，但测试中的断言值未随代码变更同步更新。

**严重等级**: **LOW（优化）** -- 不影响任何功能，纯测试代码中的期望值过期，工具注册和功能均正常工作。

**修复建议**: 将 `tests/unit/agents/test_tools.py` 第 538 行的断言从 `assert result == 51` 更新为 `assert result == 54`（或使用 `assert result >= 51` 更灵活地避免类似问题）。
