## 1. 测试目标与背景

### 1.1 版本定位

release/v0.12 是 Nanobot Runner 探索期的关键版本，标志着**智能跑步计划系统**的完整交付。该版本覆盖 v0.10.0-v0.12.0 三个迭代的累计功能，是首个实现"数据驱动→执行反馈→智能调整→目标预测"完整闭环的版本。

### 1.2 测试目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| 验证智能跑步计划全链路闭环 | 计划生成→执行反馈→智能调整→目标预测→长期规划 | P0 |
| 验证核心数据分析功能正确性 | VDOT/TSS/心率漂移/训练负荷 | P0 |
| 验证数据导入存储链路稳定性 | FIT解析→去重→Parquet存储→查询 | P0 |
| 验证CLI命令完整性与可用性 | 全量命令可用，输出格式正确 | P0 |
| 验证已知Bug修复效果 | BUG-001~BUG-004回归验证 | P0 |
| 评估遗留风险并给出上线结论 | UAT遗留问题风险评估 | P1 |

## 3. 门禁规则

### 3.1 测试准入规则

代码进入测试环节前，必须满足以下条件：

| 条件 | 验证方式 | 责任人 | 状态 |
|------|---------|--------|------|
| 需求规格已评审通过 | 文档存在且版本匹配 | 架构师 | ✅ |
| 架构设计已评审通过 | 文档存在且版本匹配 | 架构师 | ✅ |
| 开发完成并通过自测 | 开发者自测报告 | 开发工程师 | ✅ |
| 单元测试覆盖率达标 | `pytest --cov` 报告 | 开发工程师 | ✅ 86% |
| 代码质量检查通过 | `ruff check` 零警告 | 开发工程师 | ✅ |
| 类型检查通过 | `mypy` 无新增错误 | 开发工程师 | ✅ |
| 无未解决的P0/P1 Bug | Bug清单状态 | 开发工程师 | ✅ |

**准入验证命令**:
```powershell
# 代码质量检查
uv run ruff check src/ tests/
uv run ruff format --check src/ tests/
uv run mypy src/ --ignore-missing-imports

# 单元测试覆盖率
uv run pytest tests/unit/ --cov=src --cov-report=term-missing

# 覆盖率验证
# core ≥ 80% | agents ≥ 70% | cli ≥ 60% | plan ≥ 80%
```

### 3.2 测试准出规则

测试完成并允许发布，必须满足以下条件：

| 条件 | 标准 | 验证方式 | 状态 |
|------|------|---------|------|
| P0级用例通过率 | 100% | 测试报告 | ✅ |
| P1级用例通过率 | ≥95% | 测试报告 | ⚠️ 87.5% |
| 致命Bug | 0个 | Bug清单 | ✅ |
| 严重Bug | 0个 | Bug清单 | ✅ |
| 一般Bug修复率 | ≥90% | Bug清单 | ⚠️ 0% (1个遗留) |
| 核心业务流程 | 全量闭环 | E2E测试报告 | ✅ |
| 智能跑步计划功能 | 全量闭环 | 专项测试报告 | ✅ |
| 性能退化 | 无显著退化 | 性能测试报告 | ✅ |
| 安全合规 | 无敏感信息泄露 | 安全扫描报告 | ✅ |

### 3.3 上线门禁规则

**绝对禁止发布的情况**:
- ❌ 存在任何致命或严重级Bug
- ❌ P0级用例通过率 < 100%
- ❌ 核心业务流程未闭环
- ❌ 智能跑步计划核心功能未通过测试
- ❌ 测试报告未输出或未评审通过
- ❌ 安全扫描发现敏感信息泄露

**允许发布的条件**（全部满足）:
- ✅ P0-P1级用例100%通过（或P1有合理豁免）
- ✅ 无致命/严重级Bug
- ✅ 一般级Bug修复率≥90%（或遗留问题有明确修复计划）
- ✅ 核心业务流程全量闭环
- ✅ 智能跑步计划功能全量闭环
- ✅ 符合需求验收标准
- ✅ 测试报告已输出并评审通过
