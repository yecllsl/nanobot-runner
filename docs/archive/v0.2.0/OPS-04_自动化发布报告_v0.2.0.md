### 4.1 流水线触发机制

**触发条件**:
```yaml
on:
  push:
    branches: [ main, develop ]  # ✅ main分支推送触发
  pull_request:
    branches: [ main ]           # ✅ PR到main分支触发
```

**发布作业条件**:
```yaml
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```

### 4.2 流水线作业流程

**4个作业流程**:
1. **code-quality**: 代码质量检查（black, isort, mypy, bandit）
2. **test**: 测试执行（单元测试、集成测试、E2E测试）
3. **build**: 构建打包（Python包构建）
4. **release**: 发布到GitHub Releases

**依赖关系**:
```
code-quality → test → build → release
```
