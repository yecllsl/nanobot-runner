## 二、优化项明细

### 2.1 SessionRepository 重构 [P0]

**问题**: SessionRepository 存在过早 `collect()` 破坏 LazyFrame 链、严重代码重复、`storage` 缺少类型注解、返回 `List[Dict[str, Any]]` 类型不安全

**优化措施**:

1. **LazyFrame 链式构建**: 新增 `_build_session_lazy()` 方法，所有过滤和排序操作在 LazyFrame 上执行，仅在最终输出时 `collect()`
2. **消除代码重复**: 提取 `_add_computed_columns()` 方法，使用 Polars 表达式批量添加 `distance_km`/`duration_min`/`avg_pace_sec_km` 计算列，替代 `get_recent_sessions`/`get_sessions_by_date_range`/`get_sessions_by_distance` 中的重复循环
3. **类型注解**: `__init__` 参数添加 `storage: StorageManager` 类型注解
4. **Dataclass 返回类型**: 新增 `SessionSummary`/`SessionDetail`/`SessionVdot` 三个 frozen dataclass，替代 `List[Dict[str, Any]]`，提供 `to_dict()` 方法兼容现有调用
5. **空数据防护**: `_build_session_lazy()` 开头检查空 LazyFrame，避免 `group_by` 在空列上报错

**涉及文件**:
- `src/core/session_repository.py` — 核心重构
- `tests/unit/core/test_session_repository.py` — 新增 55 个单元测试用例

### 2.2 AnalyticsEngine 清理 [P0]

**问题**: 存在未使用的 `_calculate_ewma` 方法和 `math` 导入；`get_vdot_trend` 使用逐行 `iter_rows` 计算 VDOT

**优化措施**:

1. **删除死代码**: 移除 `_calculate_ewma` 方法和 `math` 导入
2. **向量化 VDOT 趋势计算**: `get_vdot_trend` 改用 `vdot_calculator.calculate_vdot_batch()` 批量计算，替代逐行循环
3. **列名自适应**: 新增 `_resolve_col()` 辅助函数，按优先级自动检测 DataFrame 中的距离/时长列名（`session_total_distance` > `total_distance` > `distance`），解决不同数据源列名不一致问题

**涉及文件**:
- `src/core/analytics.py` — 核心优化

### 2.3 agent.py 异常处理 [P1]

**问题**: 编辑器检测循环中使用裸 `except Exception` 捕获所有异常

**优化措施**: 将 `except Exception` 改为 `except (FileNotFoundError, OSError)`，精确捕获命令不存在和执行失败的异常

**涉及文件**:
- `src/cli/commands/agent.py`

### 2.4 config.py assert 修复 [P1]

**问题**: 使用 `assert ConfigManager._cache is not None` 做运行时校验，生产环境 `-O` 优化时 assert 会被跳过

**优化措施**: 改为 `if ConfigManager._cache is None: raise RuntimeError(...)` 模式，确保运行时校验始终生效

**涉及文件**:
- `src/core/config.py`

## 五、关键经验

1. **LazyFrame 链式构建**: Polars 的 LazyFrame 应保持延迟求值直到最终输出，过早 `collect()` 会破坏查询优化链
2. **Dataclass 替代 Dict**: 使用 frozen dataclass 替代 `Dict[str, Any]` 可获得类型安全和 IDE 补全，同时 `to_dict()` 方法保证向后兼容
3. **列名自适应**: 数据源列名不一致时，使用优先级候选列表自动检测，比硬编码列名更健壮
4. **assert vs if/raise**: `assert` 仅用于开发期不变量检查，运行时校验必须使用 `if/raise` 模式
