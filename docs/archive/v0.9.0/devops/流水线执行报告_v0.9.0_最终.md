## 🔍 问题发现与分析

### 问题1: CLI测试用例Mock路径错误

**发现时间**: 第一次CI流水线执行
**影响范围**: 5个单元测试失败
**根本原因**: v0.9.0架构重构后，CLI命令结构从单层改为分层结构（data/analysis/agent/gateway/system），但测试用例未同步更新

**失败测试用例**:
1. `test_stats_invalid_date_format` - Mock路径错误
2. `test_stats_with_data` - 命令路径错误
3. `test_vdot_command` - Mock路径错误
4. `test_vdot_with_output` - Mock路径错误
5. `test_vdot_invalid_limit` - 命令路径错误

**错误详情**:
```python
# 错误示例
AttributeError: module 'src.cli.handlers.data_handler' has no attribute 'StorageManager'

# 原因分析
DataHandler类通过依赖注入获取StorageManager实例（context.storage），
而非直接导入StorageManager类，导致Mock路径错误
```

### 问题2: E2E测试依赖注入错误

**发现时间**: 第二次CI流水线执行
**影响范围**: 3个E2E测试失败
**根本原因**: v0.9.0引入依赖注入架构后，RunnerTools构造函数不再接受StorageManager参数，但E2E测试工具类未更新

**失败测试用例**:
1. `test_complete_user_journey` - CLI命令路径错误
2. `test_daily_training_query_flow` - RunnerTools初始化错误
3. `test_fitness_assessment_flow` - RunnerTools初始化错误

**错误详情**:
```python
# 错误代码
tools = RunnerTools(storage)  # ❌ 错误：传递了StorageManager实例

# 正确代码
tools = RunnerTools()  # ✅ 正确：使用依赖注入自动获取
```
