## 二、开发交付物

### 2.1 核心模块

#### 数据模型 (`src/core/models.py`)

- `IntentResult`: 意图解析结果
- `UserPreferences`: 用户偏好
- `TrainingLoad`: 训练负荷
- `UserContext`: 用户上下文
- `DailyPlan`: 日计划
- `WeeklySchedule`: 周计划
- `TrainingPlan`: 训练计划
- `Violation`: 违规项
- `ValidationResult`: 校验结果
- `DimensionResult`: 维度分析结果
- `AnalysisReport`: 分析报告

#### 意图解析器 (`src/core/plan/intent_parser.py`)

**核心功能**:
- 支持自然语言输入解析
- 支持斜杠命令解析
- 提取训练意图和参数
- 支持多种意图类型（创建计划、查询计划、修改计划等）

**关键方法**:
- `parse(user_input: str) -> IntentResult`: 解析用户输入

#### 训练计划生成器 (`src/core/plan/plan_generator.py`)

**核心功能**:
- 基于用户画像和历史数据生成个性化训练计划
- 构建LLM提示词模板
- 解析LLM响应为结构化数据
- 支持重试机制（最多3次）

**关键方法**:
- `generate(user_context, goal_distance_km, goal_date, ...) -> TrainingPlan`: 生成训练计划
- `_build_prompt(...) -> str`: 构建提示词
- `_parse_llm_response(...) -> TrainingPlan`: 解析LLM响应

#### 硬性规则校验器 (`src/core/plan/hard_validator.py`)

**核心功能**:
- 实现6条硬性校验规则
- 周跑量增长限制（≤10%）
- 休息日要求（每周至少1天）
- 长距离跑比例限制（≤30%）
- 高强度训练比例限制（≤20%）
- 单次跑步距离限制（≤目标距离）
- 减量周跑量减少（≥20%）

**关键方法**:
- `validate(plan, current_weekly_distance_km, goal_distance_km) -> ValidationResult`: 校验训练计划

#### 计划分析器 (`src/core/plan/plan_analyzer.py`)

**核心功能**:
- 从4个维度分析计划合理性
  - 体能匹配度
  - 负荷递进合理性
  - 伤病风险评估
  - 目标可达性
- 生成改进建议和风险警告
- 包含医疗免责声明

**关键方法**:
- `analyze(plan, user_context) -> AnalysisReport`: 分析训练计划
