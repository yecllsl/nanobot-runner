# 版本文档归档

本目录用于归档各版本的项目文档索引。**注意：实际的归档文件以.zip格式保存在本地，不提交到Git仓库。**

---

## 📁 归档策略

### Git管理策略

- ✅ **Git历史记录**：所有文档的历史版本已通过Git版本控制记录
- ✅ **本地备份**：归档文件以.zip格式保存在本地，不提交到Git
- ✅ **.gitignore**：已配置排除.zip文件，避免仓库体积膨胀

### 查看历史版本

```bash
# 查看文档历史
git log --oneline docs/

# 查看特定版本的文档
git show v0.9.0:docs/

# 恢复特定版本的文档
git checkout v0.9.0 -- docs/
```

## 🎯 归档规则

### 归档时机

- 版本发布成功后立即归档（OPS-05 文档归档）
- 在发布准出验证通过后执行

### 归档内容

每个版本的归档目录应包含以下子目录：

#### 1. test/ - 测试相关文档

- `测试报告_*.md` - 测试执行报告
- `Bug清单_*.md` - Bug列表和修复状态
- `回归报告_*.md` - 回归测试报告
- `上线结论_*.md` - 上线决策文档
- `E2E测试报告_*.md` - E2E测试报告
- `测试策略_*.md` - 测试策略文档

#### 2. devops/ - 运维相关文档

- `流水线执行报告_*.md` - CI/CD流水线验证报告
- `发布报告_*.md` - 版本发布报告
- `故障排查报告_*.md` - 故障排查文档

#### 3. development/ - 开发相关文档

- `交付报告_*.md` - 开发交付报告
- `Bug修复报告_*.md` - Bug修复报告
- `代码优化报告_*.md` - 代码优化报告
- `文档评估报告_*.md` - 文档评估报告

#### 4. review/ - 评审相关文档

- `代码评审报告_*.md` - 代码评审报告
- `架构评审报告_*.md` - 架构评审报告

#### 5. planning/ - 规划相关文档

- `task_list_*.md` - 任务清单
- `规划文档_*.md` - 规划文档

#### 6. architecture/ - 架构相关文档

- `架构设计_*.md` - 架构设计文档
- `重构规划方案_*.md` - 重构规划方案
- `功能分析_*.md` - 功能分析文档
- `项目深度设计分析报告_*.md` - 项目深度设计分析报告

### 清理原则

- 归档后删除项目目录中的版本文档
- 保留通用文档（如规范、指南等）
- 保留当前版本开发所需的文档
