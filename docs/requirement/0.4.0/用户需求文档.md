## 1. 项目定位

### 1.1 产品愿景

Nanobot Runner 是一款运行于个人电脑本地环境的**深度数据分析与智能训练规划中枢**。v0.4.0 版本聚焦于**智能个性化**与**飞书深度集成**，通过用户画像系统实现真正个性化的训练建议，并通过飞书平台实现日程级别的训练管理。

### 1.2 核心价值主张

| 价值维度 | v0.3.1 基线 | v0.4.0 增强 |
|---------|------------|-------------|
| **数据隐私** | 本地存储，零外联设计 | 保持不变 |
| **深度分析** | 企业级 BI 能力 | 增加用户画像与预测 |
| **智能交互** | 自然语言交互 | 画像感知的个性化对话 |
| **多端协同** | CLI + 飞书推送 | 飞书双向交互 + 日历同步 |
| **个性化** | 基础统计 | 多维度用户画像驱动 |

### 1.3 版本目标

- **构建用户画像系统**：基于历史数据构建多维度跑步画像，实现个性化训练建议
- **训练计划智能化**：根据用户画像和目标自动生成周期化训练计划
- **飞书深度集成**：训练计划同步到飞书日历，实现双向交互
- **增强用户体验**：智能训练回顾、比赛预测

## 2. 目标用户画像

### 2.1 主要用户群体

#### 👨‍💻 技术极客/开发者

- **特征**: 习惯终端操作，追求数据的颗粒度与脚本化控制能力
- **痛点**: 商业跑步 App 数据封闭，无法自定义分析
- **期望**: 命令行工具、API 接口、可编程的数据处理
- **v0.4.0 关注**: 用户画像 API、训练计划可编程接口

#### 🏃 硬核跑者

- **特征**: 关注 TSS、VDOT 等进阶指标，有明确比赛目标
- **痛点**: 需要专业分析工具，缺乏个性化训练计划
- **期望**: 深度数据分析、训练负荷管理、成绩预测、训练计划
- **v0.4.0 关注**: 个性化训练计划、比赛预测、飞书日历同步

#### 🔒 隐私主义者

- **特征**: 拒绝将身体数据上传至商业云服务的用户
- **痛点**: 担心运动数据被商业利用或泄露
- **期望**: 完全本地化的数据处理与存储
- **v0.4.0 关注**: 画像数据本地存储、可选的飞书推送

#### 📱 飞书用户

- **特征**: 日常使用飞书进行工作协作的用户
- **痛点**: 训练计划与工作日程分离，管理不便
- **期望**: 训练计划与工作日程统一管理
- **v0.4.0 关注**: 飞书日历同步、机器人交互

## 3. 功能需求

### 3.1 P0 级功能（核心差异化）

#### 3.1.1 智能跑步画像系统 (FR-001)

**需求描述**:
基于用户历史训练数据构建多维度画像，实现真正个性化的训练建议。

**功能规格**:

**FR-001-1 画像维度定义**

| 维度 | 数据来源 | 计算方法 | 更新频率 |
|------|---------|---------|---------|
| 跑力水平 | VDOT趋势、最佳成绩 | 历史VDOT加权平均 | 每次训练后 |
| 训练习惯 | 训练频率、时段偏好、距离分布 | 统计分析 | 每周 |
| 恢复能力 | TSB趋势、心率漂移恢复率 | EWMA计算 | 每次训练后 |
| 目标偏好 | 用户配置的比赛目标 | 用户输入 | 手动更新 |
| 伤病风险 | 训练负荷变化率、心率异常 | 风险模型 | 每次训练后 |

**FR-001-2 画像数据结构**

```python
@dataclass
class RunnerProfile:
    user_id: str
    profile_version: str
    last_updated: datetime
    
    fitness_level: FitnessLevel
    training_pattern: TrainingPattern
    recovery_capacity: RecoveryCapacity
    goal_preference: GoalPreference
    injury_risk: InjuryRisk
    
    summary: ProfileSummary

@dataclass
class FitnessLevel:
    current_vdot: float
    vdot_trend: str
    best_vdot: float
    vdot_history: List[float]
    fitness_score: int
    
@dataclass
class TrainingPattern:
    weekly_frequency: float
    preferred_time: str
    avg_distance_km: float
    distance_distribution: Dict[str, float]
    consistency_score: int
```

**FR-001-3 画像查询接口**

用户可通过自然语言查询画像信息：
- "我的跑步能力怎么样？"
- "我的训练习惯有什么特点？"
- "我的恢复能力如何？"
- "我现在的伤病风险高吗？"

**验收标准**:
- ✅ 画像数据准确反映用户实际能力
- ✅ 画像更新响应时间 < 2 秒
- ✅ 支持画像数据导出
- ✅ 画像数据本地存储，不上传云端

#### 3.1.2 个性化训练计划生成 (FR-002)

**需求描述**:
根据用户画像和目标，自动生成周期化训练计划，支持动态调整。

**功能规格**:

**FR-002-1 训练计划结构**

```python
@dataclass
class TrainingPlan:
    plan_id: str
    goal: str                    # 目标（如"全马400"）
    target_race: Optional[str]   # 目标比赛
    race_date: Optional[date]    # 比赛日期
    duration_weeks: int          # 周期长度
    start_date: date             # 开始日期
    current_phase: str           # 当前阶段
    current_week: int            # 当前周次
    
    weekly_schedule: List[WeeklySchedule]
    target_vdot: float           # 目标VDOT
    estimated_race_time: str     # 预计完赛时间
    
    created_at: datetime
    updated_at: datetime

@dataclass
class WeeklySchedule:
    week_number: int
    phase: str                   # 基础期/强化期/巅峰期/减量期
    total_distance_km: float
    total_duration_hours: float
    daily_plans: List[DailyPlan]

@dataclass
class DailyPlan:
    day_of_week: str
    workout_type: str            # 轻松跑/节奏跑/间歇跑/长距离跑/休息
    distance_km: Optional[float]
    duration_min: Optional[int]
    intensity: str               # 低/中/高
    target_hr_zone: Optional[str]
    description: str
    completed: bool
    actual_data: Optional[Dict]
```

**FR-002-2 训练计划生成流程**

```
用户输入目标（如：全马破4小时）
        ↓
画像系统分析当前能力
        ↓
计算能力差距与所需周期
        ↓
生成周期化训练计划（12-16周）
        ↓
分解为周计划和日计划
        ↓
动态调整（根据训练完成情况）
```

**FR-002-3 训练阶段划分**

| 阶段 | 周期占比 | 主要目标 | 典型训练 |
|------|---------|---------|---------|
| 基础期 | 40% | 建立有氧基础 | 轻松跑、长距离跑 |
| 强化期 | 30% | 提升乳酸阈值 | 节奏跑、间歇跑 |
| 巅峰期 | 20% | 达到最佳状态 | 比赛配速训练 |
| 减量期 | 10% | 恢复与调整 | 轻松跑、休息 |

**FR-002-4 动态调整机制**

- 根据训练完成率自动调整后续计划
- 根据体能状态（TSB）调整训练强度
- 支持手动调整单个训练日
- 支持重新生成计划

**验收标准**:
- ✅ 训练计划符合运动科学原理
- ✅ 计划生成时间 < 5 秒
- ✅ 支持导出为 JSON/CSV 格式
- ✅ 支持与飞书日历同步

## 5. 用户场景

### 5.1 场景一：新手用户首次使用

**用户**: 刚开始跑步的新手，想了解自己的能力水平

**场景流程**:
1. 用户导入首批 FIT 文件（约 10 条记录）
2. 系统自动生成初始用户画像
3. 用户询问："我的跑步能力怎么样？"
4. Agent 返回画像分析，包括 VDOT、训练建议
5. 用户设置目标："我想在半年后完成首个半马"
6. 系统生成 16 周训练计划
7. 训练计划同步到飞书日历

**用户价值**:
- 快速了解自身能力
- 获得科学的训练计划
- 训练与日程统一管理

### 5.2 场景二：进阶跑者备战比赛

**用户**: 有一定跑步基础，准备参加全马比赛

**场景流程**:
1. 用户查看当前画像："我的VDOT是多少？"
2. 用户询问预测："我能破4吗？"
3. 系统返回预测结果和训练建议
4. 用户创建训练计划："给我制定一个全马400的训练计划"
5. 系统生成 16 周周期化训练计划
6. 计划自动同步到飞书日历
7. 每日飞书推送训练提醒
8. 每周自动生成训练回顾报告
9. 根据训练完成情况动态调整计划

**用户价值**:
- 科学评估比赛目标可行性
- 获得个性化训练计划
- 实时监控训练状态
- 自动调整训练节奏

### 5.3 场景三：飞书用户日常训练

**用户**: 日常使用飞书办公，希望通过飞书管理训练

**场景流程**:
1. 用户在飞书收到每日晨报推送
2. 用户在飞书私聊机器人："今天该练什么？"
3. 机器人返回今日训练计划详情
4. 用户完成训练后，在飞书记录："今天跑了10公里，感觉不错"
5. 系统更新画像和训练计划完成状态
6. 周末收到周报推送，了解本周训练情况

**用户价值**:
- 无需切换应用管理训练
- 训练与工作日程统一
- 及时获取训练提醒
- 便捷记录训练感受
