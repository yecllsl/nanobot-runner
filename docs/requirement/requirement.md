# 项目名称：Nanobot Runner – 桌面端私人AI跑步助理
## 1. 项目定位
一款运行于个人电脑本地环境的**深度数据分析与智能训练规划中枢**。本项目基于**nanobot-ai**底座构建，充分利用其本地推理能力与Agent扩展性，为跑者提供企业级的数据洞察深度，同时确保所有隐私数据不出本地。通过**CLI（命令行）**与**飞书**双端协同，打造技术型跑者的专属“数字教练”。
## 2. 目标用户
*   **技术极客/开发者**：习惯终端操作，追求数据的颗粒度与脚本化控制能力。
*   **硬核跑者**：关注TSS（训练压力分数）、VDOT（跑力值）等进阶指标，依赖Garmin等设备导出的.fit文件进行深度分析。
*   **隐私主义者**：拒绝将身体数据上传至商业云服务的用户。
## 3. 核心功能需求
### 3.1 数据导入与管理模块（核心基础）
本模块是数据分析的基石，需确保数据接入的高效性与准确性。
*   **格式支持**：
    *   **首选格式**：`.fit`（Flexible and Interoperable Data Transfer）。这是Garmin、Wahoo等主流运动设备的标准格式，包含丰富的心率、步频、功率及轨迹元数据。
    *   系统需底层解析.fit文件字段，无需依赖外部API。
*   **导入模式**：
    *   **单文件导入**：`nanobotrun import /path/to/run.fit`
    *   **目录批量导入**：`nanobotrun import /path/to/activities/`。系统自动扫描目录下所有`.fit`文件并尝试导入。
*   **智能去重机制（幂等性设计）**：
    *   **指纹校验**：解析.fit文件头部的`Serial Number`（设备序列号）、`Time Created`（创建时间戳）及`Total Distance`（总距离），生成唯一活动指纹。
    *   **增量更新**：执行导入命令时，系统自动比对本地数据库指纹。
        *   若指纹已存在：跳过导入，并在CLI输出提示（如：`[跳过] 文件已存在: 2023-10-01-morning.fit`）。
        *   若新文件：执行解析入库，输出（如：`[新增] 导入成功: 2023-10-02-evening.fit`）。
    *   这确保了用户可以放心多次执行目录导入命令，而不会造成数据污染。
### 3.2 CLI交互模块（遵循nanobot-ai交互规范）
CLI作为核心控制台，遵循 `nanobotrun [action] [params]` 的命令结构。
*   **数据管理指令集**：
    *   `nanobotrun import <path>`：智能导入单文件或目录。
    *   `nanobotrun db:stats`：查看本地库状态（总记录数、时间跨度、重复检测记录数）。
*   **自然语言交互模式**：
    *   执行 `nanobotrun` 进入交互式终端，直接输入自然语言，底座Agent自动调用工具。
    *   示例：
        *   用户输入：`分析我上周心率漂移最严重的一次跑步`
        *   Agent行为：检索数据库 -> 计算心率漂移率 -> 输出分析报告。
*   **报告生成**：
    *   `nanobotrun report --type pdf/html`：调用本地渲染引擎，生成可视化分析报告。
### 3.3 飞书交互模块
作为CLI的延伸，侧重于消息触达与轻量级查询。
*   **每日晨报推送**：Bot每日定时推送“今日训练课表”与“身体疲劳度评估”。
*   **IM对话式查询**：用户在飞书发送“我现在的全马PB预测是多少？”，Bot返回本地计算的VDOT值及预测成绩。
*   **导入状态通知**：当CLI端完成大批量导入后，飞书可收到汇总通知（如：“本次共扫描100个文件，新增导入80条，跳过重复20条”）。
### 3.4 深度数据分析引擎
利用nanobot-ai的本地算力，基于.fit文件的高精度数据进行分析。
*   **体能指标体系构建**：
    *   **TSS/ATL/CTL计算**：基于.fit文件中的心率区间与记录频率，精确计算实时体能状况（疲劳、fitness、状态）。
    *   **VDOT跑力值追踪**：根据每一条记录的距离与时间，动态计算当前VDOT。
*   **跑步动力学分析**：
    *   **原生数据挖掘**：直接读取.fit文件中的高阶数据（如步幅、垂直振幅、左右触地平衡，视设备支持情况而定）。
    *   **心率策略复盘**：分析实时心率曲线与配速曲线的匹配度，识别“心率漂移”拐点。
### 3.5 后续版本：语音交互模块
*   **本地语音控制**：支持在CLI环境下语音输入指令。
*   **语音播报反馈**：支持朗读核心结论。
## 4. 数据隐私与安全架构
*   **本地沙盒运行**：nanobot-ai底座与所有.fit原始文件、解析后的数据严格限制在用户本地文件夹内。
*   **零上传原则**：除非用户主动配置飞书Webhook发送摘要消息，否则默认不发起任何外部网络请求。原始.fit文件严禁上传。
## 5. 用户使用流程示例
**场景：新设备数据迁移与初始化**
1.  **批量导入**：
    用户将从Garmin手表导出的历年数据文件夹（内含500个.fit文件）拖入本地目录。
    在终端执行：
    ```bash
    nanobotrun import ./garmin_activities/
    ```
    系统输出：
    ```text
    [扫描中] 发现 500 个 .fit 文件...
    [处理中] 正在校验文件指纹...
    [新增] 2020-05-01-Run.fit
    [新增] 2020-05-03-Run.fit
    ...
    [跳过] 2023-10-01-Run.fit (重复记录)
    --------------------------------------------------
    导入完成: 新增 480 条记录, 跳过 20 条重复记录.
    正在重建体能模型索引...
    ```
2.  **数据验证**：
    用户输入自然语言：
    ```
    我最早的一笔跑步数据是什么时候？
    ```
    Agent回答：`根据本地数据库记录，您最早的跑步数据是2020年5月1日。`
## 6. 总结
本项目通过引入**智能.fit文件解析引擎**，解决了本地数据管理的痛点。通过文件指纹去重机制，保证了数据仓库的清洁与高效。配合 `nanobotrun` 命令体系，实现了从数据采集、清洗到深度分析的完整闭环，满足了技术型跑者对数据资产全生命周期的掌控需求。
