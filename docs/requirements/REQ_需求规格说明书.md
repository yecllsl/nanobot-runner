## 1. 项目概述

### 1.1 项目背景

Nanobot Runner 是一款桌面端私人 AI 跑步助理，基于 nanobot-ai 框架构建，旨在帮助跑步爱好者管理和分析运动数据。

**产品愿景**: 成为技术型跑者首选的本地化 AI 跑步数据分析工具

### 1.2 项目目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| 提供本地化的跑步数据管理方案 | 数据完全存储在本地，用户可控 | P0 |
| 实现智能化的运动数据分析 | VDOT、TSS、心率漂移等专业指标 | P0 |
| 构建自然语言交互的 AI 助理 | 自然语言查询和智能建议 | P0 |
| 保护用户数据隐私 | 零外联设计，不收集用户隐私 | P0 |

### 1.3 目标用户

#### 主要用户：技术型严肃跑者

| 属性 | 描述 |
|------|------|
| **年龄段** | 25-45 岁 |
| **职业背景** | 程序员、工程师、数据分析师等技术从业者 |
| **跑步经验** | 规律跑步 2 年以上，有马拉松参赛经历 |
| **数据意识** | 习惯佩戴心率带、功率计等专业设备 |
| **核心痛点** | 担心云端数据隐私，需要更专业的数据分析 |
| **技术能力** | 熟悉命令行操作，具备一定编程基础 |

#### 次要用户：跑步数据分析师

| 属性 | 描述 |
|------|------|
| **使用场景** | 分析运动员训练数据，制定训练计划 |
| **核心需求** | 批量数据处理、自定义分析脚本、报告生成 |
| **技术能力** | Python 数据分析能力，熟悉 Pandas/Polars |

## 2. 功能需求

### 2.1 数据导入功能

#### 2.1.1 FIT 文件解析

**需求描述**: 支持解析标准 FIT 格式的运动数据文件

**功能要点**:
- 解析 Garmin、Coros 等设备导出的 FIT 文件
- 提取运动数据（距离、时长、心率、配速、步频、功率等）
- 支持批量导入（单个文件、目录递归）
- 解析错误处理和日志记录

**验收标准**:
- [ ] AC-01: 成功解析标准 FIT 文件，数据提取完整准确
- [ ] AC-02: 导入速度 < 1 秒/文件
- [ ] AC-03: 支持批量导入，100 个文件批量导入成功率 100%
- [ ] AC-04: 解析错误时提供清晰的错误信息

#### 2.1.2 数据去重

**需求描述**: 避免重复导入相同运动记录

**功能要点**:
- 基于 SHA256 指纹去重
- 支持强制重新导入（`--force` 参数）
- 记录导入历史（时间、文件路径、指纹）
- 去重冲突提示和跳过机制

**验收标准**:
- [ ] AC-01: 去重准确率 100%
- [ ] AC-02: 指纹计算速度 < 100ms/文件
- [ ] AC-03: 重复文件导入时提示"已存在，跳过"
- [ ] AC-04: `--force` 参数可强制重新导入

### 2.3 数据分析功能

#### 2.3.1 VDOT 计算

**需求描述**: 计算跑步能力值（VDOT）

**功能要点**:
- 基于 Powers 公式计算
- 距离要求 >= 1500m
- 提供趋势分析（图表/表格）
- 支持历史 VDOT 查询

**验收标准**:
- [ ] AC-01: 计算结果误差 < 1%
- [ ] AC-02: 支持历史趋势查询
- [ ] AC-03: 距离 < 1500m 时提示"距离不足，无法计算 VDOT"
- [ ] AC-04: 提供月度/季度 VDOT 趋势图

#### 2.3.2 训练负荷分析

**需求描述**: 计算训练压力和体能状态

**功能要点**:
- TSS（训练压力分数）计算：时长×IF²×100
- ATL（急性训练负荷）计算：7 天 EWMA
- CTL（慢性训练负荷）计算：42 天 EWMA
- TSB（训练压力平衡）计算：CTL - ATL
- 支持自定义时间窗口

**验收标准**:
- [ ] AC-01: 计算结果符合运动科学标准
- [ ] AC-02: 支持自定义时间窗口（7/14/28/42 天）
- [ ] AC-03: 提供 ATL/CTL/TSB 趋势图
- [ ] AC-04: TSB 负值过大时提示"过度训练风险"

#### 2.3.3 心率漂移分析

**需求描述**: 分析运动过程中心率变化趋势

**功能要点**:
- 计算心率与时间的相关性
- 判断心率漂移程度（相关性<-0.7 判定为漂移）
- 提供有氧能力评估
- 漂移判断阈值可配置

**验收标准**:
- [ ] AC-01: 相关性计算准确，误差 < 5%
- [ ] AC-02: 漂移判断阈值可配置（默认 -0.7）
- [ ] AC-03: 漂移时提示"有氧能力不足，建议降低配速"
- [ ] AC-04: 提供心率区间分布图

#### 2.3.4 用户画像管理（v0.9 新增）

**需求描述**: 管理用户个人数据和训练目标

**功能要点**:
- 个人基本信息（年龄、性别、体重、静息心率）
- 训练目标（5K/10K/半马/全马目标成绩）
- 心率区间配置（最大心率法/储备心率法）
- 训练偏好（训练频率、偏好时段）

**验收标准**:
- [ ] AC-01: 支持用户信息 CRUD 操作

| tsb | float | 训练压力平衡 | 计算 |
| hr_drift | float | 心率漂移相关性 | 计算 |

### 6.2 数据量估算

| 数据类型 | 年增长量 | 5 年增长量 |
|---------|---------|-----------|
| 运动记录 | ~500 条/年 | ~2500 条 |
| 存储空间 | ~50MB/年 | ~250MB |
| 索引文件 | ~5MB/年 | ~25MB |

### 6.3 数据生命周期

- **活跃数据**: 最近 1 年，高频访问
- **历史数据**: 1-3 年，低频访问
- **归档数据**: 3 年以上，支持查询

#### 7.1.1 v0.9.5 版本详细需求

**版本目标**：架构解耦与初始化流程整合

**核心策略**：全面使用nanobot-ai提供的所有模块能力，不重复造轮子，仅替换配置管理层。

1. **新用户**：首次安装nanobot-runner，运行`nanobotrun system init`完成初始化，无需关心nanobot-ai的存在。
2. **已使用nanobot用户**：已有nanobot配置，运行`nanobotrun system init --mode migrate`迁移配置，原有nanobot配置保留。
3. **仅使用数据功能用户**：不使用Agent/Gateway功能，无需配置LLM，数据功能正常使用。
