## 1. 开发概览

### 1.1 功能目标

实现 **数字孪生引擎**，让跑者"在训练前看到训练后的自己"。核心能力：

- **5维跑者状态向量**：体能/负荷/身体信号/风险/训练模式
- **What-If 推演**：基于当前状态模拟训练计划执行后的变化
- **多计划对比**：对多个训练方案执行推演并推荐最优方案

### 1.2 架构方案

采用 **薄编排层方案**，聚合现有模块（PredictionEngine / BodySignalEngine / TrainingLoadAnalyzer / SessionRepository），不引入新依赖。

```
DigitalTwinEngine (编排层)
├── StateVectorBuilder (5维聚合)
│   ├── FitnessDimension ← PredictionEngine
│   ├── LoadDimension ← TrainingLoadAnalyzer
│   ├── BodySignalDimension ← BodySignalEngine
│   ├── RiskDimension ← PredictionEngine + TrainingLoadAnalyzer
│   └── TrainingPatternDimension ← SessionRepository
└── WhatIfSimulator (逐周推演)
    ├── 周TSS估算 (强度分布 × IF² × 100)
    ├── CTL/ATL逐周更新 (EWMA)
    ├── VDOT增量估算
    └── 置信度指数衰减
```

## 5. CLI 使用方式

```bash
# 获取5维状态快照
uv run nanobotrun twin snapshot

# What-If推演（通过JSON指定训练计划）
uv run nanobotrun twin simulate --plan-name "备赛计划" --weeks '[{"weekly_volume_km": 50, "easy_ratio": 0.7, "tempo_ratio": 0.15, "interval_ratio": 0.15, "long_run_km": 15, "intensity_multiplier": 1.0}]'

# 多计划对比
uv run nanobotrun twin compare --plans '[{"name": "保守计划", "weeks": [...]}, {"name": "激进计划", "weeks": [...]}]'
```

---

## 8. 已知限制

1. **ML增强模式**：`prediction_type="ml_enhanced"` 依赖已训练的ML模型，若模型未就绪会自动降级为parametric模式
2. **VDOT增量估算**：当前采用简化线性模型（每周0.3-0.8 VDOT），后续可接入更精确的Banister模型
3. **伤病风险推演**：基于ACWR阈值判断，未考虑外部因素（天气、路面等）
4. **置信度衰减**：采用固定指数衰减因子0.95，长期推演（>8周）置信度较低
