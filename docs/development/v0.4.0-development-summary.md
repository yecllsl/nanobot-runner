## 六、关键设计决策

### 6.1 双存储机制
- **决策**: Profile 数据同时存储在 profile.json 和 MEMORY.md
- **原因**: 
  - profile.json：结构化数据，便于程序读取
  - MEMORY.md：Agent 可读格式，保留 Agent 注释
- **效果**: 实现了程序可用性和 Agent 可读性的平衡

### 6.2 VDOT 公式选择
- **决策**: 采用 Jack Daniels VDOT 公式
- **原因**: 
  - 业界公认的跑步能力评估标准
  - 科学准确的配速预测
  - 与现有 AnalyticsEngine 兼容
- **效果**: 提供了可靠的比赛预测能力

### 6.3 冲突解决策略
- **决策**: 实现 4 种冲突解决策略
- **原因**: 
  - 不同场景需要不同的冲突处理方式
  - 用户应该有选择权
- **效果**: 提供了灵活的日历同步能力
