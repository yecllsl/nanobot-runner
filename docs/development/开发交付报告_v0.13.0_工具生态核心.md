## 启动方式

本模块为库模块，无需独立启动。通过以下方式集成：

```python
from src.core.tools.tool_manager import ToolManager
from pathlib import Path

manager = ToolManager(Path("config.json"))
tools = manager.list_tools()
```

## 已知问题

1. `_extract_server_name` 依赖已知服务器名称匹配，对于配置中不存在的服务器名称无法解析
2. `import_claude_desktop_config` 仅支持Claude Desktop的标准配置格式，非标准格式可能导入失败

## 关键设计决策

1. **不可变数据类**: ToolInfo/MCPServerConfig 使用 `frozen=True`，确保配置数据安全
2. **配置管理 vs 工具调用分离**: ToolManager 仅负责配置管理，工具调用由 nanobot SDK 自动处理
3. **服务器名称匹配**: 工具名称解析采用基于已知服务器名称的前缀匹配，避免下划线歧义问题
