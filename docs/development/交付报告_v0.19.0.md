## 5. 已知问题

1. **HRV估算精度**: 当前HRV估算基于心率数据推导，精度有限。如设备支持RR间期数据，精度可提升。
2. **休息日效果判定**: 当前基于静息心率和TSB变化，未来可结合睡眠质量等多维度数据。
3. **疲劳度评分**: 主观RPE为可选输入，无RPE时权重重分配可能影响评分敏感度。

## 6. 使用示例

### CLI命令
```bash
# 每日身体信号摘要
uv run nanobotrun status daily

# 每周身体信号摘要
uv run nanobotrun status weekly

# 异常预警
uv run nanobotrun status alerts

# HRV分析
uv run nanobotrun analysis hrv --days 30

# 疲劳度评估（带RPE）
v run nanobotrun analysis fatigue --rpe 7

# 恢复状态
uv run nanobotrun analysis recovery --days 7
```

### Agent工具
- `get_hrv_analysis` - 获取HRV分析
- `get_fatigue_assessment` - 获取疲劳度评估
- `get_recovery_status` - 获取恢复状态
- `get_body_signal_alerts` - 获取身体信号预警
