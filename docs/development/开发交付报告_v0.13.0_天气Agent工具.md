## 5. 架构说明

### 5.1 天气Agent工具接入流程

```
config.json (tools.mcp_servers.weather)
       ↓
init向导自动配置天气MCP服务器
       ↓
Nanobot.from_config() 自动加载MCP配置
       ↓
connect_mcp_servers() 连接天气MCP服务器
       ↓
agent.tools.register() 注册工具到AgentLoop
       ↓
Agent可通过自然语言调用天气工具
```

### 5.2 关键设计决策

1. **MCP协议接入**: 天气工具通过MCP协议自动接入，无需手动编写Agent工具类
2. **自动注册**: MCP工具通过`Nanobot.from_config()`自动注册到AgentLoop
3. **命名规范**: 天气工具命名格式为`mcp_weather_{tool_name}`
4. **配置驱动**: 通过config.json管理天气MCP服务器配置
5. **优雅降级**: MCP连接失败不影响Agent启动，仅记录警告日志

### 5.3 测试覆盖范围

| 测试类型 | 覆盖内容 |
|---------|---------|
| 单元测试 | MCP配置加载、连接器功能、错误处理 |
| 集成测试 | 天气MCP服务器连接、工具注册、配置验证 |
| E2E测试 | Agent自然语言调用、工具调用成功率、错误处理 |

## 6. 已知问题与注意事项

1. **Node.js依赖**: 天气MCP服务器需要Node.js环境运行npx命令
2. **网络依赖**: 首次运行时npx需下载`@dangahagan/weather-mcp`包
3. **AsyncExitStack生命周期**: 当前MCP连接的exit_stacks未在Agent退出时显式清理，依赖进程退出自动释放
4. **测试Mock**: 所有测试使用Mock模拟MCP服务器连接，未进行真实MCP服务器连接测试
