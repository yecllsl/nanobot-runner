## 5. CLI命令使用说明

### 5.1 初始化配置

```bash
# agent_mode初始化自动包含天气MCP配置
nanobotrun init
```

### 5.2 工具管理

```bash
# 列出所有已配置工具
nanobotrun tools list

# 添加MCP服务器
nanobotrun tools add weather --command npx --args '["-y", "@dangahagan/weather-mcp"]'

# 移除MCP服务器
nanobotrun tools remove weather

# 启用/禁用服务器
nanobotrun tools enable weather
nanobotrun tools disable weather

# 验证配置
nanobotrun tools validate

# 导入Claude Desktop配置
nanobotrun tools import-claude ~/path/to/claude_desktop_config.json
```

### 5.3 Agent交互

```bash
# 启动chat模式（自动连接MCP工具）
nanobotrun agent chat
```

## 6. 架构说明

### 6.1 MCP工具接入流程

```
config.json (tools.mcp_servers)
       ↓
load_mcp_servers_config()  →  过滤disabled服务器
       ↓
connect_mcp_servers()      →  nanobot SDK连接MCP服务器
       ↓
agent.tools.register()     →  注册工具到AgentLoop
       ↓
Agent可调用MCP工具
```

### 6.2 关键设计决策

1. **延迟导入**: `connect_mcp_servers`在函数内部导入，避免nanobot SDK不可用时模块加载失败
2. **优雅降级**: MCP连接失败不影响Agent启动，仅记录警告日志
3. **配置驱动**: 通过config.json管理MCP服务器，支持运行时增删改
4. **默认配置**: agent_mode初始化自动包含天气MCP服务器配置

## 7. 已知问题与注意事项

1. **Node.js依赖**: 天气MCP服务器需要Node.js环境运行npx命令
2. **网络依赖**: 首次运行时npx需下载`@dangahagan/weather-mcp`包
3. **AsyncExitStack生命周期**: 当前MCP连接的exit_stacks未在Agent退出时显式清理，依赖进程退出自动释放
4. **Gateway同步启动**: `_connect_mcp_tools_sync`使用`asyncio.run()`，在已有事件循环时可能冲突
