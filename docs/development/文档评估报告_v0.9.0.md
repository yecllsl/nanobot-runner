## 🔍 详细评估

### 1. AGENTS.md - 项目结构过时

**问题描述**:
AGENTS.md的项目结构部分未反映v0.9.0的CLI分层重构。

**当前内容**:
```
src/
├── core/            # 核心模块: parser, storage, indexer, analytics, profile, config, exceptions...
├── agents/tools.py  # Agent 工具集: BaseTool + RunnerTools
├── notify/          # 飞书通知: feishu.py, feishu_calendar.py
├── cli.py           # CLI 入口
└── cli_formatter.py # Rich 格式化输出
```

**应更新为**:
```
src/
├── core/            # 核心模块: parser, storage, indexer, analytics, profile, config, exceptions...
│   ├── context.py   # 应用上下文 (v0.9.0新增)
│   └── session_repository.py  # Session仓储层 (v0.9.0新增)
├── agents/tools.py  # Agent 工具集: BaseTool + RunnerTools
├── notify/          # 飞书通知: feishu.py, feishu_calendar.py
├── cli/             # CLI 模块 (v0.9.0重构)
│   ├── commands/    # 命令模块: data, analysis, agent, report, system, gateway
│   ├── handlers/    # 业务逻辑调用层
│   ├── app.py       # CLI 应用入口
│   └── __main__.py  # 模块入口
└── cli_formatter.py # Rich 格式化输出
```

**影响范围**:
- 开发工程师查看项目结构时会困惑
- 新加入的开发者无法快速理解项目架构

**建议操作**:
立即更新AGENTS.md的项目结构部分，反映v0.9.0的架构变化。

### 2. AGENTS.md - 常用命令过时

**问题描述**:
AGENTS.md的常用命令部分未使用v0.9.0的CLI分层命令格式。

**当前内容**:
```bash
# 运行
uv run nanobotrun --help
uv run nanobotrun import-data <path> [--force]
uv run nanobotrun stats [--year YYYY]
uv run nanobotrun chat
```

**应更新为**:
```bash
# 运行
uv run nanobotrun --help
uv run nanobotrun data import-data <path> [--force]
uv run nanobotrun data stats [--year YYYY]
uv run nanobotrun agent chat
uv run nanobotrun analysis vdot
uv run nanobotrun analysis load
uv run nanobotrun analysis hr-drift
```

**影响范围**:
- 开发工程师使用错误的命令格式
- 测试工程师编写测试用例时使用错误的命令

**建议操作**:
立即更新AGENTS.md的常用命令部分，使用v0.9.0的CLI分层命令格式。
