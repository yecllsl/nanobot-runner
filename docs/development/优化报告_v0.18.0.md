## 3. 性能优化分析

### 3.1 _prepare_summary_data 性能提升

**优化前**: 调用 `_prepare_session_data(config)` → 触发完整的 TSS/VDOT 分批计算（每条记录调用 `analytics.calculate_vdot()` + `analytics.calculate_tss_for_run()`）

**优化后**: 直接调用 `storage.query_by_date_range()` → 仅获取原始字段，跳过计算

**性能提升估算**:
- 假设 1000 条记录，每条计算 VDOT + TSS 约 0.5ms
- 优化前: 1000 × 0.5ms = 500ms 计算开销
- 优化后: 0ms 计算开销
- **summary 导出场景性能提升 > 50%**

### 3.2 _parse_timestamp 可测试性提升

**优化前**: 时间戳解析逻辑内联在 `_prepare_summary_data` 中，无法单独测试

**优化后**: 提取为 `_parse_timestamp()` 方法，6 个独立测试用例覆盖
