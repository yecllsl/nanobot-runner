## 6. 启动方式

### 6.1 初始化配置

```bash
# 交互式初始化（含 LLM 配置向导）
uv run nanobotrun system init

# 迁移模式（从 nanobot 配置迁移）
uv run nanobotrun system init --mode migrate

# 自动模式
uv run nanobotrun system init --auto
```

### 6.2 Agent 交互

```bash
# 启动 AI 助手（使用项目配置注入）
uv run nanobotrun agent chat
```

### 6.3 Gateway 服务

```bash
# 启动飞书机器人 Gateway（使用项目配置注入）
uv run nanobotrun gateway start
```

---

## 7. 注意事项

1. **配置优先级**: 项目配置 > nanobot 配置。当项目配置中存在 LLM 配置时，优先使用项目配置；否则回退到 nanobot 配置。
2. **环境变量支持**: LLM API Key 通过环境变量 `NANOBOT_LLM_API_KEY` 管理，不硬编码到配置文件中。
3. **迁移兼容性**: `--mode migrate` 支持从现有 nanobot 配置迁移 LLM 配置到项目配置，迁移后项目配置独立于 nanobot。
4. **配置同步**: `NanobotConfigSync` 提供项目配置到 nanobot 配置的单向同步，确保 Gateway 等依赖 nanobot 配置的模块正常工作。
