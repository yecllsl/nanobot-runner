```python
def delete_year_data(self, year: int) -> bool:
    """删除指定年份的数据文件"""
    if year < 2000 or year > 2100:
        raise ValidationError(
            message="年份必须在2000-2100范围内",
            recovery_suggestion="请输入有效的年份",
        )

    filename = f"activities_{year}.parquet"
    filepath = self.data_dir / filename

    if filepath.exists():
        filepath.unlink()  # 删除文件
        logger.info(f"删除年份数据: {filename}")
        return True
    else:
        logger.debug(f"数据文件不存在，无需删除: {filename}")
        return False
```

**调用情况**:

**结论**: 此方法虽然存在，但用户无法通过任何 CLI 命令触发，不会导致数据删除。

## 💡 潜在改进建议

### 建议 1: 添加数据管理命令

**当前问题**: 用户无法删除错误导入的数据

**建议方案**:

```python
@app.command()
def data(
    action: str = typer.Argument(..., help="操作：list/delete"),
    year: Optional[int] = typer.Option(None, "--year", "-y", help="年份"),
    force: bool = typer.Option(False, "--force", "-f", help="强制删除，无需确认"),
):
    """
    管理跑步数据

    示例:
        nanobotrun data list
        nanobotrun data delete --year 2024
    """
    config = ConfigManager()
    storage = StorageManager(config.data_dir)

    if action == "list":
        years = storage.get_available_years()
        console.print("[bold]可用数据年份:[/bold]")
        for y in years:
            stats = storage.get_year_stats(y)
            console.print(f"  {y}: {stats['total_runs']} 次跑步")

    elif action == "delete":
        if not year:
            console.print("[red]请指定年份[/red]")
            raise typer.Exit(1)

        if year not in storage.get_available_years():
            console.print(f"[red]年份 {year} 的数据不存在[/red]")
            raise typer.Exit(1)

        if not force:
            from rich.prompt import Confirm
            if not Confirm.ask(f"[red]确定要删除 {year} 年的数据吗？此操作不可恢复[/red]"):
                console.print("[yellow]已取消[/yellow]")
                return

        result = storage.delete_year_data(year)
        if result:
            console.print(f"[green]✓[/green] 已删除 {year} 年数据")
        else:
            console.print(f"[yellow]数据文件不存在[/yellow]")
```

**优点**:
- ✅ 用户可以删除错误导入的数据
- ✅ 有二次确认机制
- ✅ 需要明确指定年份

### 建议 2: 添加数据备份功能

**建议方案**:

```python
@app.command()
def backup(
    output: Path = typer.Option(..., "--output", "-o", help="备份文件路径"),
):
    """
    备份跑步数据

    示例:
        nanobotrun backup --output backup_2024-01-01.zip
    """
    import shutil
    from datetime import datetime

    config = ConfigManager()
    data_dir = config.data_dir

    # 创建备份
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    backup_name = f"nanobotrun_backup_{timestamp}"
    backup_path = output / backup_name if output.is_dir() else output

    shutil.make_archive(str(backup_path), 'zip', data_dir)
    console.print(f"[green]✓[/green] 备份完成: {backup_path}.zip")
```

**优点**:
- ✅ 用户可以定期备份数据
- ✅ 防止数据丢失
- ✅ 支持数据迁移

### 建议 3: 添加数据恢复功能

**建议方案**:

```python
@app.command()
def restore(
    backup_file: Path = typer.Argument(..., help="备份文件路径"),
    force: bool = typer.Option(False, "--force", "-f", help="强制覆盖现有数据"),
):
    """
    从备份恢复跑步数据

    示例:
        nanobotrun restore backup_2024-01-01.zip
    """
    import shutil

    config = ConfigManager()
    data_dir = config.data_dir

    if not force and data_dir.exists() and any(data_dir.iterdir()):
        from rich.prompt import Confirm
        if not Confirm.ask("[red]数据目录不为空，是否覆盖？[/red]"):
            console.print("[yellow]已取消[/yellow]")
            return

    # 解压备份
    shutil.unpack_archive(str(backup_file), data_dir.parent)
    console.print(f"[green]✓[/green] 恢复完成")
```

**优点**:
- ✅ 支持数据恢复
- ✅ 有覆盖确认机制
- ✅ 支持数据迁移
