## 基于 nanobot SDK 的扩展开发指南

### 一、MCP 扩展能力

#### 1. 配置方式

在 `config.json` 中通过 `tools.mcp_servers` 声明 MCP 服务器：

```json
{
  "tools": {
    "mcp_servers": {
      "filesystem": {
        "type": "stdio",
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
        "tool_timeout": 30,
        "enabled_tools": ["*"]
      },
      "github": {
        "type": "streamableHttp",
        "url": "https://api.example.com/mcp",
        "headers": {"Authorization": "Bearer xxx"},
        "enabled_tools": ["create_issue", "list_repos"]
      }
    }
  }
}
```

#### 2. 三种传输协议

| 协议 | 配置要点 | 适用场景 |
|------|---------|---------|
| `stdio` | `command` + `args` + `env` | 本地进程，如 npx、python 脚本 |
| `sse` | `url`（以 `/sse` 结尾） | Server-Sent Events 服务端 |
| `streamableHttp` | `url` + `headers` | 标准 HTTP MCP 端点 |

#### 3. SDK 中使用 MCP

```python
from nanobot import Nanobot

bot = Nanobot.from_config()  # MCP 服务器自动加载为工具

# Agent 自动调用 mcp_filesystem_read_file 等工具
result = await bot.run("读取 /tmp/data.json 的内容")
```

MCP 工具自动注册到 `ToolRegistry`，命名规则为 `mcp_{server_name}_{tool_name}`，Agent 在推理时自动发现和调用。

#### 4. 自定义 MCP 服务器开发

```python
# 你的 MCP Server（Python）
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("my-tools")

@mcp.tool()
def calculate_tax(amount: float, rate: float) -> float:
    """计算税费"""
    return amount * rate

@mcp.resource("config://app")
def get_config() -> str:
    """返回应用配置"""
    return "version=1.0"

if __name__ == "__main__":
    mcp.run(transport="stdio")
```

然后在 `config.json` 中指向它：

```json
{
  "tools": {
    "mcp_servers": {
      "my-tools": {
        "type": "stdio",
        "command": "python",
        "args": ["/path/to/my_mcp_server.py"]
      }
    }
  }
}
```

### 二、SKILL 扩展能力

#### 1. Skill 文件结构

```
skills/
├── my-skill/
│   └── SKILL.md          # 必须
└── another-skill/
    └── SKILL.md
```

#### 2. SKILL.md 格式

```markdown
---
name: my-skill
description: 描述这个技能的作用
always: false              # true 表示始终加载到上下文
metadata:
  nanobot:
    emoji: 🔧
    requires:              # 依赖检查
      bins: ["ffmpeg"]     # 需要系统有 ffmpeg 命令
      env: ["OPENAI_API_KEY"]  # 需要环境变量
---

# 技能名称

详细说明、使用示例、注意事项等...
```

#### 3. 两种 Skill 来源

| 来源 | 路径 | 优先级 |
|------|------|--------|
| 工作区 Skill | `workspace/skills/` | 高（覆盖同名内置 Skill） |
| 内置 Skill | `nanobot/skills/` | 低 |

#### 4. SDK 中管理 Skill

```python
from pathlib import Path
from nanobot.agent.skills import SkillsLoader

loader = SkillsLoader(
    workspace=Path("/my/workspace"),
    disabled_skills={"weather"},  # 禁用某些技能
)

# 列出所有可用技能
skills = loader.list_skills()

# 加载特定技能内容
content = loader.load_skill("weather")

# 构建技能摘要（供 Agent 上下文）
summary = loader.build_skills_summary()
```

#### 5. 运行时动态添加 Skill

```python
# 在工作区创建 skill 目录
skill_dir = Path("~/.nanobot/workspace/skills/my-tool").expanduser()
skill_dir.mkdir(parents=True, exist_ok=True)

(skill_dir / "SKILL.md").write_text("""---
name: my-tool
description: 我的自定义工具技能
---

# My Tool

使用说明...
""")

# 重新创建 Nanobot 实例即可加载新 Skill
bot = Nanobot.from_config()
```

### 三、Memory 长短期记忆能力

#### 1. 记忆分层架构

```
workspace/
├── SOUL.md              # 长期：Bot 人格和沟通风格（Dream 管理）
├── USER.md              # 长期：用户画像和偏好（Dream 管理）
└── memory/
    ├── MEMORY.md        # 长期：项目事实和决策（Dream 管理）
    ├── history.jsonl    # 短期：对话摘要归档（Consolidator 写入）
    ├── .cursor          # Consolidator 写入游标
    └── .dream_cursor    # Dream 消费游标
```

#### 2. 短期记忆（Session + Consolidator）

- 对话消息保存在 `Session.messages` 中
- 当上下文窗口接近限制时，`Consolidator` 自动将旧消息摘要写入 `history.jsonl`
- 可通过 `session_ttl_minutes` 配置自动压缩空闲会话

```json
{
  "agents": {
    "defaults": {
      "context_window_tokens": 65536,
      "session_ttl_minutes": 30
    }
  }
}
```

#### 3. 长期记忆（Dream）

Dream 是慢速记忆层，定时运行：

```json
{
  "agents": {
    "defaults": {
      "dream": {
        "interval_h": 2,           // 每 2 小时运行一次
        "max_batch_size": 20,      // 每次处理 20 条历史条目
        "max_iterations": 15       // 编辑阶段最多 15 次工具调用
      }
    }
  }
}
```

Dream 的工作流程：
1. 读取 `history.jsonl` 中未处理的新条目
2. 结合现有 `SOUL.md`、`USER.md`、`MEMORY.md`
3. 分两阶段：分析新内容 → 精准编辑长期记忆文件
4. 通过 `GitStore` 记录变更历史（可回溯）

#### 4. SDK 中直接操作记忆

```python
from pathlib import Path
from nanobot.agent.memory import MemoryStore

store = MemoryStore(workspace=Path("~/.nanobot/workspace").expanduser())

# 读取长期记忆
memory = store.read_memory()
soul = store.read_soul()
user = store.read_user()

# 写入记忆（通常由 Dream 自动管理）
store.write_memory("项目使用 Python 3.11+，FastAPI 框架")

# 查询历史
unprocessed = store.read_unprocessed_history(since_cursor=0)
```

#### 5. 在 Agent 对话中利用记忆

```python
from nanobot import Nanobot

bot = Nanobot.from_config()

# 第一次对话：建立上下文
await bot.run("记住：我喜欢简洁的代码风格", session_key="user-alice")

# 后续对话：Agent 自动从 USER.md/MEMORY.md 读取偏好
result = await bot.run("帮我重构这个函数", session_key="user-alice")
# Agent 会记住用户偏好，输出简洁代码

# 不同用户隔离
result2 = await bot.run("你好", session_key="user-bob")
```

#### 6. 手动触发 Dream

通过 CLI 命令或 API：

```bash
nanobot run "/dream"           # 立即运行 Dream
nanobot run "/dream-log"       # 查看最近 Dream 变更
nanobot run "/dream-restore"   # 列出可恢复的记忆版本
```

### 四、综合应用示例

```python
import asyncio
from pathlib import Path
from nanobot import Nanobot
from nanobot.agent import AgentHook, AgentHookContext

class MemoryAuditHook(AgentHook):
    """观察记忆系统行为的钩子"""
    
    def __init__(self):
        super().__init__()
        self.memory_changes = []

    async def after_iteration(self, context: AgentHookContext) -> None:
        # 检查是否有工具修改了记忆文件
        for tc in context.tool_calls:
            if "write" in tc.name and "memory" in str(tc.arguments):
                self.memory_changes.append(tc)

async def main():
    bot = Nanobot.from_config(
        config_path="~/.nanobot/config.json",
        workspace="/my/project"
    )
    
    hook = MemoryAuditHook()
    
    # 多轮对话，记忆自动积累
    await bot.run("项目使用 PostgreSQL 数据库", session_key="proj-x", hooks=[hook])
    await bot.run("API 端口是 8080", session_key="proj-x", hooks=[hook])
    
    # 长期记忆会逐步积累到 MEMORY.md
    result = await bot.run("总结项目技术栈", session_key="proj-x")
    print(result.content)
    
    print(f"观察到 {len(hook.memory_changes)} 次记忆变更")

asyncio.run(main())
```

### 五、关键设计要点

| 能力 | 核心机制 | 开发者控制点 |
|------|---------|-------------|
| **MCP** | `ToolRegistry` 动态注册 | `config.json` 声明 + 自定义 MCP Server |
| **SKILL** | `SkillsLoader` 加载 Markdown | `workspace/skills/` 目录放置 SKILL.md |
| **短期记忆** | `Session` + `Consolidator` | `context_window_tokens`、`session_ttl_minutes` |
| **长期记忆** | `Dream` 定时整理 | `dream.interval_h`、`dream.max_batch_size` |
| **版本控制** | `GitStore` | 自动记录 SOUL/USER/MEMORY 变更 |
