## 二、必须修正项（开发前必须落实）

### F-01：AgentDefaults 品牌字段设计错误

**问题**：架构设计 §10.6.4 提出"AgentDefaults 新增 bot_name 和 bot_icon 字段"，但项目 `AgentDefaults`（`src/core/provider_adapter.py`）是独立的数据类，与 nanobot-ai 的 `AgentsConfig.defaults` 是两个不同的对象。品牌字段应设置在 nanobot 配置的 `agents.defaults` dict 中，而非项目的 `AgentDefaults` dataclass。

**影响**：如果按原设计实施，品牌字段仅存在于项目侧 dataclass，无法传递到 nanobot-ai 的 WebUI 显示层。

**修正方案**：
1. 不修改项目 `AgentDefaults` dataclass
2. 在 `_build_nanobot_config_from_runner()` 中，将 `bot_name`/`bot_icon` 写入 `AgentsConfig(defaults={"model": ..., "bot_name": "Nanobot-Runner", "bot_icon": "🏃‍♂️"})`
3. 品牌配置可从 config.json 读取，也可硬编码默认值

**优先级**：P0 — 不修正则 REQ-D-16 无法实现

### F-02：`--webui` 标志到 RunnerProviderAdapter 的传递路径缺失

**问题**：架构设计 §10.6.2 提出新增 `--webui` 标志，但未设计该标志如何传递到 `RunnerProviderAdapter._build_nanobot_config_from_runner()`。当前 `RunnerProviderAdapter` 在 `__init__` 时创建，此时 CLI 标志尚未解析。

**影响**：如果按原设计实施，`--webui` 标志无法影响 WebSocket 通道的启用状态。

**修正方案**：
1. 方案A（推荐）：`RunnerProviderAdapter.__init__` 新增 `webui_enabled: bool = False` 参数，`gateway.py` 在创建 adapter 时传入
2. 方案B：`_build_nanobot_config_from_runner()` 新增 `webui_enabled` 参数，调用时传入
3. 方案C：通过环境变量 `NANOBOT_WS_ENABLED` 传递（不够显式）

**优先级**：P0 — 不修正则 REQ-D-19 无法实现

### F-03：ConfigManager 缺少 WebSocket 配置读取方法

**问题**：架构设计 §10.6.3 提出新增 `get_websocket_config()` 方法，但未定义该方法的具体接口和返回类型。当前 `ConfigManager` 没有读取 `websocket` 配置节的逻辑。

**影响**：如果按原设计实施，`RunnerProviderAdapter` 无法从 config.json 读取 WebSocket 配置。

**修正方案**：
1. `ConfigManager` 新增 `get_websocket_config() -> dict[str, Any]` 方法
2. 返回 config.json 中 `websocket` 配置节的完整 dict
3. 如果配置节不存在，返回空 dict（由 RunnerProviderAdapter 使用默认值）
4. 支持环境变量覆盖（`NANOBOT_WS_*`）

**优先级**：P0 — 不修正则 REQ-D-17 无法实现

## 三、增强建议（可选采纳）

### S-01：WebSocket 端口与 Gateway 端口分离说明

**建议**：当前架构设计中，Gateway 服务端口（`--port`，默认 18790）与 WebSocket 端口（默认 8765）是两个不同的端口。建议在架构文档和 CLI 帮助信息中明确说明这两个端口的区别和用途，避免用户混淆。

**理由**：Gateway 端口用于飞书 WebSocket 长连接，WebSocket 端口用于 WebUI 客户端连接。两者功能不同，端口不同。

### S-02：WebUI 启动后自动打开浏览器

**建议**：当 `--webui` 标志启用时，启动完成后自动打开默认浏览器访问 WebUI 地址。可通过 `webbrowser.open()` 实现。

**理由**：提升用户体验，减少手动输入 URL 的步骤。可作为 `--webui-open` 子标志控制。

## 四、风险识别与评估

| 风险编号 | 风险描述 | 等级 | 影响 | 缓解措施 |
|----------|---------|------|------|---------|
| R-01 | WebSocket token 认证配置不当导致 WebUI 无法连接 | 中 | 本地访问时若 token_issue_secret 未配置且 websocket_requires_token=True，WebUI 客户端可能无法获取有效 token | 默认配置 token_issue_path="/token"，本地访问时 token_issue_secret 留空即可自动签发 |
| R-02 | nanobot-ai 版本更新导致 WebSocket Channel API 变更 | 低 | 上游 API 变更可能破坏 WebSocket 配置兼容性 | 锁定 nanobot-ai 版本；配置项使用标准字段名 |
| R-03 | 多通道同时运行时 AgentLoop 资源竞争 | 低 | 飞书和 WebSocket 通道共享同一 AgentLoop，高并发时可能排队 | 单用户场景下并发极低；nanobot-ai AgentLoop 天然支持多通道 |
| R-04 | WebUI SPA 资源文件缺失 | 低 | 如果 nanobot-ai wheel 未包含 web dist，WebUI 无法加载 | 启动时检查 `_default_webui_dist()` 返回值，缺失时给出明确错误提示 |

3. ✅ F-03：`ConfigManager` 新增 `get_websocket_config()` 方法
