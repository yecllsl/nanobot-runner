### 3.2 ✅ 已修正：进化状态API"只读检查"语义保障

**问题** (§12.3.2)：`GET /api/webui/evolution/status` 调用 `EvolutionController.check_triggers()`，虽然当前实现是只读的（返回TriggerCheckResult不执行动作），但API文档未明确这一语义约束。未来如果`check_triggers()`实现变更（如加入自动执行），GET请求将产生副作用，违反HTTP语义。

**修正结果**：已在架构设计 §12.3.2 + §12.4 中明确只读语义+注释保护。

```python
@router.get("/evolution/status")
async def get_evolution_status(...):
    """获取进化引擎状态（只读，不触发任何进化动作）"""
    # check_triggers() 仅检查条件，不执行动作
    result = await run_in_threadpool(controller.check_triggers)
    ...
```

### 3.3 ✅ 已修正：进化报告列表API数据源定义

**问题** (§12.3.2)：`GET /api/webui/evolution/reports` 需要返回历史报告列表，但`EvolutionReporter.generate_report()`是实时生成报告，不持久化报告对象。`EvolutionStore`也没有`list_reports()`方法。

**修正方案（已采用）**：扫描`data_dir/decisions/`目录月份分片 + 按需生成策略：

1. 报告列表API：扫描`data_dir/decisions/`目录下的月份子目录（`YYYY-MM`格式），作为可生成报告的月份列表
2. 报告详情API：调用`EvolutionReporter.generate_report(month)`实时生成
3. 不新增报告持久化机制，保持与CLI `evolution report` 行为一致

```python
# GET /api/webui/evolution/reports 响应结构
{
    "available_months": ["2026-06", "2026-05", "2026-04"],
    "count": 3
}
```

### 4.2 🟡 建议补充：API错误响应格式统一

当前v0.28.0路由未定义统一的错误响应格式。建议v0.29.0新增路由统一使用FastAPI的`HTTPException`：

```python
from fastapi import HTTPException

raise HTTPException(status_code=404, detail="计划不存在")
```
