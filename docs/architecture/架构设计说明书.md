### 10.5 ADR 决策记录

> ADR 是项目知识资产，永久保留作为设计依据。

#### ADR-015：WebSocket 通道配置方式

**背景**：需要为 nanobot-ai 内置 WebSocket 通道提供配置，使其能启用 WebUI。

**决策**：在项目 config.json 新增 `websocket` 配置节，由 RunnerProviderAdapter 翻译为 nanobot WebSocketConfig。

**影响**：
- ✅ 与飞书通道配置模式一致，配置驱动
- ✅ 用户无需直接操作 nanobot 配置文件
- ✅ 支持环境变量覆盖
- ❌ 需要修改 ConfigManager 和 RunnerProviderAdapter

**替代方案**：
- 直接修改 `~/.nanobot/config.json`：破坏项目配置封装，不采用
- 仅通过 CLI 标志配置：无法持久化配置，不采用
- 仅通过环境变量配置：配置项过多，不采用

#### ADR-016：WebUI 启用方式

**背景**：用户需要一种便捷方式启用 WebUI，同时不影响现有飞书通道行为。

**决策**：采用 `gateway start --webui` CLI 标志 + config.json `websocket.enabled` 双重控制。CLI 标志优先级高于配置文件。

**影响**：
- ✅ 显式启用，不影响现有行为（默认不启用）
- ✅ 配置文件可持久化启用状态
- ✅ CLI 标志适合临时启用场景
- ❌ 两个启用入口可能造成混淆（文档需明确优先级）

**替代方案**：
- 仅 config.json 控制：需要用户手动编辑配置文件，不够便捷
- 仅 CLI 标志控制：无法持久化，每次启动需手动指定
- 新增独立 `webui` 命令：与 gateway 职责重叠，不采用

#### ADR-017：安全认证策略

**背景**：WebSocket 通道需要认证机制防止未授权访问。

**决策**：默认启用 token 认证（`websocket_requires_token=True`），采用 token_issue_path 短期令牌签发机制。本地访问（127.0.0.1）时 token_issue_secret 可选。

**影响**：
- ✅ 安全默认，防止未授权访问
- ✅ 短期令牌机制比静态令牌更安全
- ✅ 本地访问零配置即可使用
- ❌ 非本地访问需额外配置 token_issue_secret

**替代方案**：
- 无认证：安全风险高，不采用
- 仅静态 token：不够灵活，不采用
- OAuth2：过度设计，个人项目不需要

### 12.6 架构决策记录

#### ADR-023: 计划调整AI模式采用跳转8765而非嵌入8766

**背景**：REQ-D-40要求通过AI对话调整计划，WebUI已有nanobot-ai对话服务（8765端口）。

**决定**：点击"AI调整"按钮跳转到8765端口nanobot-ai WebUI，不在8766中嵌入对话组件。

**理由**：
- 8765已有完整的对话UI和Agent工具链，嵌入需重复实现
- 跳转方式零开发成本，立即可用
- 对话上下文注入可通过URL参数传递plan_id

**替代方案**：
- iframe嵌入8765页面：跨域安全风险，样式不统一
- 在8766中实现独立对话组件：重复开发，维护成本高

#### ADR-024: 日历组件采用自建轻量实现（CSS Grid），不引入第三方日历库

**背景**：REQ-D-37需要日历视图展示训练计划，需选择日历组件实现方案。

**决定**：使用CSS Grid自建日历组件，不引入第三方日历库（如react-calendar、fullcalendar）。

**理由**：
- 训练计划日历需求简单（按月/周展示训练卡片），无需复杂交互
- 第三方库体积大（fullcalendar ~200KB+），引入过度
- CSS Grid + date-fns（~3KB）足以实现
- 与项目"零新增依赖"策略一致

**替代方案**：
- react-calendar：功能过多，样式定制成本高
- fullcalendar：重量级，训练计划场景不需要其高级功能

#### ADR-025: 进化状态采用手动刷新而非WebSocket推送

**背景**：REQ-D-42需要展示进化引擎状态，需决定状态更新方式。

**决定**：采用手动刷新（页面加载时请求 + 用户点击刷新按钮），不使用WebSocket实时推送。

**理由**：
- 进化引擎状态变化频率极低（按天/周级别），实时推送无必要
- WebSocket引入复杂度（连接管理、断线重连），与个人项目轻量化定位不符
- 手动刷新与v0.28.0现有页面交互模式一致

**替代方案**：
- WebSocket实时推送：过度设计，维护成本高
- 轮询（setInterval）：折中方案，但进化状态变化频率不支持轮询必要性
