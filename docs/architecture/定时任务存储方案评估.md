# 定时任务存储方案评估报告

## 方案概述

**目标**：将定时任务配置存储在业务目录（`~/.nanobot-runner/cron/jobs.json`），同时使用 nanobot 框架的 `CronService` 作为执行器。

## 可行性分析

### 1. CronService 设计评估

**核心接口**：
```python
class CronService:
    def __init__(
        self,
        store_path: Path,  # 接受任意路径
        on_job: Callable[[CronJob], Coroutine[Any, Any, str | None]] | None = None
    ):
        self.store_path = store_path
        ...
```

**关键发现**：
- ✅ `CronService` 不限制存储路径，可以接受任意 `Path` 对象
- ✅ 存储路径通过构造函数传入，完全可配置
- ✅ 不依赖 `~/.nanobot/` 目录的硬编码路径

**结论**：CronService 设计支持自定义存储路径。

### 2. 当前实现分析

#### gateway 模式（cli.py）
```python
# 当前实现
cron_store_path = get_data_dir() / "cron" / "jobs.json"  # ~/.nanobot/cron/jobs.json
cron = CronService(cron_store_path)

async def on_cron_job(job: CronJob) -> str | None:
    response = await agent.process_direct(job.payload.message, ...)
    
cron.on_job = on_cron_job
await cron.start()
```

#### report 命令（report_service.py）
```python
# 当前实现
self.cron_store = get_data_dir() / "cron" / "jobs.json"  # ~/.nanobot/cron/jobs.json
self.cron_service = CronService(store_path=self.cron_store)
self.cron_service.add_job(name="daily_report", ...)
```

**问题**：两者都使用 `get_data_dir()` 返回 `~/.nanobot/`，违反 AGENTS.md 原则。

### 3. 方案实现

#### 修改点1：统一 cron 存储路径

**位置**：`src/core/config.py` - ConfigManager 类

```python
class ConfigManager:
    def __init__(self):
        self.base_dir = Path.home() / ".nanobot-runner"
        self.data_dir = self.base_dir / "data"
        self.config_file = self.base_dir / "config.json"
        self.index_file = self.data_dir / "index.json"
        
        # 新增：cron 存储路径
        self.cron_dir = self.base_dir / "cron"
        self.cron_store = self.cron_dir / "jobs.json"
        
        self._ensure_dirs()
        self._ensure_config()
    
    def _ensure_dirs(self):
        self.base_dir.mkdir(parents=True, exist_ok=True)
        self.data_dir.mkdir(parents=True, exist_ok=True)
        self.cron_dir.mkdir(parents=True, exist_ok=True)  # 新增
```

#### 修改点2：gateway 使用业务 cron 存储

**位置**：`src/cli.py` - gateway 命令

```python
# 修改前
from nanobot.config.loader import get_data_dir
cron_store_path = get_data_dir() / "cron" / "jobs.json"

# 修改后
from src.core.config import ConfigManager
config_manager = ConfigManager()
cron_store_path = config_manager.cron_store
```

#### 修改点3：report 使用业务 cron 存储

**位置**：`src/core/report_service.py` - ReportService 类

```python
# 修改前
from nanobot.config.loader import get_data_dir
self.cron_store = get_data_dir() / "cron" / "jobs.json"

# 修改后
self.cron_store = self.config.cron_store  # 使用 ConfigManager 的路径
```

### 5. 潜在问题与解决方案

#### 问题1：路径不一致风险
**风险**：如果 gateway 和 report 使用不同的路径，会导致定时任务无法执行。

**解决方案**：
- 在 `ConfigManager` 中统一管理 `cron_store` 路径
- 所有模块都从 `ConfigManager` 获取路径
- 添加单元测试验证路径一致性

#### 问题2：现有数据迁移
**风险**：用户可能已经在 `~/.nanobot/cron/jobs.json` 中有定时任务配置。

**解决方案**：
```python
# 在 ConfigManager 初始化时迁移
def _ensure_dirs(self):
    # 创建新目录
    self.cron_dir.mkdir(parents=True, exist_ok=True)
    
    # 迁移现有数据
    old_cron_store = Path.home() / ".nanobot" / "cron" / "jobs.json"
    if old_cron_store.exists() and not self.cron_store.exists():
        import shutil
        shutil.copy2(old_cron_store, self.cron_store)
        logger.info(f"已迁移定时任务配置：{old_cron_store} -> {self.cron_store}")
```

#### 问题3：执行器依赖 gateway
**风险**：定时任务配置存储在业务目录，但执行器需要 gateway 启动才能运行。

**解决方案**：
- 文档说明：定时任务需要 gateway 运行
- 未来扩展：可以提供独立的定时任务执行器命令
