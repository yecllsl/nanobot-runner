## 3. 问题清单

### 3.1 关键问题（必须在开发前修复）

| 编号 | 问题 | 严重度 | 说明 |
|------|------|--------|------|
| **ISSUE-01** | FastAPI 默认 multipart 上传大小限制为 1MB | 🔴 高 | 设计文档设定单文件 ≤50MB，但 FastAPI 默认 `max_body_size` 为 1MB，50MB 文件会被直接拒绝。**需要在路由中显式配置**：`Request(max_body_size=50*1024*1024)` 或通过 `uvicorn --limit-max-requests` 设置。**设计文档未提及此配置**。 |

### 3.2 建议（非阻塞）

| 编号 | 建议 | 说明 |
|------|------|------|
| SUG-01 | 前端上传后自动触发导入 vs 手动点击「开始导入」按钮 | 设计文档布局显示有「开始导入」按钮，但需求描述为"选中后自动触发导入"。两处存在歧义，建议统一为**手动点击按钮**（更可控，避免用户误选文件后立即触发上传） |
| SUG-02 | 考虑 `Request` 对象直接读取 `max_body_size` 或使用 Starlette 的 `request.max_content_length` | 与 ISSUE-01 配套，建议在路由函数签名中显式声明 `request: Request`，并在 uvicorn 配置中或不使用额外配置，而是通过 FastAPI 的 `Request` 参数或中间件控制 |
| SUG-03 | 前端 `Content-Type: multipart/form-data` 不应手动设置 | axios 在发送 FormData 时会自动设置正确的 Content-Type（含 boundary）。手动设置 `headers: { 'Content-Type': 'multipart/form-data' }` 会丢失 boundary，导致服务端解析失败。**建议删除该 header**，让 axios 自动处理 |

## 4. 风险清单

| 风险 | 等级 | 设计已有缓解 | 补充建议 |
|------|------|-------------|----------|
| 大批量上传请求超时 | 中 | 前端 timeout 5min + 文件数限制 50 个 | 建议后端也设 timeout（uvicorn 默认 60s，需确认 ≥5min） |
| 临时文件磁盘占用 | 低 | 每请求独立 mkdtemp + finally 清理 | 无 |
| ImportService 无 TTY 时 rich.Console 输出 | 低 | 已验证不报错 | 建议集成测试中验证无异常输出 |
| 文件名冲突 | 低 | 临时目录隔离 | 无 |
| 前后端文件大小限制不一致 | 中 | 前端 50MB 校验 | **ISSUE-01 修复后解决** |
