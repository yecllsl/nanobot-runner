**[MEDIUM] Section 6.8 中 `RacePredictionEngine` 注释与实现不一致**

- **文件**: [架构设计说明书](file:///d:/yecll/Documents/LocalCode/RunFlowAgent/docs/architecture/架构设计说明书.md) L1805

- **问题描述**:
  ```python
  @property
  def race_prediction_engine(self) -> Any:
      """获取比赛预测引擎（v0.20.0暴露为AppContext属性）

      注: RacePredictionEngine为无状态工具类，内部不依赖其他核心组件，
      无需通过依赖注入传入外部依赖，直接实例化即可。
      """
  ```
  注释说"直接实例化即可"，但实际代码中 `RacePredictionEngine()` 确实是无参数构造。然而这与项目铁律"禁止直接实例化核心组件"存在语义冲突。建议注释改为说明为何此处是例外（无状态、无外部依赖），避免开发工程师在其他地方模仿此模式。

- **整改方案**:
  保持代码不变，优化注释为："RacePredictionEngine 为纯函数式工具类（无内部状态、无外部依赖），作为特例允许直接实例化。其他有状态或有外部依赖的核心组件必须通过 AppContext 属性注入。"
