| P-01 | **CRITICAL** | DecisionLogHook继承ObservabilityHook的设计与代码实际不符。架构设计说明书Section 8.2.6描述DecisionLogHook继承ObservabilityHook，但实际代码中ObservabilityHook继承自AgentHook（nanobot-ai基类），其`finalize_content`方法在结束时将`_current_trace_id`置为None。DecisionLogHook如果先调`super().finalize_content()`，则`self._current_trace_id`已被父类置为None，后续逻辑无法使用trace_id关联决策。同时父类在finalize_content中会调用`self.engine.generate_explanation(decision)`创建AIDecision，DecisionLogHook在同一钩子中也会创建DecisionLog，两者职责重叠、语义冲突。 | 继承链设计在运行时会产生状态竞争和语义冲突，DecisionLogHook无法可靠地在finalize_content中完成决策日志记录。直接阻塞开发。 | **方案A（推荐）**: DecisionLogHook直接继承AgentHook而非ObservabilityHook，作为独立Hook注册。在create_composite_hook()中ObservabilityHook之后注册，两者互不干扰。通过构造函数注入DecisionLogger实现解耦。**方案B**: 若坚持继承，需重写finalize_content不调super()，但违反"先调super()再扩展"原则，且丧失可观测性追踪能力。 | 1. 验证DecisionLogHook与ObservabilityHook在finalize_content中无状态竞争；2. 验证两者独立注册后Agent迭代生命周期中均正常触发；3. 性能测试确认双Hook延迟<100ms |

## 6. 评审总结

### 6.1 核心问题

v0.23.0决策追踪模块的架构设计在模块划分、数据模型、存储策略方面设计合理，薄编排层模式与项目现有架构风格一致。但存在两个CRITICAL级问题阻塞开发：

1. **DecisionLogHook继承链设计缺陷**（P-01）：ObservabilityHook的finalize_content会将_current_trace_id置为None并调用engine.generate_explanation()，DecisionLogHook继承后无法安全扩展。建议改为独立继承AgentHook。

2. **关键接口未定义**（P-02）：runner_state摘要字段和PlanManager执行数据适配方案缺失，导致两个核心功能（决策日志记录和计划执行忠实度计算）无法实现。

### 6.2 整改优先级

| 优先级 | 整改项 | 预计工时 |
|--------|--------|---------|
| P0-阻塞 | P-01: DecisionLogHook继承方案重新设计 | 2h |
| P0-阻塞 | P-02: runner_state摘要字段定义 + PlanExecutionDataAdapter设计 | 4h |
| P1-必须 | P-03: execution_status枚举值统一 | 0.5h |
| P1-必须 | P-04: 异步写入错误恢复机制 | 2h |
| P1-必须 | P-05: MAE和偏差方向字段补充 | 1h |
| P1-必须 | P-06: create_composite_hook注册方案调整 | 1h |
| P2-建议 | P-07~P-11 | 3h |

### 6.3 整改后重新评审条件

1. P-01和P-02整改完成，架构设计说明书Section 8.2更新
2. P-03~P-06整改完成，架构设计说明书同步更新
3. 三文档（架构设计/需求规格/产品规划）关键不一致项已对齐

### 6.4 架构亮点

尽管存在上述问题，本次架构设计有以下值得肯定的亮点：

1. **DecisionLog + OutcomeRecord分离模型**：职责清晰、查询高效、与存储分片自然对齐，优于单表设计
2. **复用DecisionType枚举**：避免枚举碎片化，与transparency模块保持一致
3. **复用AskUserConfirmManager**：扩展ConfirmScenario.DECISION_FEEDBACK场景，减少新代码量
4. **Parquet按月分片**：与现有data/按年分片策略一致，查询粒度适配决策追踪场景
5. **排除范围明确**：预测校准、训练响应性分析、提示策略优化等延后至v0.24/v0.25，MVP边界清晰
