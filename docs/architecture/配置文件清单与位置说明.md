## 1. 配置文件总览

| **环境变量** | `.env.local` | 项目根目录 | 项目级 | ❌ 不纳入Git |
| **业务配置** | `config.json` | `./nanobot-runner/` | 项目级 | ❌ 不纳入Git |

### 2.1 项目根目录配置文件

#### 2.1.1 `.env.example` - 环境变量模板

**位置**：`<project_root>/.env.example`

**用途**：环境变量配置模板，包含所有支持的环境变量及其说明。

**是否纳入Git**：✅ 纳入版本控制

**内容示例**：
```bash
# Nanobot Runner 环境变量配置模板
# 复制此文件为 .env.local 并填写实际值

# ========================================
# nanobot-ai 框架配置
# ========================================

# LLM提供商配置（必填）
NANOBOT_PROVIDERS_OPENAI_APIKEY=sk-xxxxxxxxxxxxxxxx
NANOBOT_PROVIDERS_OPENAI_BASEURL=https://api.openai.com/v1

# Agent默认配置（可选）
NANOBOT_AGENTS_DEFAULTS_MODEL=gpt-4
NANOBOT_AGENTS_DEFAULTS_TEMPERATURE=0.7

# ========================================
# 业务配置
# ========================================

# 配置目录路径（可选，默认为 ./nanobot-runner）
NANOBOT_CONFIG_DIR=./nanobot-runner

# 数据目录路径（可选，默认为 ./nanobot-runner/data）
NANOBOT_DATA_DIR=./nanobot-runner/data

# 时区设置（可选，默认为 Asia/Shanghai）
NANOBOT_TIMEZONE=Asia/Shanghai

# 默认查询年份（可选，默认为当前年份）
NANOBOT_DEFAULT_YEAR=2024

# ========================================
# 飞书通知配置（可选）
# ========================================

# 飞书应用配置（如需使用飞书通知，必填）
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_RECEIVE_ID=ou_xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_RECEIVE_ID_TYPE=user_id

# 是否自动推送到飞书（可选，默认为 false）
NANOBOT_AUTO_PUSH_FEISHU=false

# ========================================
# 性能优化配置（可选）
# ========================================

# 配置缓存TTL（秒，默认为 300）
NANOBOT_PERFORMANCE_CACHE_TTL=300

# 是否启用LazyFrame优化（默认为 true）
NANOBOT_PERFORMANCE_LAZY_FRAME_OPTIMIZATION=true

# 批处理大小（默认为 100）
NANOBOT_PERFORMANCE_BATCH_SIZE=100
```

## 3. 配置文件优先级

### 3.1 配置加载优先级

配置加载遵循以下优先级（从高到低）：

```
环境变量 > 项目配置文件 > 全局配置文件 > 默认值
```

**具体优先级**：

1. **环境变量**（最高优先级）
   - `NANOBOT_*` 环境变量
   - `.env.local` 文件

2. **项目配置文件**
   - `./nanobot-runner/config.json`（业务配置）
   - `./.nanobot/config.json`（框架配置，可选）

3. **全局配置文件**（兼容旧架构）
   - `~/.nanobot-runner/config.json`（业务配置）
   - `~/.nanobot/config.json`（框架配置）

4. **默认值**（最低优先级）
   - 代码中定义的默认值

### 3.2 配置覆盖示例

**场景1：环境变量覆盖配置文件**

```bash
# 配置文件中 data_dir 为 ./data
# 环境变量覆盖为 ./custom-data
export NANOBOT_DATA_DIR=./custom-data

# 最终生效：./custom-data
```

**场景2：项目配置覆盖全局配置**

```bash
# 全局配置：~/.nanobot-runner/config.json
{
  "data_dir": "~/.nanobot-runner/data"
}

# 项目配置：./nanobot-runner/config.json
{
  "data_dir": "./nanobot-runner/data"
}

# 最终生效：./nanobot-runner/data（项目配置优先）
```

## 4. 配置文件管理最佳实践

### 4.1 版本控制策略

**纳入Git的配置文件**：
```gitignore
# 纳入版本控制
.env.example
config.example.json
nanobot-runner/AGENTS.md
nanobot-runner/SOUL.md

**不纳入Git的配置文件**：

nanobot-runner/memory/

.nanobot/config.json
```

### 4.2 配置文件初始化流程

**新成员加入项目**：

```bash
# 1. 克隆项目
git clone <project_url>
cd <project_root>

# 2. 复制环境变量模板
cp .env.example .env.local

# 3. 编辑环境变量（填写实际值）
vim .env.local

# 4. 复制配置文件模板
cp config.example.json nanobot-runner/config.json

# 5. 编辑配置文件（填写实际值）
vim nanobot-runner/config.json

# 6. 初始化数据目录
mkdir -p nanobot-runner/data
mkdir -p nanobot-runner/memory
mkdir -p nanobot-runner/sessions
```

### 4.3 配置文件迁移流程

**从旧架构迁移到新架构**：

```bash
# 1. 创建项目配置目录
mkdir -p nanobot-runner

# 2. 迁移配置文件
cp ~/.nanobot-runner/config.json nanobot-runner/config.json
cp ~/.nanobot-runner/AGENTS.md nanobot-runner/AGENTS.md
cp ~/.nanobot-runner/SOUL.md nanobot-runner/SOUL.md
cp ~/.nanobot-runner/USER.md nanobot-runner/USER.md

# 3. 迁移数据目录
cp -r ~/.nanobot-runner/data nanobot-runner/data
cp -r ~/.nanobot-runner/memory nanobot-runner/memory
cp -r ~/.nanobot-runner/sessions nanobot-runner/sessions

# 4. 更新配置文件路径
# 编辑 nanobot-runner/config.json，将 data_dir 改为相对路径
vim nanobot-runner/config.json

# 5. 设置环境变量（可选）
export NANOBOT_CONFIG_DIR=./nanobot-runner
```

## 7. 常见问题

### 7.1 配置文件找不到怎么办？

**问题**：运行时报错"配置文件不存在"。

**解决方案**：
1. 检查环境变量 `NANOBOT_CONFIG_DIR` 是否正确设置
2. 检查项目目录下是否存在 `nanobot-runner/config.json`
3. 如果是首次运行，复制配置模板：
   ```bash
   cp config.example.json nanobot-runner/config.json
   ```

### 7.2 配置优先级不明确怎么办？

**问题**：不确定哪个配置会生效。

**解决方案**：
1. 使用配置验证脚本查看实际生效的配置：
   ```bash
   python scripts/validate_config.py --verbose
   ```
2. 检查环境变量是否覆盖了配置文件：
   ```bash
   env | grep NANOBOT_
   ```

### 7.3 敏感信息泄露怎么办？

**问题**：不小心将敏感信息提交到Git。

**解决方案**：
1. 立即撤销提交：
   ```bash
   git reset --hard HEAD~1
   ```
2. 从Git历史中删除敏感信息：
   ```bash
   git filter-branch --force --index-filter \
     "git rm --cached --ignore-unmatch .env.local" \
     --prune-empty --tag-name-filter cat -- --all
   ```
3. 强制推送：
   ```bash
   git push origin --force --all
   ```
4. 更新所有敏感信息（API密钥等）
