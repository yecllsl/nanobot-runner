## 1. 架构设计概述

### 1.1 设计目标
基于现有架构实现**Agent 自然语言交互能力**，确保：
- ✅ **无重大重构**：充分利用现有模块，最小化架构调整
- ✅ **接口完整覆盖**：支持 6 个核心工具的 Agent 调用
- ✅ **零额外成本**：部署架构保持不变，无新增基础设施

### 1.2 设计原则
| 原则 | 说明 | 实施方式 |
|------|------|---------|
| **本地优先** | 所有数据处理在本地完成 | 使用 nanobot-ai 本地模型 |
| **插件化扩展** | 基于现有工具集扩展 | RunnerTools 新增 2 个查询工具 |
| **高性能查询** | 利用 Polars Lazy API | 谓词下推、列剪枝优化 |
| **隐私保护** | 无外部网络请求 | 本地推理、数据不出本地 |

### 1.3 与原有架构的关系
本迭代架构是原有架构的**增量扩展**：
- **保持不变的模块**：
  - 数据存储架构（Parquet 分片存储）
  - 去重索引机制（index.json）
  - 数据导入流程（解析 - 校验 - 落盘）
  - 基础分析引擎（VDOT、心率漂移计算）
  
- **新增/调整的模块**：
  - CLI 交互入口（新增 chat 命令实现）
  - Agent 集成层（新增 nanobot-ai 集成）
  - 工具集扩展（新增 2 个查询工具）
  - 响应格式化（新增 Rich 可视化）

**文件位置**: `src/cli.py`

```python
@app.command()
def chat():
    """
    启动自然语言交互模式
    """
    console.print("[bold green]🤖 Nanobot Runner Agent[/bold green]")
    console.print("[dim]基于 nanobot-ai 的本地跑步数据助理[/dim]")
    console.print("=" * 60)
    
    # 初始化 Agent
    from nanobot_ai import NanobotAgent
    from src.agents.tools import RunnerTools, TOOL_DESCRIPTIONS
    
    storage = StorageManager()
    tools = RunnerTools(storage)
    
    agent = NanobotAgent(
        tools=[tools],
        tool_descriptions=TOOL_DESCRIPTIONS,
        memory_window=10,  # 保留最近 10 轮对话
        local_model=True,  # 使用本地模型
    )
    
    # REPL 循环
    while True:
        try:
            # 获取用户输入
            user_input = Prompt.ask("\n[bold cyan]您[/bold cyan]")
            
            # 退出命令
            if user_input.lower() in ["exit", "quit", "q"]:
                console.print("[yellow]再见！祝您跑步愉快！[/yellow]")
                break
            
            # 空输入处理
            if not user_input.strip():
                continue
            
            # Agent 处理
            with console.status("[bold green]思考中...", spinner="dots"):
                response = agent.chat(user_input)
            
            # 显示回复
            console.print()
            console.print(f"[bold magenta]Agent[/bold magenta] {response}")
            
        except KeyboardInterrupt:
            console.print("\n[yellow]检测到中断，已退出[/yellow]")
            break
        except Exception as e:
            console.print(f"[red]错误：{str(e)}[/red]")
```

**关键技术点**:
- 使用 `Prompt.ask()` 实现交互式输入
- 使用 `console.status()` 显示加载状态
- 异常处理覆盖所有边界情况

#### 4.2.1 query_by_date_range 实现

**文件位置**: `src/agents/tools.py`

**接口定义**:
```python
def query_by_date_range(
    self,
    start_date: str,  # 格式：YYYY-MM-DD
    end_date: str     # 格式：YYYY-MM-DD
) -> List[Dict[str, Any]]:
    """
    按日期范围查询跑步记录
    
    Args:
        start_date: 开始日期（YYYY-MM-DD）
        end_date: 结束日期（YYYY-MM-DD）
        
    Returns:
        list: 跑步记录列表，每条记录包含：
            - timestamp: 时间戳
            - distance: 距离（公里）
            - duration: 用时（秒）
            - avg_heart_rate: 平均心率
            - avg_pace: 平均配速
            
    性能要求:
        - 百万级数据量下查询响应 < 3 秒
        - 使用 Polars Lazy API 优化
    """
    # 解析日期
    start_dt = datetime.strptime(start_date, "%Y-%m-%d")
    end_dt = datetime.strptime(end_date, "%Y-%m-%d")
    
    # 使用 Polars LazyFrame 查询
    lf = self.storage.read_parquet()
    
    # 过滤日期范围
    filtered_lf = lf.filter(
        pl.col("timestamp").is_between(start_dt, end_dt)
    )
    
    # 选择需要的列（列剪枝优化）
    selected_lf = filtered_lf.select([
        "timestamp",
        "total_distance",
        "total_timer_time",
        "avg_heart_rate",
        "avg_pace",
    ])
    
    # 排序并收集
    df = selected_lf.sort("timestamp", descending=True).collect()
    
    # 转换为字典列表
    results = []
    for row in df.iter_rows(named=True):
        results.append({
            "timestamp": str(row.get("timestamp", "N/A")),
            "distance": row.get("total_distance", 0) / 1000,  # 米转公里
            "duration": row.get("total_timer_time", 0),
            "heart_rate": row.get("avg_heart_rate", "N/A"),
            "pace": row.get("avg_pace", "N/A"),
        })
    
    return results
```

**性能优化**:
- 使用 `filter().select()` 链式调用，触发谓词下推
- 只选择需要的列，减少内存占用
- 使用 LazyFrame 延迟执行，优化查询计划

#### 4.2.2 query_by_distance 实现

**文件位置**: `src/agents/tools.py`

**接口定义**:
```python
def query_by_distance(
    self,
    min_distance: float,  # 单位：公里
    max_distance: Optional[float] = None  # 单位：公里，None 表示无上限
) -> List[Dict[str, Any]]:
    """
    按距离范围查询跑步记录
    
    Args:
        min_distance: 最小距离（公里）
        max_distance: 最大距离（公里），None 表示无上限
        
    Returns:
        list: 跑步记录列表
        
    示例:
        # 查询 10 公里以上的跑步
        query_by_distance(min_distance=10)
        
        # 查询 5-10 公里的跑步
        query_by_distance(min_distance=5, max_distance=10)
    """
    # 转换为米
    min_meters = min_distance * 1000
    max_meters = max_distance * 1000 if max_distance else None
    
    # 使用 Polars LazyFrame 查询
    lf = self.storage.read_parquet()
    
    # 构建距离过滤条件
    if max_meters:
        distance_filter = pl.col("total_distance").is_between(min_meters, max_meters)
    else:
        distance_filter = pl.col("total_distance") >= min_meters
    
    # 过滤并选择列
    filtered_lf = lf.filter(distance_filter).select([
        "timestamp",
        "total_distance",
        "total_timer_time",
        "avg_heart_rate",
        "avg_pace",
    ])
    
    # 排序并收集
    df = filtered_lf.sort("timestamp", descending=True).collect()
    
    # 转换为字典列表
    results = []
    for row in df.iter_rows(named=True):
        results.append({
            "timestamp": str(row.get("timestamp", "N/A")),
            "distance": row.get("total_distance", 0) / 1000,
            "duration": row.get("total_timer_time", 0),
            "heart_rate": row.get("avg_heart_rate", "N/A"),
            "pace": row.get("avg_pace", "N/A"),
        })
    
    return results
```

```python
from rich.table import Table

def format_runs_table(runs: List[Dict[str, Any]]) -> Table:
    """
    将跑步记录格式化为 Rich 表格
    """
    table = Table(title="最近跑步记录", show_header=True, header_style="bold magenta")
    
    # 添加列
    table.add_column("日期", style="cyan", width=12)
    table.add_column("距离", style="green", width=8, justify="right")
    table.add_column("用时", style="yellow", width=10, justify="right")
    table.add_column("平均心率", style="red", width=10, justify="right")
    table.add_column("配速", style="blue", width=10, justify="right")

    for run in runs:
        table.add_row(
            run["timestamp"][:10],  # 只取日期部分

            format_pace(run["pace"]) if run["pace"] != "N/A" else "N/A",
        )
    
    return table
```

#### 4.5.1 异常分层处理

**实现方案**:
```python
from typing import Optional, Callable
from functools import wraps

def handle_agent_errors(default_response: str = "抱歉，暂时无法完成此操作"):
    """
    Agent 错误处理装饰器
    """
    def decorator(func: Callable) -> Callable:
        @wraps(func)
        def wrapper(*args, **kwargs):
            try:
                return func(*args, **kwargs)
            except FileNotFoundError:
                return "暂无数据，请先使用 import 命令导入跑步数据"
            except ValueError as e:
                return f"参数错误：{str(e)}"
            except Exception as e:
                # 记录详细日志
                import logging
                logging.error(f"工具调用失败：{e}", exc_info=True)
                return default_response
        return wrapper
    return decorator

# 使用示例
@handle_agent_errors()
def query_by_date_range(self, start_date: str, end_date: str):
    # ... 实现代码 ...
```

#### 4.5.2 边界情况处理

| 场景 | 检测条件 | 处理策略 | 用户提示 |

#### 5.2.2 工具调用协议

**Agent 调用流程**:
```
1. 用户输入："我上周跑了多少次？"
2. Agent 意图识别：查询统计
3. Agent 选择工具：get_running_stats
4. Agent 提取参数：start_date="上周", end_date="本周"
5. 调用工具：tools.get_running_stats(start_date, end_date)
6. 工具返回：{"total_runs": 5, ...}
7. Agent 生成回复："您上周跑了 5 次..."
```

**参数映射规则**:

| "最近 10 次" | limit | 提取数字 10 |
| "10 公里以上" | min_distance | 提取数字 10 |

### 5.3 数据接口变更

#### 5.3.1 Polars 查询优化

**原查询方式**（Eager）:
```python
df = pl.read_parquet(filepath)
filtered = df.filter(pl.col("timestamp") > start_date)
```

**优化后查询方式**（Lazy）:
```python
lf = pl.scan_parquet(filepath)
filtered = lf.filter(pl.col("timestamp") > start_date).collect()
```

**性能提升**:
- 谓词下推：只读取符合条件的行
- 列剪枝：只读取需要的列
- 内存占用降低 60-80%
- 查询速度提升 2-3 倍

## 11. 附录

### 11.1 关键代码示例

#### 11.1.1 完整 chat 命令实现

```python
# src/cli.py

from nanobot_ai import NanobotAgent
from src.agents.tools import RunnerTools, TOOL_DESCRIPTIONS
from src.core.storage import StorageManager
from rich.prompt import Prompt
from rich.console import Console

console = Console()

@app.command()
def chat():
    """
    启动自然语言交互模式
    """
    console.print("[bold green]🤖 Nanobot Runner Agent[/bold green]")
    console.print("[dim]基于 nanobot-ai 的本地跑步数据助理[/dim]")
    console.print("=" * 60)
    console.print("[dim]输入 'exit' 或 'quit' 退出[/dim]")
    console.print()
    
    # 初始化
    storage = StorageManager()
    tools = RunnerTools(storage)
    
    agent = NanobotAgent(
        tools=[tools],
        tool_descriptions=TOOL_DESCRIPTIONS,
        memory_window=10,
        local_model=True,
    )
    
    # REPL 循环
    while True:
        try:
            user_input = Prompt.ask("\n[bold cyan]您[/bold cyan]")
            
            if user_input.lower() in ["exit", "quit", "q"]:
                console.print("[yellow]再见！祝您跑步愉快！[/yellow]")
                break
            
            if not user_input.strip():
                continue
            
            with console.status("[bold green]思考中...", spinner="dots"):
                response = agent.chat(user_input)
            
            console.print()
            console.print(f"[bold magenta]Agent[/bold magenta] {response}")
            
        except KeyboardInterrupt:
            console.print("\n[yellow]检测到中断，已退出[/yellow]")
            break
        except Exception as e:
            console.print(f"[red]错误：{str(e)}[/red]")
```
