## 二、软件安装

### 2.1 环境要求

在开始之前，请确认您的电脑满足以下条件：

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 或 Linux |
| Python 版本 | 3.11 或更高版本 |
| 磁盘空间 | 至少 500MB 可用空间 |
| 网络连接 | 需要（用于下载依赖和访问 AI 服务）|

### 2.2 检查 Python 版本

打开 PowerShell（Windows）或终端（macOS/Linux），输入以下命令：

```powershell
python --version
```

**预期结果**: 显示 `Python 3.11.x` 或更高版本。

如果未安装或版本过低，请访问 [Python 官网](https://www.python.org/downloads/) 下载安装。

### 2.3 安装 uv 包管理器

`uv` 是一个快速的 Python 包管理工具，我们用它来安装软件。

在 PowerShell 中执行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**预期结果**: 显示安装成功的提示信息。

验证安装：

```powershell
uv --version
```

### 2.4 下载并安装 Nanobot Runner

#### 方式一：一键安装（推荐）

```powershell
# 创建安装目录
mkdir -p ~/.nanobot-runner-app
cd ~/.nanobot-runner-app

# 克隆项目
git clone https://github.com/yecllsl/nanobot-runner.git .

# 创建虚拟环境并安装依赖
uv venv
uv sync
```

#### 方式二：手动安装

```powershell
# 1. 选择安装位置（示例：用户目录下）
cd ~

# 2. 克隆项目
git clone https://github.com/yecllsl/nanobot-runner.git

# 3. 进入项目目录
cd nanobot-runner

# 4. 创建虚拟环境
uv venv

# 5. 激活虚拟环境（Windows PowerShell）
.venv\Scripts\Activate.ps1

# 6. 安装依赖
uv sync
```

**预期结果**: 
- 虚拟环境创建成功（显示 `.venv` 文件夹）
- 依赖安装完成，无错误提示

## 三、初始配置

### 3.1 初始化系统

运行初始化命令，创建必要的配置文件和数据目录：

```powershell
uv run nanobotrun system init
```

**预期结果**: 
- 显示 `✓ 初始化完成！`
- 创建 `~/.nanobot-runner/` 目录结构
- 配置文件路径: `~/.nanobot-runner/config.json`

### 3.2 配置 LLM API

Nanobot Runner 需要接入大语言模型（LLM）才能提供 AI 功能。支持以下服务商：

| 服务商 | 默认模型 | 特点 |
|--------|----------|------|
| OpenAI | gpt-4o-mini | GPT 系列，能力强，全球通用 |
| Anthropic | claude-3-5-sonnet-20241022 | 推理优秀，适合深度分析 |
| DeepSeek | deepseek-chat | 性价比高，中文能力出色 |
| 智谱 AI | glm-4.7-flash | 国内稳定，GLM 系列 |
| SiliconFlow | Qwen/Qwen3-235B-A22B | 聚合多模型，支持通义千问等 |
| other | 自定义 | 手动输入服务商和模型 |

#### 获取 API 密钥

1. 访问您选择的服务商官网注册账号
2. 进入 API 管理页面创建密钥
3. 复制密钥备用（通常以 `sk-` 开头）

> ⚠️ **安全提示**: API 密钥相当于密码，请勿分享给他人或上传到公开平台！

#### 配置步骤

初始化过程中会引导您完成 LLM 配置：

```powershell
uv run nanobotrun system init
```

按提示依次输入：

1. **选择 Provider**: 使用方向键选择服务商
2. **输入模型名称**: 确认或修改默认模型
3. **输入 API Key**: 粘贴您的密钥
4. **输入 Base URL**: 可选，留空使用默认地址

**配置示例**:

```
? 选择 LLM Provider: openai
? 输入模型名称: gpt-4o-mini
? 输入 API Key: sk-xxxxxxxxxxxxxxxxxxxxxxxx
? 输入 Base URL（可选，留空使用默认）:
✅ 配置已保存到 ~/.nanobot-runner/config.json
```

**预期结果**: 显示 `✅ 初始化完成`，并列出配置摘要。

### 3.3 验证配置

测试 LLM 连接是否正常：

```powershell
uv run nanobotrun agent chat
```

输入 `你好` 后按回车，**预期结果**: AI 回复问候语，表示配置成功。

输入 `exit` 或 `q` 退出交互模式。

## 四、导入跑步数据

### 4.1 准备数据文件

Nanobot Runner 支持导入 `.fit` 格式的运动数据文件，这是 Garmin、高驰(COROS)等运动手表的标准格式。

**数据来源**:
- Garmin Connect：导出活动记录
- 高驰手表：通过 APP 导出或同步
- 其他运动手表：导出为 FIT 格式

### 4.2 导入单个文件

假设您的 FIT 文件位于 `D:\运动数据\2024-05-20.fit`：

```powershell
uv run nanobotrun data import "D:\运动数据\2024-05-20.fit"
```

## 五、启动 WebUI

### 5.1 启动 Gateway 服务

WebUI 通过 Gateway 服务内建的 WebSocket 通道提供，执行以下命令启动：

```powershell
uv run nanobotrun gateway start --webui
```

`--webui` 标志启用 WebSocket 通道，允许通过浏览器与 Agent 实时交互。启用后，Gateway 会自动启动 WebSocket 服务并显示访问地址。

### 5.3 获取访问令牌（如需要）

WebUI 默认启用 Token 认证，如果界面提示需要 Token，通过 HTTP 请求获取临时令牌：

```powershell
curl http://127.0.0.1:8765/token
```

**返回示例**:
```json
{"token": "eyJhbGciOiJIUzI1NiIs...", "expires_in": 300}
```

复制输出的 Token 粘贴到 WebUI 登录框。令牌有效期为 5 分钟（由 `token_ttl_s` 控制），过期后需重新获取。

> **安全提示**: 生产环境建议保持 Token 认证开启（`websocket_requires_token: true`），并在配置中设置强密钥 `token_issue_secret`。

## 七、常见问题

### Q1: 启动 WebUI 时提示端口被占用

**解决方法**: 修改配置文件使用其他端口

编辑 `~/.nanobot-runner/config.json`，添加 websocket 配置：

```json
{
  "websocket": {
    "enabled": true,
    "host": "127.0.0.1",
    "port": 8766
  }
}
```

然后重启 Gateway 服务，访问 `http://127.0.0.1:8766`

### Q2: 导入数据时提示格式不支持

**解决方法**: 确保文件是 `.fit` 格式。如果是 `.gpx`、`.tcx` 等格式，需要先用其他工具转换为 FIT 格式。

### Q3: AI 回复很慢或超时

**可能原因**:
1. LLM API 响应慢
2. 网络连接问题
3. 数据量过大

**解决方法**:
- 检查网络连接
- 尝试更换 LLM 服务商
- 减少查询的时间范围（如从 90 天改为 30 天）

### Q4: 如何修改 LLM 配置

手动编辑配置文件：

```powershell
notepad ~/.nanobot-runner/config.json
```

或重新运行初始化向导：

```powershell
uv run nanobotrun system init --force
```

### Q5: 数据存储在哪里

所有数据存储在本地：

```
~/.nanobot-runner/
├── data/           # 跑步数据（Parquet 格式）
├── config.json     # 配置文件
├── memory/         # 记忆数据
└── sessions/       # 会话历史
```

### Q6: 如何备份数据

数据备份功能将在后续版本提供。目前建议手动备份数据目录：

```powershell
# 复制数据目录到备份位置
Copy-Item -Recurse ~/.nanobot-runner/data ~/.nanobot-runner/data_backup_$(Get-Date -Format "yyyyMMdd")
```
