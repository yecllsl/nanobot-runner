## 问题一：飞书日历异步测试 (14个失败)

### 错误信息
```
RuntimeError: There is no current event loop in thread 'MainThread'.
```

### 根本原因
测试代码使用 `asyncio.get_event_loop().run_until_complete()` 运行异步方法，但在 Python 3.10+ 中，如果没有运行中的事件循环，`get_event_loop()` 会抛出错误。

### 受影响的测试
- `TestFeishuCalendarAPI::test_create_event`
- `TestFeishuCalendarAPI::test_update_event`
- `TestFeishuCalendarAPI::test_delete_event`
- `TestFeishuCalendarAPI::test_get_event`
- `TestFeishuCalendarAPI::test_get_calendar_list`
- `TestFeishuCalendarSync::test_sync_plan_success`
- `TestFeishuCalendarSync::test_sync_plan_disabled`
- `TestFeishuCalendarSync::test_sync_plan_no_api`
- `TestFeishuCalendarSync::test_sync_daily_workout_success`
- `TestFeishuCalendarSync::test_sync_daily_workout_disabled`
- `TestFeishuCalendarSync::test_update_event_success`
- `TestFeishuCalendarSync::test_delete_event_success`
- `TestFeishuCalendarSync::test_check_conflicts_no_api`
- `TestIntegration::test_full_sync_workflow`

### 修复方案
将所有 `asyncio.get_event_loop().run_until_complete()` 替换为 `asyncio.run()`：

```python
# 修复前
result = asyncio.get_event_loop().run_until_complete(api.create_event(...))

# 修复后
result = asyncio.run(api.create_event(...))
```

### 修改文件
- `tests/unit/notify/test_feishu_calendar.py`

## 问题二：Analytics 列名不匹配 (33个失败)

### 错误信息
```
unable to find column "session_start_time"; valid columns: ["total_distance", "total_timer_time", "avg_heart_rate"]
unable to find column "session_avg_heart_rate"; valid columns: [...]
unable to find column "session_total_timer_time"; valid columns: [...]
```

### 根本原因
`AnalyticsEngine` 的方法（如 `get_running_summary`, `get_running_stats`, `get_heart_rate_zones`, `get_pace_distribution`）期望数据包含 `session_` 前缀的列名：
- `session_start_time`
- `session_total_distance`
- `session_total_timer_time`
- `session_avg_heart_rate`

但测试数据使用的是简单的列名：
- `total_distance`
- `total_timer_time`
- `avg_heart_rate`

### 受影响的测试
- `TestAnalyticsEngine::test_get_running_summary_success`
- `TestAnalyticsEngine::test_get_running_summary_with_date_filter`
- `TestAnalyticsEngineAdvanced::test_get_running_summary_with_single_record`
- `TestAnalyticsEngineAdvanced::test_get_running_stats_success`
- `TestHeartRateZones::test_get_heart_rate_zones_with_avg_heart_rate`
- `TestHeartRateZones::test_get_heart_rate_zones_avg_hr_no_valid_data`
- `TestHeartRateZones::test_calculate_zones_from_avg_hr_success`
- `TestPaceDistribution::*` (10个测试)

### 修复方案
修改测试数据，使用正确的列名格式：

```python
# 修复前
mock_df = pl.DataFrame({
    "total_distance": [5000.0, 10000.0],
    "total_timer_time": [1800, 3600],
    "avg_heart_rate": [140, 150],
})

# 修复后
mock_df = pl.DataFrame({
    "session_start_time": [datetime(2024, 1, 1, 6, 0), datetime(2024, 1, 2, 6, 0)],
    "session_total_distance": [5000.0, 10000.0],
    "session_total_timer_time": [1800, 3600],
    "session_avg_heart_rate": [140, 150],
})
```

### 修改文件
- `tests/unit/test_analytics.py`

## 问题三：Schema 类型转换 (1个失败)

### 错误信息
```
polars.exceptions.InvalidOperationError: conversion from `str` to `datetime[μs]` failed in column 'timestamp' for 1 out of 1 values: ["2024-01-01"]
```

### 根本原因
`ParquetSchema.normalize_dataframe()` 方法在转换 `timestamp` 列时，字符串 `"2024-01-01"` 无法直接转换为 datetime 类型。

### 受影响的测试
- `TestParquetSchema::test_normalize_dataframe_convert_types`

### 修复方案
修改 `src/core/schema.py` 中的 `normalize_dataframe` 方法，对 timestamp 列使用 `str.to_datetime()` 进行转换：

```python
# 在 normalize_dataframe 方法中添加
if "timestamp" in df.columns and df["timestamp"].dtype == pl.String:
    df = df.with_columns(
        pl.col("timestamp").str.to_datetime().alias("timestamp")
    )
```

### 修改文件
- `src/core/schema.py`
