## 📝 实施步骤

### 阶段一：环境准备与分支清理

#### 步骤 1: 确认远程仓库状态
```bash
# 获取最新远程分支和标签
git fetch --all --prune --tags
```

**目的**：确保本地仓库与远程同步，清理已删除的远程分支引用

#### 步骤 2: 创建 develop 分支（从 main 分支）
```bash
# 基于 main 分支创建 develop 分支
git checkout main
git checkout -b develop
git push -u origin develop
```

**目的**：建立标准的双分支模型（main + develop），符合 Git Flow 规范

#### 步骤 3: 清理旧的功能分支（可选）
```bash
# 查看 feature/v0.3.0 分支内容
git log feature/v0.3.0 --oneline -5

# 如果确认不再需要，删除本地和远程分支
git branch -d feature/v0.3.0
git push origin --delete feature/v0.3.0
```

**目的**：保持仓库整洁，避免分支混乱

### 阶段二：创建 v0.4.0 发布分支

#### 步骤 4: 创建 v0.4.0 发布分支
```bash
# 确保在 develop 分支上
git checkout develop
git pull

# 创建 v0.4.0 发布分支
git checkout -b release/v0.4.0
```

**目的**：创建发布准备分支，用于 v0.4.0 的预检和最终发布

#### 步骤 5: 更新 v0.4.0 版本号
```bash
# 更新 pyproject.toml: 0.3.1 → 0.4.0
# 更新 AGENTS.md: 0.3.1 → 0.4.0
# 更新 README.md: v0.3.1 → v0.4.0

# 提交版本变更
git add pyproject.toml AGENTS.md README.md
git commit -m "chore(version): bump version to 0.4.0 for release"
```

**目的**：为 v0.4.0 版本设置正确的版本号

#### 步骤 6: 推送发布分支触发预检
```bash
# 推送 release/v0.4.0 分支到远程
git push -u origin release/v0.4.0
```

**目的**：触发 `.github/workflows/release-pre-check.yml` 预检 workflow

### 阶段三：预检验证

#### 步骤 7: 监控预检 workflow
```bash
# 访问 GitHub Actions 查看预检状态
# https://github.com/yecllsl/nanobot-runner/actions
# 筛选器：release-pre-check workflow
```

**检查项**：
- ✅ Code Quality Checks (black, isort, mypy, bandit)
- ✅ Tests (Python 3.11, 3.12)
- ✅ Build Package
- ✅ Pre-Check Summary

#### 步骤 8: 处理预检失败（如有）
```bash
# 如果预检失败，在 release/v0.4.0 分支修复
git checkout release/v0.4.0
# ... 进行代码修复 ...
git add .
git commit -m "fix: 修复预检发现的问题"
git push origin release/v0.4.0
# 重新触发预检 workflow
```

### 阶段四：正式发布（预检通过后）

#### 步骤 9: 合并到 main 分支
```bash
# 确认预检通过后，合并到 main
git checkout main
git pull
git merge release/v0.4.0
```

#### 步骤 10: 创建并推送标签
```bash
# 创建标签
git tag -a v0.4.0 -m "版本 0.4.0 发布"

# 推送代码
git push origin main

# 推送标签（明确指定，避免 --tags）
git push origin v0.4.0
```

**⚠️ 关键注意事项**：
- ✅ 使用 `git push origin v0.4.0`（只推送新标签）
- ❌ 避免 `git push origin main --tags`（会推送所有标签）

#### 步骤 11: 合并回 develop 分支
```bash
# 合并 main 到 develop
git checkout develop
git merge main
git push origin develop
```

#### 步骤 12: 删除发布分支
```bash
# 删除本地分支
git branch -d release/v0.4.0

# 删除远程分支
git push origin --delete release/v0.4.0
```

## 📝 备注

### 标签推送最佳实践
```bash
# ✅ 推荐：分步推送
git push origin main
git push origin v0.4.0

# ✅ 推荐：使用 && 组合
git push origin main && git push origin v0.4.0

# ❌ 避免：使用 --tags
git push origin main --tags
```

### 预检失败处理
如果预检 workflow 失败：
1. 在 `release/v0.4.0` 分支修复代码
2. 重新推送分支触发预检
3. **不要**创建标签，直到预检通过

### 版本管理策略
如果正式发布时 CI/CD 失败：
- **推荐**：修复后发布 v0.4.1 补丁版本
- **备选**：删除标签重打（仅限早期项目，需谨慎）
