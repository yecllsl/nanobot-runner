# 指令手册

本手册定义产品及项目经理、架构师、开发工程师、测试工程师、发布运维工程师五类智能体的核心执行指令。

---

## 1. 角色定义

| 角色 | 核心职责 | 关键交付物 |
|------|----------|-----------|
| **产品及项目经理** | 产品规划、需求管理、项目管理、进度跟踪 | 产品规划方案、需求池、项目计划、进度报告 |
| **架构师** | 需求分析、架构设计、任务拆解、方案评审 | 需求规格说明书、架构设计方案、任务清单 |
| **开发工程师** | 功能开发、单元测试、Bug修复、代码优化 | 功能代码、单元测试、开发交付报告 |
| **测试工程师** | 测试策略、用例设计、测试执行、质量评估 | 测试用例、测试报告、Bug清单、上线结论 |
| **发布运维** | CICD搭建、自动化发布、故障排查、监控配置 | 流水线配置、发布报告、运维文档 |
| **代码评审工程师** | 代码质量评审、架构一致性检查、安全审计 | 代码评审报告 |

---

## 2. 核心指令

### 2.1 产品及项目经理指令

| ID | 指令 | 目的 | 输入 | 输出 | 验收标准 |
|----|------|------|------|------|----------|
| PM-01 | 产品规划 | 输出产品规划方案 | 市场调研、业务目标 | `/docs/product/产品规划方案.md` | 产品愿景清晰，路线图合理，版本规划可执行 |
| PM-02 | 需求管理 | 管理需求全生命周期 | 需求来源（用户反馈、市场调研） | `/docs/product/需求池.md` + `/docs/product/需求分析报告.md` | 需求无歧义，优先级合理，验收标准可量化 |
| PM-03 | 项目管理 | 制定项目计划并管理风险 | 任务清单、资源配置 | `/docs/planning/项目计划.md` + `/docs/planning/风险管理计划.md` | 项目计划完整，资源分配均衡，风险识别全面 |

## 3. 指令调用示例

### 3.1 产品规划 (PM-01)

```markdown
输入：
- 市场调研：跑步数据管理工具市场分析
- 业务目标：打造个人跑步数据助理

执行步骤：
1. 定义产品愿景和核心价值主张
2. 制定产品路线图（探索期→成长期→成熟期）
3. 规划版本迭代（MVP v0.1.0 → v1.0.0）
4. 评估资源需求和风险

输出：
- /docs/product/产品规划方案.md
```

### 3.2 需求分析 (ARC-01)

```markdown
输入：
- 项目名称：训练计划制定与飞书日历同步
- 核心场景：用户通过自然语言创建训练计划，同步到飞书日历

执行步骤：
1. 拆解业务诉求为功能点
2. 区分MVP核心需求与非核心需求
3. 定义每个需求的验收标准
4. 输出标准化文档

输出：
- /docs/requirements/PRD_训练计划制定与飞书日历同步.md
```

### 3.3 功能开发 (DEV-01)

```markdown
输入：
- 任务清单：/docs/planning/task_list_v0.5.0.md
- 架构方案：/docs/architecture/架构设计说明书.md

执行步骤：
1. 按任务优先级开发
2. 编写单元测试
3. 本地验证核心场景
4. 输出交付报告

输出：
- 代码提交至Git分支
- 单元测试：/tests/unit/
- 交付报告：/docs/development/
```

### 3.4 自动化发布 (OPS-02)

```markdown
输入：
- 上线结论：建议上线
- 版本号：v0.5.0

执行步骤：
1. 合并develop至main
2. 创建版本Tag
3. 推送GitHub触发CICD
4. 验证线上服务

输出：
- GitHub版本Tag
- 发布报告：/docs/devops/release.md
```
